# Susan Fowler 加入 Stripe，担任新季度出版物 Increment 的主编

Stripe 今天推出了一份名为 [Increment](https://web.archive.org/web/20221006233307/http://increment.com/) 的新季度出版物，其中包含针对工程受众的观点文章、专家访谈和操作指南。为了管理网站和所有为其制作的内容，该公司宣布聘请[苏珊·福勒](https://web.archive.org/web/20221006233307/https://www.linkedin.com/in/susan-fowler-263b6343/)担任主编。

Increment 背后的想法是创建一个独立的网站来探索对软件工程师和团队来说很重要的不同主题。它旨在充当 Stripe 和业内其他人共享知识的中央存储库，包括构建软件和扩展技术业务的技巧和最佳实践。

掌舵的是福勒，一位工程师和《T4》作者，他写了一本关于在软件团队中构建标准化系统的书，并在像 Plaid、PubNub 和优步这样的科技公司工作过。然而，最近，福勒在发表了一篇博客文章[公开指责优步人力资源部门](https://web.archive.org/web/20221006233307/https://www.susanjfowler.com/blog/2017/2/19/reflecting-on-one-very-strange-year-at-uber)[忽视公司内部性骚扰和性别歧视的多份报告](https://web.archive.org/web/20221006233307/https://beta.techcrunch.com/2017/02/19/former-uber-engineer-says-company-ignored-repeated-reports-of-harassment/)后，成为公众关注的焦点。

这篇文章是这家打车公司一系列丑闻和挫折的一部分。与此同时，福勒一直在 Stripe 工作，试图让增量离开地面。她无法接受采访，部分原因是与正在进行的对她的指控的调查有关的法律问题。

据 Stripe 的代表称，该网站的框架是在福勒和 Stripe 的联合创始人帕特里克·科利森会面讨论她的博客和写作后形成的。这最终导致了后来成为增量的讨论。在提出他们的蓝图时，他们受到了诸如[高可扩展性](https://web.archive.org/web/20221006233307/http://highscalability.com/)、 [DZone](https://web.archive.org/web/20221006233307/https://dzone.com/) 、 [InfoQ](https://web.archive.org/web/20221006233307/https://www.infoq.com/) 、 [C2 维基](https://web.archive.org/web/20221006233307/http://wiki.c2.com/)和 Y Combinator 的[宏](https://web.archive.org/web/20221006233307/https://blog.ycombinator.com/the-macro/)等网站的启发。

虽然 Stripe 推出一个围绕大规模开发软件提供建议和技巧的网站可能看起来很奇怪，但这是在技术社区内建立知识共享工具的更广泛计划的一部分。为此，该公司最近[推出了一个论坛](https://web.archive.org/web/20221006233307/https://beta.techcrunch.com/2017/04/06/stripe-atlas-us/)作为其 Stripe Atlas 计划的一部分，并于本周早些时候宣布[收购了知识共享社区独立黑客](https://web.archive.org/web/20221006233307/https://beta.techcrunch.com/2017/04/11/stripe-acquires-indie-hackers-a-knowledge-sharing-community-for-entrepreneurs/)。

对于增量，计划是每期关注一个特定的主题。本季度，该出版物完全是关于软件团队如何处理随叫随到和事件响应的，但一位发言人表示，Stripe 打算在网站上分享即将到来的问题的主题，并将鼓励读者提交关于专题和未来观点的建议。
