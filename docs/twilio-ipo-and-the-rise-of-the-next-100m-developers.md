## 每个人都可以不用代码来构建

开发人员不应该是唯一可以利用 API 的人，因为，虽然拥有它们很好(我们很高兴拥有我们所有的 API！)，他们来之不易。随着我们越来越多的业务功能依赖于不同的 SaaS 服务，citizen 开发者正在寻找增强自身能力和连接应用的方法。他们想要一个可定制的解决方案，这需要开发人员，但不需要开发人员。

Salesforce 平台是这方面的先驱，它是第一个允许非编码人员(“管理员”)定制应用程序、创建新的数据库和工作流以及构建报告的云应用程序。在 Salesforce 任职期间，我目睹了许多生活和职业的转变，因为不认为自己是“技术人员”的人开始构建和定制应用程序，成为他们公司的关键人物。

新的零代码工具箱包括但不限于:

*   Salesforce —客户和账户相关应用的 CRM 平台
*   Box.com—存储平台
*   Twilio —完全通过短信管理所有申请人互动所需的一切
*   谷歌表单——交互式问题和答案出现在谷歌表单中
*   Workato —一个强大的无代码平台，可跨这些应用程序自动执行工作流

## 云支持无需代码的应用

像亚马逊网络服务、Azure 和其他云平台允许开发者从服务器的日常供应和管理业务中解脱出来。但是像 [Salesforce](https://web.archive.org/web/20221226023344/https://www.salesforce.com/) 、 [Box](https://web.archive.org/web/20221226023344/https://www.box.com/) 、 [Twilio](https://web.archive.org/web/20221226023344/https://www.twilio.com/) 等云 app 平台。更进一步，甚至将代码层隐藏在干净的 API 后面。使用像 [Workato](https://web.archive.org/web/20221226023344/http://workato.com/) 、 [Boomi](https://web.archive.org/web/20221226023344/http://www.boomi.com/) 或 [Snaplogic](https://web.archive.org/web/20221226023344/http://www.boomi.com/) 这样的软件，这些 API 可以在没有代码的情况下连接在一起，创建一个工作流，调用这些应用程序来做诸如创建线索、删除文件、发起电话呼叫等事情。

> 软件正在吞噬世界，但我们没有足够多的开发者来足够快地吞噬世界。

以 Twilio 为例，您不需要知道 SIP 服务器是什么，也不需要将它部署到应用服务器上——使用 Twilio API，您只需传递正确的参数，电话或短信就会发出。

## n ext 大投资机会

使用像 Twilio 这样的云应用的公司不是单独使用它们，而是与其他云应用或他们自己的软件一起使用。我们可以很容易地看到用几个应用程序创建一个全面的工作流来构建整个流程的影响。

以一家依赖源源不断的千禧一代临时工来维持服务运营的公司为例。他们的解决方案需要回答以下几个问题:你如何让大量申请者轻松申请？如何简化大批量筛选流程？如何自动化入职流程？

如果没有开发人员，您将需要一套不同的工具，但是您可以获得相同的结果。通过使用各种云应用和灵活且无需代码的集成平台，无需定制开发即可创建定制工作流。

感兴趣的申请人可以通过 Twilio 发送短信申请，并附上简历照片。他们的信息将自动上传到 Box，招聘经理可以手动接受或拒绝申请人，而不会破坏自动工作流程。

如果经理批准了，申请人会进入 Box 中的一个新文件夹，Twilio 会给他们发一条短信，让申请人知道他们被录用了。同时，Workato 将通过 SurveyMonkey 进行测试，以确保申请人适合临时工作；完成后，经过全面审查的申请人就可以走了。

软件正在吞噬世界，但我们没有足够多的开发者来足够快地吞噬世界。使下一个 1 亿人能够构建应用程序，使我们的日常工作自动化，连接我们的移动应用程序并定制我们的报告，这有可能成为比数十亿美元的 IT 堆栈更大的业务。
