亚马逊推出 AWS 物联网——一个用于构建、管理和分析物联网的平台

为另一个进入物联网领域的大玩家让路。亚马逊今天在拉斯维加斯的 re:Invent 开发者大会上宣布了期待已久的 AWS 物联网平台。

正如亚马逊所描述的，它是一个托管的云平台，“让互联设备轻松安全地与云应用程序和其他设备进行交互。”该平台正处于测试阶段，将能够支持数十亿设备和数万亿条消息，“并能够可靠安全地处理和路由这些消息到 AWS 端点和其他设备。”

AWS 物联网将与 Lambda、亚马逊 Kinesis、亚马逊 S3、亚马逊机器学习和亚马逊 DynamoDB 集成，以构建物联网应用程序、管理基础设施和分析数据。

“互联并不一定意味着有用，”亚马逊首席技术官沃纳·威格尔在今天的主题演讲中说，事实上这也是亚马逊旨在通过其大型平台解决的问题:在这个平台上，为不同互联对象创建服务的许多不同部分可以汇集到一个地方，以一种连贯的方式运行(所有这些最终也会为一家整合的公司带来流量和收入:亚马逊)。

沃格尔斯指出，该平台将包括一个设备网关、一个基于规则的引擎、一个保存设备数据的注册表(沃格尔斯提到的例子是“给我这个特定房间里的所有设备”)，以及一个被亚马逊称为设备阴影的功能，该功能允许你在设备离线时跟踪设备数据，并在设备恢复在线后将其发送回系统。

今年早些时候，公司收购了物联网平台2 商店，所以今天的宣布并不令人惊讶，特别是考虑到亚马逊的一些竞争对手如微软最近已经将物联网支持作为其云计算战略的基石之一。

该平台将使开发人员更容易从使用标准网关和协议(如 MQTT 和 HTTPS)的设备获取输入数据。

与其他 AWS 服务一样，物联网平台的定价基于你使用的东西，没有最低费用。价格基于发布到 AWS IoT 的消息数量(发布成本)，以及 AWS IoT 交付到设备或应用程序的消息数量(交付成本)。为了让你留在 AWS 生态系统中，定价不包括某些 AWS 服务的交付，特别是亚马逊 S3、亚马逊 DynamoDB、AWS Lambda、亚马逊 Kinesis、亚马逊 SNS 和亚马逊 SQS。
