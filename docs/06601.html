物联网(IoT)的预期规模有可能让任何人从外部看起来都有广场恐惧症。然而，从内部来看，观点却大相径庭。仔细观察，它是由一系列复杂的组织排列成的一系列错综复杂的线索。

与任何新的技术时代一样，围绕形态、所有权和监管的问题开始出现。想象一下试图重新建立互联网。就像那样，但是规模更大。

第一个障碍是技术标准。我们正处于物联网发展的关键时刻。随着互联事物的多样性增加，不允许每个“事物”相互交流的潜在风险也在增加。

这要从网络标准说起。从 ZigBee 到 Z-Wave、EnOcean、Bluetooth LE 或 SigFox 和 LoRa，有太多竞争和不兼容的网络标准可供选择。幸运的是，事情似乎正在融合和巩固。

此外，已经稳固的联盟正在重组。首先在室内世界，ZigBee 3.0(T4)正越来越接近谷歌的线程(T7)，尽管仍然受到即将发布蓝牙网状标准的蓝牙联盟的挑战。更有趣的是，Wi-Fi 联盟正在研究被称为 HaLow 的 IEEE 802.11ah。所有三个标准都专门针对物联网的更低功耗要求和更好的覆盖范围。

同样，在户外世界，下一代移动网络(NGMN)联盟(与久负盛名的 GSMA 密切合作，统治着移动标准世界)正在致力于智能世界的一个重要拼图:5G。随着数据范围的增加、延迟的降低和覆盖范围的扩大，处理大量的单个连接变得至关重要，并将成为现有 LPWAN(低功耗无线区域网络)的严重全球竞争对手，如 SigFox 和 LoRa 。

安全性是阻碍主流消费者采用物联网的最大障碍之一。

虽然目前正在进行试验，但商业部署预计要到 2020 年。在此之前，必须完成频谱拍卖；通常情况下，政府会裁决科技公司和电信公司之间的纠纷，战线以价格为基础。重要的是尽早与监管机构合作，以确保以鼓励物联网蓬勃发展的成本提供足够的频谱，而不是受制于膨胀的批发价格。

但是挑战并不止于网络层面；数据或应用程序级别也是游戏的一大部分。随着科技巨头开始试图占领这一领域，应用协议的分歧只会越来越大。苹果 HomeKit、谷歌 Weave 和许多其他公司都在试图推广自己的生态系统，每个公司都有自己的商业议程。

如果任由其以不受管理的方式发展，我们最终将会采用独立的不同方法，不可原谅地限制物联网作为开放生态系统运行的能力。这是一部我们以前看过的电影。

网络已经经历了这个混乱的过程，最终通过达尔文式的技术原则和使用实践使自己标准化。web 为 Internet 提供了一个简单且可扩展的应用层，这是一组标准，Internet 的任何节点都可以使用它用来连接 Internet 的任何物理技术。

网络使互联网变得有用并最终成功。这就是物联网(WoT)方法至关重要的原因。这种方法已经得到了大量支持。一个物联网模型最近被提交给 W3C，该模型基于由科技巨头、初创公司和学术机构共同完成的研究。这些都是面向物联网开放和单一愿景的初步尝试。

这个问题的解决开辟了一个巨大的协作网络的可能性，在这个网络中，统一的数据可以优化大量现有的流程。然而，随着数据逐渐成为一系列曾经无生命的物体中最有价值的资产，这对那些制造之前没有数据战略的产品的传统公司来说意味着什么？

技术部门乐于分享和使用这类信息，但对于那些从灯泡到汽车都有制造基础的公司来说，这是一个新概念。传统上，这类组织的运营方式要封闭得多，将数据像知识产权一样对待——一些应该锁起来的东西。

要改变这一点，任何企业都需要进行文化转变。尽管这并不是不可克服的，但它凸显了改变董事会内部思维定势的必要性。要实现这样的巨变，需要教育、人力资源和技术投资。

智能对象的安全性取决于其最薄弱的连接环节。

安全性是阻碍主流消费者采用物联网的最大障碍之一。Fortinet 的一项调查发现，68%的全球房主担心联网设备的数据泄露。它们应该是:快速浏览一下 Shodan ，一个物联网搜索引擎，让你即时访问数以千计的不安全物联网设备，包括婴儿监视器！2015 年，美国美国联邦贸易委员会表示，“感知到的隐私和安全风险……削弱了消费者对技术充分发挥潜力所必需的信心。”

对于制造商来说，要提高消费者的信心，他们必须能够证明他们的产品是安全的，这一点最近似乎受到了越来越大的压力。安全性的问题在于，它根本就没有实现。安全性是一场与时间赛跑的持久战，需要及时部署补丁和改进。

这显然会让产品制造商不知所措。为了做到这一点，依靠已经建立的物联网平台是明智之举，该平台已经实施了全面而强大的安全方法，可以引导他们通过这样一个复杂的领域。

消费者也在增加他们的数据安全方面分担一些责任——通过为产品用户帐户和面向互联网的设备(如路由器或智能设备)使用强密码；设置 Wi-Fi 网络时使用加密技术(如 WPA2)；并及时安装任何软件更新。

然而，随着消费者越来越多地采用物联网，制造商必须确保智能互联产品的安全性是其物联网战略的核心。毕竟，智能对象的安全性取决于其最薄弱的连接环节。

加上安全性，围绕数据隐私、共享和使用的紧急问题将成为每个人都必须解决的问题，而不仅仅是科技公司。在数据驱动的物联网世界中，共享的数据比当前的数字经济更加个人化和私密化。

例如，消费者能够通过他们的浴室秤交易受保护的数据，如健康和医疗信息，也许是为了更好的健康保险费。但是，如果消费者本应该减肥，结果反而增加了，会发生什么呢？消费者可以对访问他们的数据施加什么控制，会有什么后果？

消费者应该拥有精细的数据共享控制(而不是全有或全无的共享)，并且应该能够将他们拥有和产生的数据货币化。消费者还应该与产品制造商签订一份“合同”，随着时间的推移进行调整——无论是主动还是自动的——并阐明数据共享中的裂缝或数据本身不利的情况的影响。

监管机构也有责任确保法律框架到位，从一开始就在物联网的核心建立信任。行业需要接受这一点，并从一开始就与用户进行开放和诚实的对话。知情同意从未如此重要，因为来自互联设备的数据和元数据能够构建个人的超个性化图片。

品牌应该明白，即将涌入的消费者数据是一种潜在的收入流，必须加以保护和培育。因此，对隐私和尊重的认知等同于与客户的长期互动。以至于产品制造商可能会开始改变他们的商业模式，以创造数据共享的激励机制，甚至可能免费赠送他们的产品。

由于其巨大的潜力，物联网正在快速发展，主要由技术公司和学术机构推动。然而，只有通过该群体之外的大规模教育和合作，它才能真正实现全面发展，并使我们的流程、资源利用以及最终我们的生活变得更好。
