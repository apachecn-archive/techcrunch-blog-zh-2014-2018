# Strix Leviathan 希望为加密交易构建更好的企业平台 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/01/26/strix-leviathan-wants-to-build-a-better-enterprise-platform-for-crypto-trading/>

我们仍然处于加密货币的早期——或者至少这是所有进入这个领域的初创公司现在所希望的。当出现淘金热时，向矿工出售铲子(或 ASIC 钻机)可以赚很多钱。因此，毫不奇怪，我们现在也看到了一类新的 B2B 加密创业公司的出现(想想交易技术和 TradeDesk)。

总部位于西雅图的 Strix Leviathan 是这一行业的最新成员之一，该公司正在为对冲基金、银行和传统企业等不想建立自己的交易基础设施的大型实体建立一个加密交易平台。

Strix Leviathan 由 Jesse Proudman 创建，他之前在 2003 年创建了 Blue Box，这是一家专注于私有云部署的云计算公司，然后在 2015 年将其出售给 IBM。

Proudman 告诉我，他不久前开始跟踪加密货币市场，但直到去年夏天，当他在 IBM 获得一个常驻企业家的职位时，他才开始关注这个问题。“这套技术让我想起了 1997 年我的初创公司——互联网刚刚起步的时候，”他告诉我。他认为，我们今天拥有的是一套已经在现实世界中部署的技术，它们提供了一个足够大的生态系统，但存在一个巨大的用户体验问题，因为需要几个月的学习才能真正理解这些系统。

这个项目开始是因为 Proudman 想要为加密货币建立自己的算法交易平台，这看起来是一个及时的想法，因为简单的 [hodling](https://web.archive.org/web/20221207214720/https://www.marketwatch.com/story/are-you-hodling-bitcoin-2017-09-12) 的日子似乎即将结束。不过，在这个过程中，他也了解了许多主要交易所的错综复杂和古怪之处。“我们在市场上处于一个不成熟的时期，许多交易所都存在 API 问题，”他告诉我。这意味着，举例来说，一个交易所可能永远不会确认交易是否成功，或者一个请求是否超时。

通过 Strix Leviathan，Proudman 和他的团队现在正在构建一个可插拔的交易引擎，它可以处理新交易所上线时的支持以及平台的特性。

在其当前的化身中，Strix Leviathan(这个名字不会真的脱口而出吧？)由两部分组成:一个是数据摄取引擎，用于获取各种加密货币价值的历史数据和相关数据，以便该公司构建其自动交易算法；另一个是交易执行引擎。现在的焦点似乎是在执行引擎上，Proudman 认为，现有的加密公司可能会对它感兴趣，因为他们对当前的交易基础设施不满意。该平台目前支持所有主要交易所，包括 GDAX、Bitfinex、币安等。

至于业务的自动化交易方面，该团队仍在努力确定如何继续进行以及如何将其推向市场。

该项目目前是自筹资金，但鉴于 Proudman 的记录，我们可能很快就会看到一轮融资。
