# Snyk 以 1 亿美元的估值筹集了 2200 万美元，用于检测开源代码中的安全漏洞

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/09/25/snyk-raises-22m-on-a-100m-valuation-to-detect-security-vulnerabilities-in-open-source-code/>

开源软件现在是一个价值 140 多亿美元的市场，并且正在快速增长，95%的企业都在以这样或那样的方式使用它。但这种扩张也伴随着一个阴影:开源组件可能会带来漏洞，因此它们在应用程序中的广泛使用会成为公司网络安全的负担。

现在，英国一家名为 [Snyk](https://web.archive.org/web/20230402111220/http://snyk.io/) 的初创公司已经建立了一种检测这些应用程序或组件何时被入侵的方法，该公司宣布了一轮 2200 万美元的融资，以满足希望正面解决这一问题的企业的需求。

由 Accel 牵头，GV 加上之前的投资者 Boldstart Ventures 和 Heavybit 参与，这是 Snyk 在七个月内筹集的第二轮资金——它在 3 月筹集了 700 万美元[的 A 轮资金。这是衡量该公司发展状况的一个指标(也是衡量投资者对该公司迄今为止的成就有多热情的一个指标)。这家初创公司没有透露其估值，但一位接近交易的消息人士称，目前该公司的估值约为 1 亿美元(迄今已筹集约 3300 万美元)。](https://web.archive.org/web/20230402111220/https://techcrunch.com/2018/03/06/snyk-snares-7-million-investment-to-help-developers-secure-open-source-code/)

作为衡量 Snyk 增长的另一项指标，该公司表示，它现在拥有 200 多名付费客户和 15 万名用户，收入在过去九个月中增长了五倍。今年 3 月，它有 130 名付费用户。

(该公司表示，目前的客户包括 ASOS、数字海洋、New Relic 和 Skyscanner。)

Snyk 正好处于当今企业服务格局的中间。它为组织提供了内部使用、云使用或两者混合使用的选择，并提供了一系列付费和免费级别，让用户熟悉这项服务。

该公司首席执行官盖伊·波德贾尼(Guy Podjarny)与阿萨夫·赫夫茨(Assaf Hefetz)和丹尼·格兰德(Danny Grander)共同创立了 Snyk，他解释说，Snyk 分为两个部分。首先，这家初创公司建立了一个“监听开源活动”的威胁情报系统 Snyk 利用开放对话平台——例如 GitHub commits 和 forum chatter——使用机器学习来检测潜在的漏洞提及。然后，它将这些信息传递给一组人类分析师，“他们在我们的漏洞数据库中验证和筛选真实的信息。”

其次，该公司分析了源代码库——同样包括 GitHub 和 bit bucket——“以了解每一个都使用了哪些开源组件，标记出易受攻击的组件，然后通过提出正确的依赖版本并通过我们的安全团队构建的补丁来自动修复它们。”

他补充说，开源组件并不比闭源组件有更多的漏洞，“但它们的大量重用使这些漏洞更具影响力。”组件可以用于成千上万的应用程序中，根据 Snyk 的估计，[大约 77%的应用程序](https://web.archive.org/web/20230402111220/https://snyk.io/blog/77-percent-of-sites-use-vulnerable-js-libraries)最终会使用有安全漏洞的组件。“因此，一个组织被易受攻击的开源组件攻破的几率远远大于纯粹的代码安全漏洞。”

Podjarny 表示，该计划不是为了长期解决专有代码问题，而是为了扩大其监控基于开源的应用程序的方式。

“我们的重点在两个方面——开发开发者喜欢的安全工具，以及修复开源安全，”他说。“我们认为，不安全使用开源代码的风险远远大于您自己的代码，并且在行业中没有得到很好的解决。我们确实打算扩大我们的保护范围，从修复开源组件中的已知漏洞，到在运行时监控和保护它们，标记和遏制恶意和受损的组件。”

虽然对于安全团队来说，这是一个相对较新的监控和解决领域，但他补充说，Equifax 违规凸显了如果此类问题未被检测到，在最糟糕的情况下可能会发生什么。Snyk 并不是唯一一家发现市场缺口的公司。Black Duck 专注于标记不合规的开源许可证，并提供一些安全特性。

然而，Snyk 似乎是目前最受关注的，它的名字来源于一个关于“潜行”一词的游戏，结合了意思是“所以现在你知道了”的首字母缩写词。

“近年来一些最大的数据泄露事件是由于开源依赖关系中未修复的漏洞造成的；因此，我们看到监控和修复此类漏洞的工具的采用呈指数级增长，”Accel 的合伙人 Philippe Botteri 说，他将参加本轮董事会。“我们也看到应用安全的所有权正在向开发者转移。我们认为，鉴于 Snyk 团队深厚的安全领域知识和以开发人员为中心的心态，Snyk 在市场中处于独特的地位，我们很高兴能够加入他们的行列，为开发人员带来安全工具。”
