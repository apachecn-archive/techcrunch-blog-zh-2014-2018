谷歌解决了 AMP 的一个大问题，现在可以让你查看和分享出版商自己的链接

原文：https://web.archive.org/web/https://techcrunch.com/2017/02/06/google-fixes-a-big-problem-with-amp-now-lets-you-view-and-share-publishers-own-links/?ncid=rss

谷歌今天推出了对其在谷歌搜索中的 AMP 集成的改变，允许您查看、复制和共享出版商自己的相关网页链接，而不是 AMP URL。做出这一改变的决定之前，一些出版商强烈反对，他们认为谷歌窃取了他们的流量，将他们自己的网址更改为名字中有“谷歌”的网址，所有这些都是以移动优化的名义。

Google AMP(Accelerated Mobile Pages)正如你可能还记得的，是 Google 的项目，通过提供出版商网站的优化版本来加快移动设备上的网页浏览速度，而不会被第三方脚本和其他会减慢或停止页面渲染的扩展机制所拖累。

然而，关于 AMP 如何工作一直存在一些误解。去年 10 月，一篇广为流传的博文称，谷歌通过其 AMP 页面从出版商那里窃取流量。但那不是真的。谷歌确实在搜索结果中显示了 AMP URL，它从谷歌的缓存中提供页面内容，但是流量仍然是发布者的，并且内容是从发布者的网站提供的。

尽管如此，人们还是对这个网址系统感到担忧。

AMP URLs 将把访问者重定向到出版商的网站，作为临时的重定向，而不是永久的重定向，去年秋天搜索引擎土地解释。没有对直接链接的支持，一些人担心如果谷歌做出任何改变，书签网址在未来可能会停止工作。此外，用户对如何共享链接普遍感到困惑。人们会在地址字段中看到 AMP 的 URL，他们不想复制并粘贴它，因为它不是网页的“真实”链接。

正如谷歌今天在宣布变革的博客文章中所说:

“URL 和来源在某种程度上代表了对内容的信任和所有权。当您阅读《纽约时报》的一篇文章时，快速浏览该网址会让您相信您所阅读的内容代表了《纽约时报》的声音。归属、品牌和所有权都很清楚。”

谷歌放大器搞乱了这个系统。

不是一个网址(出版商的)，而是三个。这包括出版商的 URL、AMP 缓存 URL(通过 AMP 缓存提供的文档，但用户通常看不到)和 Google AMP Viewer URL(您在搜索结果中看到的 AMP URL)。

谷歌继续详细解释为什么做出这些设计选择，举了一个例子，即出版商的易用性。该公司还解释了它如何能够在搜索结果页面上的隐藏 iframes 中预先呈现 AMP 文档，以加快加载用户最终点击的链接的时间。但这也意味着用户看到的网址必须来自 google.com。

为了让用户了解网页内容实际上来自哪里——也就是说，不是谷歌——它会在页面顶部弹出一个标题栏，显示页面的真实来源(如“nytimes.com”)。

但当用户想要复制或分享网页时，他们通常会查看地址栏。因为这里显示的是 google.com 的地址，所以这不是人们想要收藏或与他人分享的链接。

现在 Google 在标题栏上添加了一个按钮来提供这些信息。这将显示出版商自己的网页永久链接。谷歌表示，该功能允许用户通过长时间点击这个链接来使用浏览器的原生共享功能。

该功能已经在 iOS 谷歌应用中提供，并将在未来几周内推出安卓版本。

它还在开发一个网络共享 API ，该 API 将允许 AMP 观众将原始 URL 而不是 AMP 观众的 URL 拉入平台上的原生共享流。

鉴于出版商的强烈反对和用户的困惑，谷歌已经表示它正在研究这样一个解决方案。现在，更多的出版商可能会选择加入 AMP，因为他们自己的链接将不再完全隐藏。
