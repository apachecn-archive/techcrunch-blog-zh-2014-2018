Twitter 将在不久的将来推出书签工具 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2017/10/09/twitter-will-launch-a-bookmarking-tool-in-the-near-future/

Twitter 证实，它计划推出书签功能，以保存推文供日后阅读。这一新增功能将帮助用户保留一个单独的项目列表，他们希望返回参考，而不是使用心形按钮(又名“收藏夹”)，这可以表明更多的“喜欢”——类似于脸书上的“竖起大拇指”按钮。

这项功能的即将推出首先在 Twitter 上公布，自然，当产品负责人 Keith Coleman 宣布一种新的保存推文的方法正在工作中，这是公司 HackWeek 项目的结果，该项目被称为#SaveForLater。

其他 Twitter 员工，包括新聘用的高级产品总监、斯里拉姆·克里希南和项目经理杰萨尔·沙阿，也注意到了 Twitter 在这一领域的计划。

像科尔曼一样，沙阿说该功能是一个受欢迎的用户请求——他补充说，许多人“特别是在日本”要求这种能力。

正如 Shah 在她的帖子中所说，人们私下保存推文以供日后参考的方法有很多，包括用心将它们标记为书签，将它们标记为自己，甚至转发。(我还要补充一点，我们会在新标签页中打开它们，将它们保存到我们的 Notes 应用程序中，通过电子邮件发送给我们自己，在 Instapaper 上发布它们的链接，或者创建私人 Twitter 时刻或 Storify 收藏，等等。)

这些方法都不理想，因为它们没有简单地点击按钮来保存推文那么快，除了“心形”图标。然而，心可能会被误解，因为它暗示你对你保存的推文感觉积极，而事实可能恰恰相反。此外，对于那些经常用心回复不需要回复的推文的人来说，那些你想再次回顾的保存的推文可能很容易在你的收藏夹中丢失。

Twitter 并不是第一个意识到其不断更新的 feed 需要一个保存按钮的公司。脸书也一样，几年前推出了自己的书签工具。

在 HackWeek 期间建立的原型显示，Twitter 的书签功能出现在推文的“更多”菜单下(三个圆点)，然后你可以在那里找到一个新的“添加到书签”选项。但是这种设计在向公众发布时可能会改变。

根据 Shah 的说法，Twitter 希望在社区的帮助和反馈下建立新的书签工具。这很有趣，考虑到 Twitter 过去是如何利用用户输入来开发从社区借来的功能的。例如，Twitter 的@ reference 和 retweet 功能源于 Twitter 将人们已经在其网络上采取的行动转化为有用的产品功能。

但是 Twitter 最近的一些产品发布——比如有争议的@回复格式或者关于什么算作字符的决定——更多的是作为对社交网络 140 个字符限制问题的过度设计的回应，而不是对用户行为的深入研究。(Twitter 已经表示将测试将字符数扩展到 280，这比混乱的字符数规则更有意义。)

目前还没有关于新书签工具多快会推出或者谁能测试它的细节。但 Twitter 发言人证实，该功能不仅仅是一项实验——它将很快公开推出。他们还表示，了解更多信息的最佳方式是关注 @jesarshah 的账户，因为这里会出现关于#SaveForLater 的信息。我们被告知，产品设计师 @tinastsh 也会发微博。
