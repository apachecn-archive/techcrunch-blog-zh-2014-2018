# Fae 是一个新的基于 Rails TechCrunch 的开源内容管理系统

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/04/04/this-is-fine-fae-is-a-new-open-source-content-management-system/>

总部位于旧金山和波特兰的品牌代理公司 FINE T1 正在 T2 开放其内部开发的内容管理系统 T5，为诸如 Anchor Brewing T7、Kimpton Residences at sea fire T9、PRF 栽培品种 T11 等品牌建立网站。

众所周知，市场上已经有过多的其他 CMS 系统，很难在这个拥挤的领域脱颖而出。然而，FINE 认为，这个新的 CMS 引擎被称为 [Fae](https://web.archive.org/web/20221006110518/https://www.faecms.com/) ，它脱颖而出有几个原因——主要是因为它专注于轻量级和高度可定制性。因为 FINE 一直用它来建立自己的客户网站，所以在这一点上它应该也是久经沙场了。

值得注意的是，Fae(代表“精细管理引擎”)直接面向开发人员。没有类似 WordPress 的[设置程序](https://web.archive.org/web/20221006110518/https://github.com/wearefine/fae/blob/master/docs/installation/index.md)可以在几分钟内给你一个完整的前端和后端工作的 CMS。

正如运营总监 Lori Dunkin 告诉我的那样，几年前，公司通常为每个客户编写一个定制的基于 PHP 的 CMS。这也意味着它的很多资源都投入到了这些建设中。通过 Fae，它开发了一个 CMS 系统，可以很容易地为不同的客户定制，而不必为每个客户重新发明轮子——现在它是开源的，任何人都可以尝试一下。

[![](img/44bef49ab1e3fcf106e6e25815b3b585.png)](https://web.archive.org/web/20221006110518/https://beta.techcrunch.com/wp-content/uploads/2017/04/cms-ui.png)

通常，公司开源那些他们认为对他们来说不是主要差异化的内部产品。有人可能会说像 Fae 这样的 CMS 系统对 FINE 的工作非常重要。然而，Dunkin 指出，FINE 认为其与众不同之处在于它如何为客户配置 Fae 环境，以及如何将客户的品牌提取到数据模型中。“这些都与表格的外观或你如何登录无关，”她指出，并补充说，因为新系统是开源的，客户不会被锁定，如果他们想，可以很容易地转移到不同的机构。“这是一个没有争议的问题，符合所有人的最佳利益，”她说。

因为它是基于 Rails 的，开发者可以在 Ruby [gems](https://web.archive.org/web/20221006110518/https://en.wikipedia.org/wiki/RubyGems) 的帮助下，在 Fae 现有特性的基础上轻松添加额外的功能。得益于此，该系统保持了相当的轻量级(不像 Drupal ),同时还提供了必要的灵活性，可用于所有类型的站点和应用程序。

Fae 的核心功能包括身份验证、对多种用户类型的支持、针对字段类型、页面结构、列表和工作流的常用用户界面组件，以及多语言支持、图像上传和处理、变更跟踪、过滤和全局搜索工具。

从长远来看，该公司希望围绕 [Fae](https://web.archive.org/web/20221006110518/https://github.com/wearefine/fae) 建立一个社区，该社区现在可以在 GitHub 上获得。FINE 将维护代码并保持开源库的更新。事实上，值得注意的是，Fae 只有一个版本，而且该公司没有为自己保留任何专有部分。毕竟，这将违背其宣称的原则，即让客户轻松地获取他们的网站并转移到其他地方。
