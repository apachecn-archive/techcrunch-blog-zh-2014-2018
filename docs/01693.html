Stamplay 是后端开发的 if TTT TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2014/10/06/stamplay-is-ifttt-for-back-end-development/

如果你是一个前端开发人员，设置注册表单和集成流行的 API 会非常痛苦。 Stamplay 想让后端开发像玩乐高积木一样简单。最好的类比是将其视为开发人员的 IFTTT。

使用 Stamplay 编码就是浏览其非常直观的界面来选择和配置正确的模块。有一个模块来添加新用户，upvote 项目，发布内容，评论等。使用 Stamplay 的机制，您可以在几个小时内开发简单的 web 服务。例如，有教程可以创建类似于黑客新闻的或类似于 Stackoverflow 的网站，而无需直接管理数据库、用户注册、脸书连接集成和所有其他部分。

“Stamplay 通过将软件转化为视觉体验，正在改变软件的构建方式，”联合创始人兼首席技术官朱利亚诺·亚科贝利告诉我。“在这种情况下，您只需要关注面向用户的功能，只使用一个抽象级别。”

即使对于那些想继续手工做事情的开发人员来说，Stamplay 也可以在 API 集成方面提供帮助。例如，您可以创建一个简单的规则，说明“当用户注册脸书时，给他或她发送一封欢迎电子邮件，并将他或她添加到我的 Mailchimp 列表中。”你不必去钻研脸书和 Mailchimp API 文档。

Stamplay 集成了 Mailchimp 列表、Sendgrid、YouTube、Intercom、Pusher.com、谷歌电子表格、脸书广告等等。它可以帮助你一键注册脸书、Twitter、Dropbox、GitHub、Google 等。

平台才刚刚起步。到目前为止，大约有 10，000 名开发者试用了这项服务。这家初创公司需要添加更多模块，让这项服务对重度用户更有用。但是这已经是一个很好的尝试新事物的方式，建立一个 MVP，看看一个想法是否有一些吸引力——特别是如果你是一个前端开发人员。

“目前没有真正的直接竞争对手，但市场非常拥挤。一方面，我们有像 Parse、Kinvey 和 FeedHenry 这样的 BAAS，它们通过提供服务器端托管和数据存储来瞄准移动开发者。“然后，我们有更多像 Firebase 或 Hull.io 这样的垂直服务来解决具体问题。最后，在另一边，有像 Squarespace、Weebly 或 Wix.com 这样的网站建设者，但他们是为最终用户服务的，只是让你建立大部分是静态的网站。”

未来，开发者将能够像在 IFTTT 上一样分享现成的食谱。这家初创公司还将推出一个由社区构建的全新模块市场。

Stamplay 的主要挑战将是使它更好地用于大规模开发，而不仅仅是原型开发。至少，亚科贝利对公司的愿景深信不疑。“这听起来很疯狂，但回想一下早期人们通过完全控制主板来编写计算机程序的时候，”他说。“谁会想到有人可以通过点击几下鼠标来构建软件，而不用担心芯片的每个组件？”
