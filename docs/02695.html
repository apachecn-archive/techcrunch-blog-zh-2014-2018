随着谷歌寻求扩大钱包的使用，WePay 整合了即时购买 API TechCrunch

原文：https://web.archive.org/web/http://techcrunch.com/2015/01/26/google-looks-to-expand-wallet-use-wepay-integrates-instant-buy-api/?ncid=rss

据报道，谷歌正着眼于收购 Softcard，以将谷歌钱包服务扩展到更多的零售商销售点；但是它也想通过它的即时购买 API ，在更广阔的基于移动的交易世界中提高自己的地位，与 PayPal 这样的公司竞争。

最初于 2013 年为 Android 推出，如今即时购买 API 向前迈进了一步:谷歌正寻求通过与其他支付公司合作来整合该 API，从而扩大其使用范围。首先，支付提供商 WePay 已经将 API 集成到其支付后端。

这意味着 WePay 客户现在有一个谷歌钱包选项，让他们自己的客户支付东西，或支付发票:最终客户现在可以选择通过谷歌钱包支付，并在“两次点击”中完成交易，类似于今天应用程序、市场和服务门户网站允许的即时贝宝支付。

WePay 表示，它是第一家集成 API 的公司，这是谷歌最初接触 invoice asap(WePay 的客户)以将 API 集成到其服务中的结果。Invoice ASAP 拥有 200，000 名小企业客户，其想法是让他们更容易将谷歌钱包支付与其他支付方式相结合，作为在移动设备上支付发票的一种选择。

从今天起，WePay 也将向其他客户提供谷歌钱包集成。其中包括 Care.com、恒常联系、FreshBooks、GoFundMe 和 Meetup 等。

WePay 的联合创始人兼产品副总裁理查德·阿伯曼解释说:“这是谷歌钱包新分销模式的一部分，表明了他们对这一模式的承诺。”。“我们很高兴成为第一个使用它的人。”

谷歌试图解决的差距是，虽然许多人可能会在移动设备上查看电子邮件和阅读发票，但事实证明，通过移动设备支付更令人痛苦。

WePay 估计，目前只有大约 20%的交易发生在移动设备上，尽管事实上大约 70%的 WePay 客户最初是在他们的移动设备上开发票的。

“这是因为这种经历很糟糕，”阿伯曼告诉我。

“我们希望让人们尽可能容易地支付和获得报酬，”InvoiceASAP 的创始人兼首席执行官保罗·霍珀(Paul Hoeper)在一份声明中说。“但我们在手机上能做的事情有限。在移动设备上输入你的信用卡信息本来就很困难，这使得人们完成交易的可能性降低。”

谷歌为谷歌钱包寻找更普遍的体验可能还有另一个原因。这样做将有助于它与苹果正在推出的支付体验保持一致。

就 iPhone 制造商而言，它也希望将其新服务 Apple Pay 定位为类似 PayPal 的竞争对手，为用户提供一种支付方式，不仅可以在实体销售点支付，还可以在网上和移动设备上支付。WePay 的 Aberman 表示，他的公司现在正在整合 Apple Pay，可能会在未来几个月内开启这项服务。

目前还不清楚 Google Wallet Instant API 被广泛采用的程度，因此可能决定与 WePay 等第三方合作来发展它。谷歌于 2014 年 7 月在将谷歌钱包即时 API 支持扩展至 iOS。
