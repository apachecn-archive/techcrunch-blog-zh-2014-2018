Mesosphere和微软将Mesos引入Windows Server TechCrunch

原文：https://web.archive.org/web/http://techcrunch.com/2015/08/20/mesosphere-partners-with-microsoft-to-bring-mesos-to-windows-server/?ncid=rss

今天，旨在基于Apache Mesos项目构建以容器为中心的“数据中心操作系统”的公司Mesosphere 将在meso son上首次公开演示在Windows Server 2016预览版上运行的Mesos。

微软昨天才发布了支持Docker容器的Windows Server的第一个技术预览版，所以开发人员在生产中使用这一功能可能还需要一段时间。

正如Mesosphere的联合创始人Ben Hindman告诉我的，团队与微软密切合作，通过开源 Apache Mesos 项目来实现这一点。他还指出，许多Mesosphere的企业客户一直在要求Windows服务器支持，因为他们准备从试验容器转向投入生产。

毕竟，许多企业的现实情况是，他们在数据中心同时运行Linux工作负载和Windows服务器工作负载。因此，借助Mesos(通过扩展，还有Mesosphere)，运营商将能够使用同一个容器管理工具，运行需要在Linux机器上运行的任务，以及在Windows服务器上运行的Windows工作负载。

Windows Server上的Mesos插入了微软现在在最新的Windows Server预览版中提供的Docker APIs。通过这样做，开发人员将能够使用Windows Server Container，以及以后微软为其平台构建的更安全的Hyper-V Container。

微软Azure CTO Mark Russinovich告诉我，微软参与了这个项目，因为该公司希望为其客户提供另一种Windows容器编排技术——这也是其客户一直以来的要求。该公司已经展示了在其Azure云上运行的Mesos，但那是基于Linux的演示。

Mesosphere团队告诉我，使这一切成为可能的代码将在一两周内出现在它的GitHub库上——尽管有些部分已经在那里了。

值得注意的是，最近有的一些报道称微软实际上正在考虑收购Mesosphere。这两家公司肯定非常友好，微软显然正在购买容器概念，无论是在其云和服务器平台上。

在过去的几年里，微软也表明了它希望与开源社区合作。不过，谷歌也可以这么说，如果Mesosphere参与进来，它肯定不会在没有竞价战的情况下出售。
