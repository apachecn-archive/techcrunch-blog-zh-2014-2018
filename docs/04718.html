Mesosphere 和微软将 Mesos 引入 Windows Server TechCrunch

原文：https://web.archive.org/web/http://techcrunch.com/2015/08/20/mesosphere-partners-with-microsoft-to-bring-mesos-to-windows-server/?ncid=rss

今天，旨在基于 Apache Mesos 项目构建以容器为中心的“数据中心操作系统”的公司 Mesosphere 将在meso son上首次公开演示在 Windows Server 2016 预览版上运行的 Mesos。

微软昨天才发布了支持 Docker 容器的 Windows Server 的第一个技术预览版，所以开发人员在生产中使用这一功能可能还需要一段时间。

正如 Mesosphere 的联合创始人 Ben Hindman 告诉我的，团队与微软密切合作，通过开源 Apache Mesos 项目来实现这一点。他还指出，许多 Mesosphere 的企业客户一直在要求 Windows 服务器支持，因为他们准备从试验容器转向投入生产。

毕竟，许多企业的现实情况是，他们在数据中心同时运行 Linux 工作负载和 Windows 服务器工作负载。因此，借助 Mesos(通过扩展，还有 Mesosphere)，运营商将能够使用同一个容器管理工具，运行需要在 Linux 机器上运行的任务，以及在 Windows 服务器上运行的 Windows 工作负载。

Windows Server 上的 Mesos 插入了微软现在在最新的 Windows Server 预览版中提供的 Docker APIs。通过这样做，开发人员将能够使用 Windows Server Container，以及以后微软为其平台构建的更安全的 Hyper-V Container。

微软 Azure CTO Mark Russinovich 告诉我，微软参与了这个项目，因为该公司希望为其客户提供另一种 Windows 容器编排技术——这也是其客户一直以来的要求。该公司已经展示了在其 Azure 云上运行的 Mesos，但那是基于 Linux 的演示。

Mesosphere 团队告诉我，使这一切成为可能的代码将在一两周内出现在它的 GitHub 库上——尽管有些部分已经在那里了。

值得注意的是，最近有的一些报道称微软实际上正在考虑收购 Mesosphere。这两家公司肯定非常友好，微软显然正在购买容器概念，无论是在其云和服务器平台上。

在过去的几年里，微软也表明了它希望与开源社区合作。不过，谷歌也可以这么说，如果 Mesosphere 参与进来，它肯定不会在没有竞价战的情况下出售。
