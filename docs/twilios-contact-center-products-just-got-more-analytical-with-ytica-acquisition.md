# 收购 Ytica 后，Twilio 的联络中心产品变得更具分析性

Twilio ，一家以向开发者提供通信 API 而闻名的公司，拥有一款名为 Twilio Flex 的[产品，用于在 Twilio 的 API 之上构建复杂的客户服务应用。今天，它宣布将收购](https://web.archive.org/web/20230110114223/https://techcrunch.com/2018/03/12/twilio-launches-flex-a-fully-programmable-contact-center/) [Ytica](https://web.archive.org/web/20230110114223/https://www.ytica.com/) (发音为 Why-tica)，在客户服务解决方案的基础上提供运营和分析层。

两家公司不愿讨论收购价格，但 Twilio 表示，预计此次收购不会对其“业绩、运营或财务状况”产生实质性影响。换句话说，大概没花多少钱。

总部位于布拉格的 Ytica 实际上已经与 Twilio 合作了一段时间，所以以这种方式走到一起真的很有意义，尤其是 Twilio 一直在开发 Flex。

Twilio Flex 总经理 Al Cook 解释说，Twilio Flex 是一个用于联系中心的应用程序平台，它提供了一整套应用程序，允许用户通过多种渠道提供客户支持。“Flex 像 SaaS 一样部署，但因为它是建立在 API 之上的，你可以进入并改变 Flex 的工作方式，”他说。这是非常有吸引力的，尤其是对于寻求灵活的、基于云的解决方案，而没有内部传统产品包袱的大型企业而言。

然而，该产品所缺乏的是一种从应用程序内部管理客户服务代表，并通过分析和仪表板了解团队表现如何的本机方式。随着团队变得越来越大，衡量团队效率的能力变得更加重要，Cook 指出，一些 Flex 用户正在管理拥有 10，000 到 20，000 名员工的庞大团队。

Ytica 提供了一种衡量客户服务人员绩效的方法，允许管理层在必要时进行监控、干预和指导。“作为一个团队团结在一起非常有意义。他们在联络中心有丰富的经验，也有类似的理念，在云中构建可定制和可编程的东西，”库克说。

虽然 Ytica 与 Twilio 以外的其他供应商合作，但首席执行官 Simon Vostr 表示，他们将继续支持这些客户，即使他们加入了 Twilio 大家庭。“我们可以运行 Flex，并且可以继续单独运行它。我们有客户在其他 SaaS 平台上运行，我们将继续支持他们，”他说。

该公司将留在布拉格，成为 Twilio 的卫星办公室。预计所有 14 名员工都将加入 Twilio 团队，库克表示，扩大布拉格团队的计划已经在进行中。
