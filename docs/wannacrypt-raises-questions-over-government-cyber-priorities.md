# WannaCrypt 对政府网络优先事项提出质疑

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/05/16/wannacrypt-raises-questions-over-government-cyber-priorities/>

上周五，WannaCrypt 勒索软件攻击迅速蔓延到多个国家和系统，其后果继续在世界各地引起恐慌。

在英国，数十个国家卫生服务信托机构被感染，导致常规预约被取消，并引发了对关键医疗保健系统健壮性的巨大担忧，这些系统仍在一个有 15 年历史的软件平台(Windows XP)上运行，该平台容易受到攻击。

批评人士认为，政府资金的缺乏让 NHS 向利用过时软件的恶意软件攻击敞开了大门。

几天前，英国内政大臣 Amber Rudd 试图反驳这一批评，他认为对 NHS 系统的恶意攻击的成功与“我们的准备”无关。

“如果你看看谁受到了这种病毒的影响，就会发现不同行业和不同国际政府之间存在巨大差异。这是一种攻击 window 平台的病毒，事实是 NHS 已经成为受害者。我不认为这与我们的准备工作有关。她在周末的一份声明中说:“我们总是可以做更多的事情来确保我们免受病毒的侵害，但我认为 NHS 已经做好了充分的准备，以确保他们为这种攻击做好准备。”。

回到 [2015](https://web.archive.org/web/20230306043032/https://techcrunch.com/2015/11/18/uk-gov-to-invest-in-security-startups/) ，英国政府宣布了一项为期五年的[国家网络安全战略，当时它表示将在 2020 年前投入 19 亿英镑进行转型投资，包括保护关键基础设施和电子商务活动免受黑客攻击。](https://web.archive.org/web/20230306043032/https://techcrunch.com/2016/11/01/uk-commits-1-9b-to-national-cyber-security-strategy-working-with-microsoft-and-more/)

目前还不清楚到底有多少钱被用于升级关键系统。但是，很明显，要让公共部门摆脱 Windows XP 等传统平台，还有很多工作要做。

事实上，与多个 NHS 信托机构一样，TechCrunch 了解到英国内政部自己的一些系统仍在该平台上运行。据知情人士透露，政府部门使用 Windows XP 作为公务员和承包商使用的 Poise 最终用户计算环境。

当我们联系内政部询问哪些系统运行的是 Windows XP 时，它没有回答这个问题。相反，他们的回应是，内政部的 IT 系统处于“不断审查”之中，该部门一直在遵循国家网络安全中心的建议，以保护其系统免受 WannaCrypt 病毒的攻击。

这位女发言人补充说，国务院已经采取了额外的安全和应急措施，但没有具体说明这些措施是什么。

在其网站上的一份声明中，NCSC 敦促各组织遵循其建议，以减少被病毒感染的风险。“我们已经知道有人试图攻击国民医疗服务体系以外的组织。因此，任何认为自己可能受到影响的组织都必须遵循并执行这一指导方针。

NCSC 对公司保护自己免受勒索软件攻击的建议包括:

1.  让您组织的安全软件补丁保持最新
2.  使用适当的防病毒软件服务
3.  备份对您重要的数据

我们向 NCSC 提出了几个关于英国网络安全支出优先事项的问题——在撰写本文时，它尚未做出回应，但如果他们做出回应，我们将更新这篇文章。

我们的消息来源指出，许多组织不从较旧的操作系统升级，因为他们落后于许多升级，并且越往后的平台版本进行操作系统升级变得越来越困难和昂贵。这强调了资金对这个安全故事的重要性——无论陆克文声称英国在“准备”方面没有问题。

关于英国公共部门对脆弱的遗留系统的持续依赖，需要提出一些问题——尤其是考虑到一系列失败的中央政府 IT 项目，包括卫生和边境控制项目。

鉴于其对网络安全的公开优先考虑——包括增加专门用于支持关键系统的资金——前景肯定不妙。

在内政部的情况下，不清楚如果其一些内部系统感染 WannaCrypt 可能会有什么样的实际影响。例如，对庇护申请的中断不会像对医疗保健的中断一样具有可见性。但该部门还监管与关键通信基础设施相关的政策和项目，如紧急服务，因此这里也有潜在的公共安全问题。

内政大臣自己也承认，从 WannaCrypt 袭击事件中可以“吸取教训”。该部门似乎很可能会加快关于 Windows XP stat 升级的对话，尽管它拒绝公开谈论在这方面正在做的任何事情。

这个故事中的另一个问号是国内情报机构的角色——鉴于 WannaCrypt 正在利用的漏洞显然是从美国国家安全局窃取的。NSA 与英国国内情报机构 GCHQ 有着长期的密切合作。这至少使得英国自己的间谍机构有可能知道或参与维护造成如此大破坏的潜在缺陷。即使 GCHQ 悄悄地帮助保护内政部的内部系统免受这一特定缺陷的影响，它显然未能向 NHS 提供这样的服务。

微软因没有继续为其旧平台提供安全补丁而受到批评，这是理所当然的。但该公司昨天也发表了一份措辞强硬的声明，将 WannaCrypt 的传播归咎于政府机构，这些机构习惯于储存甚至创造安全漏洞——将其比作军方允许储存的常规武器被盗。

“这次攻击提供了另一个例子，说明为什么政府囤积漏洞是一个如此严重的问题，”公司总裁兼首席法律官布拉德·史密斯昨天说。"一次又一次，政府手中的漏洞被泄露到公共领域，造成了广泛的损害."

“世界各国政府应该把这次袭击当作一记警钟，”他补充道。“他们需要采取不同的方式，在网络空间遵守适用于现实世界武器的相同规则。我们需要政府考虑囤积这些漏洞和利用这些漏洞对平民造成的伤害。”

微软正在倡导一个“[数字日内瓦公约](https://web.archive.org/web/20230306043032/https://blogs.microsoft.com/on-the-issues/2017/02/14/need-digital-geneva-convention/#sm.0001gnysbhjsod01z7q11hvz0xg2d)”来管理这些问题，包括要求政府向供应商报告漏洞，而不是囤积、出售或利用它们。

此时此地，情报机构积极而秘密地利用安全系统中的弱点，我们离安全必须成为每个人——包括政府每个部门——的首要任务的共识还很遥远。

但是，也许我们更接近理解所涉及的风险程度了。
