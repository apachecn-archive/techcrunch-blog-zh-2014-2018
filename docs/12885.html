红帽收购 Codenvy 作为其增长的容器战略的一部分

以企业 Linux 公司而闻名的红帽公司(Red Hat)近年来一直明确表示将云计算和容器化视为其未来的重要组成部分。今天，它宣布有意收购旧金山初创公司 Codenvy ，以继续发展这一战略，并为开发者提供基于云的集成开发环境。

该公司没有透露收购价格。

Codenvy 是建立在开源项目之上的公司， Eclipse Che ，这符合 Red Hat 在开源项目之上构建商业工具的整体战略。它为个人开发者、团队或企业提供了一个基于云的集成开发环境(IDEs)。ide 本质上是编码、构建和测试应用程序的工作空间。

在一篇宣布收购的公司博客文章中，Red Hat 开发项目高级主管 Harry Mower 明确表示，他的公司认为 Codenvy 不仅仅是一个云 IDE。“Codenvy 技术的核心是一个工作空间管理系统，它允许开发人员立即启动并运行，而无需设置本地开发环境，”Mower 写道。

容器是这一愿景不可或缺的一部分，它能够将应用程序作为离散的代码片段或容器来启动，而不是启动一个单一的整体应用程序。开发人员正在使用 Docker 和 Kubernetes 等工具来构建这些容器化的环境。

Codenvy 首席执行官泰勒·朱厄尔认为，这次收购是扩展公司愿景的一种方式，通过与一个更大的组织整合资源，建立一个比他的公司单独完成的更广阔的容器愿景。“容器的快速采用使编排标准成为行业的下一步。我们认为 Kubernetes 和 Red Hat OpenShift 在这一领域处于领先地位。因此，当红帽分享他们的容器愿景时，我们加入他们的决定变得“不言而喻”，朱厄尔在博客文章中写道。

红帽实际上已经发出电报，这样的举动可能会持续一段时间。首先， Red Hat、Codenvy 和微软去年联手，用 Red Hat 的话说，“提供一种通用的方法来集成跨代码编辑器和 ide 的编程语言。”Red Hat 去年加入了 Eclipse Che 社区，并开始贡献代码。几周前，该公司发布了 Openshift.io ，这是一套基于云计算的开发者工具，部分基于——等等——Eclipse Che。

TechCrunch 的 Frederic Lardinois 是这样描述 Openshift.io 的:“不过，Red Hat 在这里所做的是将一些现有的开源项目捆绑在一起，比如 fabric8、Jenkins、Eclipse Che，当然还有 Openshift，使其成为一项免费服务，为开发人员提供类似的体验，但重点是基于容器的应用程序。”

考虑到之前的这些举措，下一个合乎逻辑的步骤似乎是收购建立在 Eclipse Che 之上的公司。交易完成后，Codenvy 的员工将加入 Openshift.io 团队。
