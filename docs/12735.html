Google Firebase 获得结构集成、更好的分析和 alpha 程序 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2017/05/17/google-firebase-gets-fabric-integration-better-analytics-and-an-alpha-program/?utm_campaign=Luis%2BAbreu%2BNewsletter&utm_medium=web&utm_source=Luis_Abreu_Newsletter_5

谷歌今天宣布了对其 Firebase 应用平台的一系列更新，这些更新触及了该服务的许多核心功能(或将在不久的将来实现)。谷歌今天表示，现在已经有超过 100 万名开发人员用 Firebase 创建了项目(当然，并非所有人都很活跃)，而且这个数字还在快速增长。

几个月前，谷歌从 Twitter 收购了Fabric 开发者平台(及其团队)。通过这次更新，该公司开始将 Fabric 的一些核心功能集成到 Firebase 中。 Crashlytics ，这是 Fabric 的旗舰服务，很快将成为 Firebase 的主要碰撞报告产品。不过，还不清楚具体什么时候会发生。

不过，开发者可以马上使用的一个 Fabric 特性是电话认证。这是基于现有的 Fabric Digits 身份验证服务，但 Google 正在淘汰这个名称，以支持 Firebase 身份验证。有了这个，Firebase 现在为开发者提供了一个全面的认证服务。

像今天谷歌的许多其他更新一样，更新后的 Firebase 也提供了新的分析和监控功能。谷歌开发产品团队的产品管理总监 Ben Galbraith 告诉我，该公司正在将 Firebase Analytics 更名为“Firebase 的谷歌分析”,顾名思义，该团队实际上与谷歌分析团队密切合作，以提供 Firebase 和谷歌分析的报告。

同样，Firebase 现在与谷歌的应用内广告服务 AdMob 进行了更深入的整合，现在与 Firebase 的分析服务共享数据，因此开发者更容易看到他们用户的终身价值(基于应用内购买和 AdMob 收入)。

但是还有更多；Firebase 现在还允许开发人员定义多达 50 个自定义事件参数，并在他们的分析报告中看到它们。

虽然 Firebase 的重点往往是移动应用程序，但该服务也提供网络托管工具。到目前为止，这主要是关于提供静态页面，但随着今天的更新，它获得了与 Firebase 云功能的集成，这是谷歌为 Firebase 开发者提供的“无服务器”计算服务。有了这个，Firebase 用户现在可以很容易地在新用户注册时发送通知，或者在有人上传图像时创建不同分辨率的缩略图。

通过此次更新，Firebase 还推出了一个新的 alpha 程序，该程序将让任何想要使用它的开发人员尽早获得 Firebase 的新功能。正如 Galbraith 告诉我的，这里的想法是让团队正在内部测试的许多早期访问程序可用于更广泛的测试。虽然任何人都可以成为这个项目的一部分，但是这个团队可能会将它的一些测试限制在一部分用户中。

今天发布的其他 Firebase 包括在 Firebase SDKs 中更好地支持苹果的 Swift 语言，以及新的性能监控功能。该团队还开源了它的 SDK。该团队在今天的声明中写道:“我们相信开源软件，不仅因为透明是一个重要的目标，还因为我们知道，当我们所有人合作时，最大的创新就会发生。”。
