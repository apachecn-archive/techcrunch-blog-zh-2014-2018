作为一名开发者，宣告网络死亡的文章很容易被忽视。人们预测(或预示着)下一件大事的到来，并宣布现状死亡，这并不罕见或出乎意料。然而，尽管移动设备在过去几年里呈爆炸式增长，但在可预见的未来，网络仍将是主导的软件平台。

首先，我说的web是指在浏览器或类似浏览器的轻量级shell中访问的HTML内容。更广泛的定义可能包括几乎每个本地移动应用程序使用的HTTP APIs。根据这个定义，网络根本没有竞争。

为什么是网络？

早在web出现之前，本地应用就已经存在了。那么，为什么网络会成为如此强大的软件分发平台呢？即使在Internet Explorer 6时代，网络也因几个关键优势而获得了发展:

  • 没有安装。要访问web应用程序，只需在web浏览器中访问它的URL。每个操作系统的每个用户都可以在希望的几秒钟内访问web应用程序。
  • 总是最新的。没有安装就没有更新过程。每次你访问一个网络应用程序，你都会免费获得最新版本(包含最新数据)。
  • 跨平台。因为网络是建立在开放标准之上的，所以任何人都可以构建网络浏览器。网络承诺真正的一次编写，随处运行的软件，并且(除了一些令人讨厌的浏览器怪癖)实现了这个承诺。
  • 网络效应。Web应用程序无需许可即可相互链接。每个链接都像其他链接一样工作。“病毒式传播”的概念之所以存在，只是因为网址是普遍共享的。
  • 够快了。在桌面上，硬件的进步产生了越来越快的设备。这导致了一个转折点:对于大多数软件来说，大多数时候，web应用程序已经足够快了。它们在功能上等同于它们的本地对应物。

这些优势是软件历史上最低的进入门槛，也是一个不容错过的好机会。作为一种传递文档的方式，网络很快就变成了一个应用平台。

随着时间的推移，网络在软件方面变得越来越好。跨平台的JavaScript库让开发人员使页面更具交互性。AJAX允许浏览器在不重新加载页面的情况下发送和接收新数据。CSS的发展使得类似应用程序的界面构造变得更加简单。Web技术总是在变化，随着时间的推移不断发展，以满足使用它的人和开发人员的需求。

移动，移动，移动

随着应用商店的迅速崛起，人们很容易忘记2007年苹果最初的“应用”计划完全是关于网络的。虽然许多因素导致了苹果转向原生第三方应用，但最常提到的原因是与性能相关的用户体验。我们稍后会谈到这一点。

App Store为原生应用分发带来了巨大的改进。对于终端用户来说，安装和更新应用突然变得几乎和访问网站一样简单。App Store还提供了单一的集中支付权限(永远不要低估存储的信用卡数据的力量！).这些因素导致许多人吹捧移动应用商店是网络的一个可行的竞争者或继承者。

随着全球智能手机渗透率超过个人电脑，网络终于遇到对手了吗？是时候将火炬传递给年轻一代的本地应用了吗？

没有。

觊觎王位的人

这不是开放网络第一次面对一个受欢迎的对手。90年代中期，美国在线拥有一个巨大的围墙花园。当时美国最大的互联网服务提供商，他们将整个服务面向生活在其中的用户。如今，AOL是一家内容公司，在开放网络上运营内容(包括TechCrunch)。网络赢了。

Flash曾经是开发者和用户的最爱。它提供了HTML不喜欢的东西:视频、动画、声音和交互性。超过97%的浏览器都安装了Flash插件。今天，Flash出现在大约14%的网站上，比50%以上的高峰有所下降。网络赢了。

总会有新技术挑战网络软件的统治地位。起初，这些网站中的许多在某个方面比网络有优势。但这就是网络的超能力:它一直在进化。正如Jason Kottke 在2007年所写的那样，“在竞争市场中，开放和混乱的王牌是封闭的，并且从长远来看是受控制的。”

加速回报

2010年7月22日是网络平台历史上最重要的日子之一。今天，谷歌宣布了为期六周的Chrome主版本发布时间表。Chrome，当时是5.0版本，现在是34.0版本。2011年，Mozilla紧随其后，宣布了针对Firefox的类似计划。快节奏的发布时间表加上“发布渠道”的概念(有意愿的人可以提前获得新功能)永远改变了web开发的前景。

考虑到Internet Explorer 7是在Internet Explorer 6之后五年才问世的。Chrome将新的正常发布速度提高了40倍。被称为“HTML5”的一套浏览器技术在同一时间内从零变得司空见惯，这并非巧合。浏览器厂商必须改进他们的平台，否则就会被越来越挑剔的用户甩在后面。

今天，阻碍网络技术进步的最大障碍不是微软，而是苹果。在iOS上，苹果迫使第三方浏览器使用更慢的JavaScript引擎，削弱了它们的能力。他们还禁止开发者在平台上发布替代的浏览器引擎。虽然令人沮丧，但这是现任者扼杀威胁创新的标准做法。更好的网络浏览器意味着对苹果iOS分销渠道的更大威胁。

不仅仅是文档查看器

现代浏览器本身就是操作系统。浏览器技术的存在几乎可以满足或超越原生浏览器的所有优势。几个例子:

  • 实时/推送消息。过去，在没有新请求的情况下，在页面加载后向浏览器发送数据是不可能的。今天服务器发送的事件和 Web Sockets 让它变得简单明了，而像GoInstant、Firebase和PubNub这样的服务让它变得微不足道。
  • 富媒体。大多数人都熟悉HTML5视频标签。现在我们也有(或即将有)方法访问网络摄像头和麦克风，建立点对点视频通信，甚至直接从浏览器处理语音输入。
  • 离线访问。Web应用程序可以离线运行，存储数据更改，并在网络可用时进行同步，就像本地应用程序一样。
  • 设备感知。现代移动浏览器可以访问设备方向、加速度计、GPS地理定位和网络连接信息。

随着新用例的出现，网络能否支持新技术不再是问题。只是什么时候的问题。

表现焦虑

对于许多网络预言家来说，唯一值得考虑的问题是性能。“网络永远不会像原生应用那样快！”他们哭了。他们是对的。但是网络不一定要和本地一样快，只要足够快就行。

最常被引用的是脸书放弃HTML5 作为网络不够快的“证据”。这个论点有几个问题:

  1. 轶事是糟糕的证据。即使在那个时候，其他人声称问题在于执行，而不是概念。
  2. 即使改进很小，脸书开发本地应用也是有意义的。脸书有能力雇佣整个开发团队来支持每个平台。大多数公司没有这么幸运。
  3. 这件事发生在大约两年前。那是15个Chrome发布周期之前。网络技术变化很快。

最近像 asm.js 这样的开发正在为浏览器带来像Unreal和Unity这样的高端游戏引擎。硬件加速转换使流畅的动画和过渡成为现实，甚至在移动设备上也是如此。与三年前相比，今天你在浏览器中所能做的事情似乎很神奇。

如果你正在构建一个图形密集型的游戏或视频/图像处理软件，尽一切可能构建原生的。然而，对于绝大多数应用程序来说，多平台原生将成为你的开发人员脖子上的锚。开发者对于软件平台的统治地位非常重要。

开发商，开发商，开发商

当用户规定应用程序功能的需求时，开发人员让它发生。作为一个平台，网络更容易学习，并且让开发者比其他平台做得更快。这就是为什么从Treehouse到Codecademy到General Assembly的公司比其他任何平台都更关注网络课程。

客观地说，网络的分布是地球上最大的。每个有本地应用的平台都有一个浏览器。这就是为什么HTML5实际上是T1的头号工作趋势。这就是为什么我能够在过去的一年里将Chromebook作为我的主要电脑。这也是为什么，不管竞争对手的短期优势如何，网络最终都会崛起。

网络确实有一些技巧可以从手机上学习。我们需要浏览器驱动的认证标准，比如谷歌在Android上的登录。我们需要更简单的方式来支付网上的东西( requestAutocomplete 是很好的第一步)。我们需要工具来使应用构建成为HTML的头等大事(由谷歌的聚合物项目领导的Web组件将改变一切)。作为一名从事web应用程序工作超过十年的开发人员，我相信这项技术会不断发展以满足开发人员的需求。

无论如何，走出去，尝试闪亮的新技术。这就是科技行业(和网络)变得更好的方式。小心不要被冲昏了头脑。从长远来看，与网络唱反调从来都是一个糟糕的赌注。我的建议是在放置它之前要非常小心。
