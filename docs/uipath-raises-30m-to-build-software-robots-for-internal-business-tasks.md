# UiPath 筹集 3000 万美元为内部商业任务制造“软件机器人”

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/04/27/uipath-raises-30m-to-build-software-robots-for-internal-business-tasks/>

当今世界充斥着机器人——部分或全部使用自然语言处理、机器学习、计算机视觉和其他人工智能技术来帮助消费者提问和回答问题、购物和完成其他事情的应用程序。在企业中，一家公司正在为大公司及其后台部门建立类似的东西，已经筹集了一轮相当大的资金。

罗马尼亚的一家初创公司 UiPath 为企业开发应用程序，以实现重复功能的自动化，如处理保险索赔或办理员工入职手续，该公司在由 Accel 牵头的首轮融资中筹集了 3000 万美元。该公司自 2012 年成立以来，一直没有宣布任何资金。这 3000 万美元还包括去年从 Earlybird、Credo Ventures 和 Seedcamp 筹集的 160 万美元种子资金。

这笔钱将用于增加更多的销售人员和生产更多的产品来满足现有的需求。“这项投资将使我们能够将智能 RPA[机器人流程自动化的缩写]的优势引入全球更多的企业，并保持在快速发展的行业的前沿，”UiPath 首席执行官兼创始人 Daniel Dines 说。

在进入市场的相对较短的时间里，UiPath 已经打开了一扇机会之窗，因为越来越多雇用数万名员工的公司正在寻找削减成本的方法，并以更有效的方式重新安排员工。

最黑暗的观点是人类最终会被取代，这给我们的社会和经济发展带来了很大的问题。更乐观的是，这些公司正在寻找方法，将员工从更平凡的事情中解放出来，专注于需要更多人际互动和智能的任务。

如果说前者是我们在适当的时候不可避免地要问自己的问题，那么现在正是后者为 UiPath 带来了商机，也引起了投资者的兴趣。(“A t Accel 我们相信人工智能，我们的投资一直围绕着这个主题，在很大程度上是一个组合，”Accel 的合伙人露丝安娜·利桑德鲁对我强调说。)

UiPath 目前的客户中有大约 200 家大型企业，包括汉莎航空、忠利、Telenor 和东能源，并与许多垂直行业的公司开展更广泛的合作:银行和金融服务；保险；制造业；公用事业；医疗保健和政府。大约 30%的客户在美国，另外 40%在欧洲，30%在亚洲。

虽然业务流程自动化并不是一个新的领域——自从我们甚至可以称之为业务流程自动化的开始，甚至在此之前(让我们称算盘为业务流程自动化的第一个例子), IT 公司就一直在构建加快文书工作和忙碌工作的方法——我们可以看看 UiPath 作为下一代如何构建这些类型的服务正在做什么。简而言之，他们正在将越来越多的“人类”特征带入等式中。

Lixandru 指出:“例如，在金融领域，许多团队花时间创建报告，但并不要求他们使用那么多的判断技能，这使得员工几乎没有时间来分析他们的工作。“使用计算机视觉理解屏幕上发生的事情并像人类一样执行任务的软件机器人成为了一种有用的工具。”

“我们正在为推动我们的业务和经济向前发展的人们提供更鼓舞人心和更有效的工作，而组织尚未开发的潜力正是使这个市场如此令人兴奋的原因，”Dines 补充道。

UiPath 提供了一个平台，可以引入各种人工智能工具，从计算机视觉和机器学习到自然语言处理，企业可以使用这些工具来启动和开发成熟的软件机器人。

像处理保险索赔这样的事情通常需要的不仅仅是基本事实，这是一个很好的例子，说明这种任务现在越来越被视为另一种可以更自动处理的形式。这些通常是根据手头的任务定制的，所以毫不奇怪，UiPath 也有一个庞大的合作伙伴网络，大约有 150 家公司帮助销售和实施其服务。其中包括埃森哲、德勤和凯捷等集成商。

其中大部分是内部流程，而不是外部的、面向客户的任务，尽管这显然是 UiPath 有潜力开发的领域。

尽管这仍是一个新兴领域，但竞争已经存在。他们包括来自美国的[自动化无处不在](https://web.archive.org/web/20230404083007/https://www.automationanywhere.com/)，该公司[似乎](https://web.archive.org/web/20230404083007/https://www.crunchbase.com/organization/automation-anywhere#/entity)从未透露任何资金，但自 2003 年以来就一直存在；在涵盖同一挑战各个方面的众多其他文章中。

对我来说，如果你认为自动化趋势是不可避免的，这里有很多机会。除了像 UiPath 这样在市场上构建解决方案的公司之外，还有像 Amazon 这样的公司已经在内部使用这样的流程，并且越来越多地通过 AWS 业务来寻找将自己的内部工具产品化的方法。对于 UiPath 这样的公司来说，这意味着竞争，也意味着潜在的收购。
