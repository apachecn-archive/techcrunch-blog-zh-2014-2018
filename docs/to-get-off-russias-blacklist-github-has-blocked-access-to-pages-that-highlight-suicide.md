# 为了摆脱俄罗斯的黑名单，GitHub 屏蔽了突出自杀的页面

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/12/05/to-get-off-russias-blacklist-github-has-blocked-access-to-pages-that-highlight-suicide/>

GitHub 正在俄罗斯互联网审查的棘手水域中缓慢航行，使用自己的平台来跟踪它是如何做到的，以努力保持透明，但也同意屏蔽监管机构认为违反内容法规的页面。

该公司表示:“此后，我们在俄罗斯联邦境内屏蔽了被俄罗斯联邦法律标记为禁止的特定内容，并正在努力恢复 GitHub。”。它引用其服务条款 A8 部分进一步阐述:“您不得将服务用于任何非法或未经授权的目的。在使用服务时，您不得违反您所在司法辖区的任何法律(包括但不限于版权法或商标法)。”

本周早些时候，在俄罗斯信息监管机构 Roskomnadzor，[称](https://web.archive.org/web/20230223072949/https://github.com/github/roskomnadzor/blob/master/2014-11-27-roskomnadzor.md)GitHub 发布非法内容后，整个 GitHub 在俄罗斯被屏蔽——特别是有人上传了提及自杀方法的页面。由于 GitHub 在 HTTPS 的设置，监管机构命令互联网服务提供商屏蔽导致整个网站关闭。

现在，GitHub 已经建立了一个 [GitHub 知识库](https://web.archive.org/web/20230223072949/https://github.com/github/roskomnadzor/)，在这里它将提供违规通知和其他声明的链接。

与此同时，它将继续遵守监管机构的规定，对有问题的特定内容进行地理屏蔽，以便 GitHub 的其他服务可以在该国不间断地继续。

屏蔽特定页面与 YouTube 等网站为了在俄罗斯保持整体在线所采取的路线相同，YouTube 也因自杀相关内容而被屏蔽。

其他网站，如脸书和推特，对此类请求的反应褒贬不一。在一些国家，这些服务是不可访问的(或偶尔如此)，除非你有一个私人 VPN 隧道，因为公司拒绝删除违规材料。在其他国家，比如法国，当涉及到反犹太主义时，Twitter 实际上满足了政府的一些要求。

值得注意的是，GitHub 的合规性会遇到一些阻力。

“虽然我们可能不总是同意俄罗斯政府做出的选择，但我们尊重该国的主权，并认识到俄罗斯人可能有不同的文化敏感性，”该公司指出。“我们对互联网审查感到担忧，并认为透明是一种美德。通过在这里张贴通知，我们可以更好地告知公众什么内容被 GitHub 拒绝，以及为什么。我们在这里张贴了撤下通知，以记录它们可能会让讲话变得冷淡。”

它接着指出，撤下并不意味着它同意这一决定。

它说，通知出现在档案库中，“仅仅意味着我们在指定的日期收到了通知。这并不意味着…拆除是非法的或错误的。这并不意味着通知中确定的用户做错了什么。我们张贴通知只是为了提供信息。我们不做或暗示对他们的主张的价值的任何判断。你可以得出自己的结论。”

俄罗斯的审查法律于 2012 年首次生效，令人担忧的是，这些法律不仅侵犯了言论自由，还可能被滥用来压制公开反对国家的人。

这篇描述自杀方式的特别笔记是一个有趣的例子，说明界限变得非常模糊。它似乎被写成了(特别愤世嫉俗，也许不太好笑)讽刺——咬舌头是方法之一——但其他选择包括加入军队，或获得一把好枪，如果你认识正确的警察，你可能会找到。或许这违反了关于不当自杀内容的规定，但这也是对权威的嘲弄。

在撰写本文时，GitHub 仍然被封锁，尽管这一举动应该是朝着澄清这一点迈出的又一步。
