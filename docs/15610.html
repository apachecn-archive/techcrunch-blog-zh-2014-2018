AWS 使查询存储在其 S3 和冰川存储服务中的数据变得更加容易和快捷

在拉斯维加斯的 re:Invent 会议上，AWS 今天宣布对其 S3 云存储和冰川冷存储服务进行了一次小而重要的更新——以及开发者如何访问其中的数据。

正如 AWS 首席执行官安迪·贾西(Andy Jassy)在他的主题演讲中指出的那样，该公司的客户倾向于将他们的大部分数据存储在其 S3 存储服务中，尽管事实上 AWS 也提供大量的数据库和数据湖服务。他们实际上使用 S3 作为他们的数据湖。

 

但是，通常当客户只需要位于 S3 存储桶中的对象的部分数据时。假设您想要一家零售店的数据，但该数据存储在一个大对象中。有了这项新服务，这些开发人员现在可以使用标准的 SQL 表达式来查询存储在 S3 和冰川中的对象的数据。

Jassy 指出，在 S3 上，这项被称为 S3 选择(cold storage 端的 Glacier Select)的新服务可以将数据访问性能提高 400%，因为你不必从存储中取出整个对象，然后再运行查询。

这项功能现已对 S3 和冰川用户开放。
