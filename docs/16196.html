如果你是一家相当大的公司的一部分，有可能某些资源只能通过内部网、内部网络或你公司称之为的任何东西获得。从公司外部访问这些内容的一种常见方式是 VPN，但 VPN 是一种相当笨拙的解决方案——像谷歌和亚马逊这样的公司正在放弃这种解决方案。现在(战地校友 ) Cloudflare 希望你也这样做，转而使用其新的访问服务。

虚拟专用网的基本概念是，不是将你的网络流量直接发送到你试图联系的网站或服务，无论是网飞还是公司内部网，而是发送到一个可信的服务器。该服务器将这些数据包发送到网站或服务，接收响应，然后将它们发送回您。

VPN 限制了敏感数据暴露给潜在的窥探，但它们通常会降低流量，并且还反映了一种过时的内部对外部的数据保护思想。

几年前，谷歌开创了一种保护事物安全的新方法:本质上，不要相信任何人，要验证每个人。身份认证的负担变得更重了，但安全性的简单性弥补了这一点，它在用户和设备而不是资源和服务面前筑起了一道墙。这意味着后者可以位于普通的公共互联网上，从任何地方都可以访问(当然是对那些被授权的人)。

亚马逊做了类似的事情，现在他们的两个系统都可供管理员使用…如果你的服务托管在他们的云平台上。但是如果你有几个这样的，几个那样的，几个第三和第四种呢？你可能会被虚拟专用网困住。

cloud flare 针对的就是这最后一种情况。新服务 Access 将与 Okta、Google Auth 等身份和认证公司合作，但提供跨多个平台的访问控制和加密，包括 GCS 和 AWS。

该公司声称它不应该降低交通速度；Cloudflare 已经在各地部署了用于 DDoS 保护和 CDN 服务的服务器，Access 将使用这些服务器，因此您对公司的请求不必经过伦敦或东京或其他任何地方。

本质上，Cloudflare 正在执行 VPN 的重要部分-检查证书和流量，建立数据包的信任链-以一种不那么笨重的方式，使公司能够让数据驻留在云服务上，而不是内部服务器上。

可以给一个员工免费试用(大方！)但在那之后，每个月的人均费用开始为 3 美元，当然还有批量折扣。您可以在 Cloudflare 访问网站了解更多。
