云计算原生计算基金会将 Linkerd 作为其第五个托管项目 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2017/01/23/cloud-native-computing-foundation-adds-linkerd-as-its-fifth-hosted-project/?ncid=rss

云本地计算基金会 (CNCF)，开源组织，也是流行的 Kubernetes 容器管理系统的家，今天宣布它已经增加了 Linkerd 作为它的第五个托管项目。Linkerd(发音为 linker-DEE)在浮力孵化，跟随其他 CNCF 项目的脚步，如 Kubernetes、Prometheus、OpenTracing 和 Fluentd。

CNCF 的使命是“创建并推动采用一种新的计算模式，这种模式针对现代分布式系统环境进行了优化，能够扩展到数万个自我修复的多租户节点。”

Linkerd 在所有这些中角色是为这些现代的、通常以容器为中心的应用程序提供一个所谓的“服务网格”。Linkerd 是一个独立的代理，其核心是使不同的服务能够相互通信。

“因此，服务 A 不会直接与服务 B 对话，而是会通过 L inkerd 实例进行对话(如果我们做得不错，服务 A 实际上不需要知道它正在通过 L inkerd )，”bubble CEO 兼联合创始人威廉·摩根向我解释道，他之前在 Twitter 上处理过类似的问题。“在实践中，你会部署数十、数百或数千个 L inkerd 实例，它们会形成一个弹性通信的‘网’。”所以就像你已经熟悉的那种代理一样(想想 Nginx 处理 web 流量)，Linkerd 为内部流量提供了类似的特性。

Morgan 认为，现在这变得尤其重要，因为现代应用程序通常不只是调用几个不同的服务(web 服务器->应用->数据库)，而是可能调用十几个微服务。“我们的基本信念是，在这个现代应用架构的世界里，对弹性应用的需求变成了对弹性服务通信的需求，”Morgan 说。

除了形成语言无关的消息层，Linkerd 还提供负载平衡、故障和延迟处理以及其他服务，旨在保持基于微服务的应用程序的响应能力。

到目前为止，浮力已经筹集了大约 350 万美元。Linkerd 是其旗舰开源工具，但其主要关注点是氦，这是一种承诺为企业提供对其云原生应用程序的洞察和控制的服务。

该公司为什么向 CNCF 捐赠 Linkerd？摩根告诉我，最有可能从项目中获得价值的公司正是那些已经在尝试运行“云原生”环境的公司。“此外，这对我们来说真的很重要，作为一家初创公司，我们正试图平衡 a)拥有一个提供具体价值的开源项目，拥有一个蓬勃发展的社区，并且作为一个独立的产品是可行的(即，感觉不像一个残废的软件)；同时 b)建立一个可以盈利、可以吸引顶级人才、可以为其他公司解决关键基础设施问题的企业——CNCF 对这种双重性非常开放，像我们一样，相信你可以做到这两者，而且你可以做得很好，”摩根补充道。
