Docker 将 dotCloud 出售给 cloudControl，专注于核心容器业务 TechCrunch

原文：https://web.archive.org/web/http://techcrunch.com/2014/08/04/docker-sells-dotcloud-to-cloudcontrol-to-focus-on-core-container-business/?ncid=rss

Docker 公司。开源 Docker 项目背后的商业公司今天宣布，它已经将其 dotCloud 平台作为服务业务出售给了总部位于柏林的 cloudControl 。这笔交易使 Docker 可以专注于其核心的软件集装箱化业务。

Docker 首席执行官 Ben Golub 告诉 TechCrunch，该公司希望专注于 Docker 业务。“在过去的 18 个月里，我们已经看到 Docker 在很大程度上有了提升。他解释说:“现在已经到了我们需要集中所有精力和资源的时候了，我们想为点云产品找到一个对得起客户的家。”。

Golub 表示，他们正在寻找一个符合一些非常严格标准的买家。首先，他们希望它是另一个在市场上受到高度重视的 PaaS 供应商，他们希望有人能够继续维护平台，而不仅仅是争夺平台的 500 名客户。

在 dotCloud 网站上的一篇博客文章中，开发者支持经理 Andrew Rothfus 证实了此次收购，“今天，我很高兴地宣布 dotCloud 平台即服务已被正在向美国扩张的德国 PaaS 提供商 cloudControl GmbH 的美国子公司收购。”

博客文章接着说，该公司将保留其名称，正如你在这些情况下所预料的那样，他们努力向客户保证将有业务连续性，他们将尽一切努力与新的公司母公司顺利整合。

Golub 补充说，“他们在欧洲很大，他们想扩展到我们这里，这是他们可以做到的。”

Docker 最近成为热门的容器化技术，并激发了开发人员的想象力。TechCrunch 上的一篇文章这样描述 Docker:

“Docker 1.0 是另一个利用谷歌开发的容器化技术的产品，在这种情况下，它提供了一种安全的方法来分发应用程序，而不会在每次做出更改或进入开发周期的不同阶段时破坏它们。”

至于 dotCloud，它不是公司使命的一部分，他们看到了一个出售它的机会，并抓住了这个机会。对于柏林初创公司 cloudControl 来说，这笔交易看起来很有意义。两家公司都专注于平台即服务。两家公司都吹嘘能够在云中部署、管理和扩展应用程序。这两家公司似乎是一个很好的组合，dotCloud 让 cloudControl 在 dotCloud 的客户方面迅速增长，并在美国市场迅速站稳脚跟。

正如博客文章所述，当公司更名为DockerInc .时，围绕 dotCloud 存在许多不确定性。这一举措使 dotCloud 客户成为一家更加忠诚的公司，完全专注于平台即服务，从长远来看，他们应该会从中受益。

Golub 表示，Docker 现在多达 50 人，其中 4 人一直致力于 dotCloud 产品。在所有权转移期间，他们将继续作为 Docker 员工，并在 90 天内帮助 dotCloud，并将慢慢退出，全职进入 Docker 职位。

Golub 不会透露这笔交易的价格，但他说，总体而言，这不是钱的问题，而是为他们的 dotCloud 客户找到一个好的归宿。“这是一笔有趣的交易，但主要价值是为客户找到一个好的归宿，这样我们就可以专注于 Docker。
