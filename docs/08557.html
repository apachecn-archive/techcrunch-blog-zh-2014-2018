这些天有很多关于“云 本土的谈论，人们对这个术语的解释很随意——就像一样，他们对与云相关的一切都很随意。

凯德·梅斯在《连线》杂志的这篇文章中说得很对:“这个词在最近几年有了很多含义。但请记住:这些含义大多来自 IBM、惠普、EMC、戴尔、思科和其他不想被它迷惑的公司。”

“云 原生”简单地指的是在云中诞生的应用程序，而不是在本地诞生和成长的应用程序。本地就像拥有一个车队。你不仅有一个巨大的资本支出(购买汽车)，而且你还必须支付维修费用。

IaaS 作为垫脚石

当本地应用程序迁移到公共云时，第一步就是简单地将它们迁移到云T21。本质上，这仅仅意味着重新创建本地基础架构。这种方法作为起点很有意义，因为它减少了未知的数量，降低了被解雇的风险。说到解雇，是时候改变说法了:

“从来没有人因为收购 IBM 而被解雇。”

收件人:

"没有人会因为转到 AWS 而被解雇."

为了进一步说明这种方法，如果一家公司将一个 100 节点的集群从本地迁移到云，它只需租用 100 个实例，并以与本地完全相同的方式运行应用程序和服务(在安装了相同版本的操作系统和支持/基准软件之后)。这种租用基础设施的方案被称为基础设施即服务，简称 IaaS。

IaaS 提供了双重好处:伸缩性和抽象性。扩展的好处体现在如何按需添加和删除机器上。这一过程只需点击几下按钮，而在本地则需要数周时间。抽象优势体现在硬件/数据中心基础设施层面。IaaS 提供全球基础设施，无需在同一地区和不同地理区域之间创建和维护多个数据中心。有了 IaaS，就不需要担心管理机器、机架、网络、冷却、电力使用等等。

总体而言，IaaS 和云计算的一个额外好处是成本从资本支出转变为运营支出。IaaS 相当于租了一个车队。你要负责的只是房租，不管你是按小时租、按周租还是按月租。

IaaS 是一个良好的开端，事实上是非常基础的。每个试图跳过 IaaS 部分直接销售 PaaS 和 SaaS 产品的云提供商都遇到了挑战。例如，微软最初从 Azure 上的 PaaS 服务开始，获得的成功非常有限。他们在 2012 年给 Azure 添加了 IaaS，从那以后就没有回头过。

当我们谈到 SaaS 的话题时，行业内似乎出现了一种新趋势，即将互联网上运行的一切称为“SaaS”，并将其与云业务捆绑在一起，以扩大云收入。看看甲骨文的这篇文章，去搞清楚什么是真实 云收入。

PaaS

在迁移到公共云之后，公司通常需要一些时间来适应新环境。对一些公司来说，可能要几个月；几年来。这些定居者也需要时间来感受他们在本土的安全感。

随着定居者同化并熟悉新环境，他们可能会观察到一些有趣的事情。他们看到的是，有现成的服务可以做他们多年来辛辛苦苦手工做的事情。IaaS 帮助他们抽象出硬件，这些服务甚至帮助他们抽象出操作系统。应用程序可以简单地在这些服务的基础上重建，这些服务为不同的需求提供了通用的平台。这种增强被称为平台即服务，简称 PaaS。

PaaS 不仅带来了简单性，而且在 IaaS 的基础上显著降低了成本。

公共云领域的所有改进都集中在实现两个目标上:降低成本和简化事情

PaaS 是一个重要的步骤，因为这是云 本地的第一个接触点。它介绍了重新构建应用程序的思想过程，以充分发挥它们在公共云上的潜力。这些应用程序以 PaaS 的形式提供，涵盖了广泛的领域。亚马逊网络服务(AWS)是 PaaS 产品的领导者，就像在 IaaS 领域一样。在数据存储方面，他们有 AWS Aurora，提供 RDBMS 的替代品；DynamoDB，替代 NoSQL 数据库；和 Redshift 为企业数据仓库提供了替代品。

PaaS 就像是在你需要的时候使用优步。您不用支付租金，而只需支付从 A 地到 b 地的车费。您接受的服务类型取决于您的需求。如果你的目标是去观光，那么也许乘坐“大巴士”是一个更好的主意。

无服务器架构

PaaS 提供的东西是惊人的，大多数应用程序将从此幸福地生活在 PaaS 的美好世界中。但是一些应用程序将更进一步，以最小化成本和增加简单性。亚马逊网络服务公司宣布 AWS Lambda 的广泛使用是朝着同一个方向迈出的一步。微软 Azure 已经宣布将 Azure Functions 作为竞争产品；谷歌也是，有了谷歌云功能。

实现这一点的关键是将应用程序分解成组成它的各个功能。函数是伟大的，因为它们是在指定的运行时上运行的小代码块，如 JVM 或 Python。应用程序开发人员甚至不必担心运行这些运行时。这是任何人能想到的最高层次的抽象(目前为止！).

对于 Docker/container 爱好者来说，这与容器所做的事情没有太大的不同。容器是运行应用程序的抽象。这是对它们的即兴创作，其中粒度已经转移到单个功能。

更接近它的是“微服务”事实上，一种流行的设计模式是在这些功能前面加上由 API 网关管理的微服务。

功能是关于你最初为什么想去旅行，就像把衣服拿到洗衣店洗熨。函数让你担心你想做什么，而不是怎么做。

摘要

公共云领域的所有改进都集中在满足两个目标上:降低成本和简化事情。IaaS、PaaS 和 Lambda 逐步实现这一目标。

众所周知，公司正在烧掉内部的桥，并且知道公共云是他们的命运，他们正在关注如何优化成本。这包括以下列方式重新构建应用程序，以最大限度地利用作为云 本地的优势:

  • 从传统数据存储(想想 Oracle/MySQL/Teradata)转移到云- 原生数据存储(想想 Aurora/Redshift)。
  • 利用容器和以应用程序为中心的抽象。
  • 最后但同样重要的是，将应用程序分解成单独的功能，并在功能级抽象上工作。
