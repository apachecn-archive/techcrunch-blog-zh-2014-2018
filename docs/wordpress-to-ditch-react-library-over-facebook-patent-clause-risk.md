# WordPress 因脸书专利条款风险放弃 React 库

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/09/15/wordpress-to-ditch-react-library-over-facebook-patent-clause-risk/>

流行的开源网络出版软件 WordPress 的联合创始人马特·莫楞威格表示，由于担心脸书开源许可证中的专利条款，社区将不再使用脸书的 React JavaScript 库。

昨天，在一篇解释这一决定的博客文章中，Mullenweg 说他希望 WordPress 正式采用 React——他指出，几年前，WordPress.com 背后的 Automattic 公司(他也是创始人之一)已经在 Calypso 对《WordPress.com》的彻底重写中使用了 React，而 WordPress 社区已经开始在其主要的 Gutenberg 核心项目中使用它。

但他说，在看到脸书在专利条款背后钻牛角尖后，他改变了主意——该条款最近被添加到阿帕奇软件基金会(ASF)的禁止许可名单中。

在 ASF 的“X 类”列表中，React 专利条款现在所在的位置写道:

> 脸书 BSD+专利许可证包括一个专利文件的规范，该规范将我们软件的风险转移给下游消费者，这种不平衡有利于许可方，而不是被许可方，因此违反了我们作为[通用捐助方](https://web.archive.org/web/20230306185552/https://s.apache.org/4Uzg)的 Apache 法律政策。脸书 BSD+专利许可条款不是 ALv2 中的子集，它们不能作为 ALv2 进行再许可。

上个月脸书写了自己的[博客文章](https://web.archive.org/web/20230306185552/https://code.facebook.com/posts/112130496157735/explaining-react-s-license/)回应 ASF 的决定——其中承认它没有“很好地解释专利条款背后的原因”。并继续声称这是必要的，因为脸书的业务已经“成为毫无价值的专利诉讼的更大目标”，写道:

> 当我们在 3 条款 BSD 许可证下发布软件时，我们决定添加一个明确的专利授权，创建了被称为 BSD +专利许可证的东西。专利授权说，如果你打算使用我们根据它发布的软件，如果你起诉我们侵犯专利，你就失去了我们的专利许可。我们认为，如果这种许可被广泛采用，它实际上可以减少所有采用者的无价值诉讼，我们希望与其他人合作探索这种可能性。
> 
> 我们尊重第三方的知识产权，包括专利，并希望其他人也尊重我们的知识产权。BSD +专利许可证只是为了给我们的团队更多的空间来为开源做出有意义的贡献，同时减少我们花在打无意义的官司上的时间。

不管怎样，穆伦韦格说他的担忧并没有减轻。他写道，凭良心说，他不能要求广泛使用的开源 WordPress 软件的用户继承专利条款和相关的法律风险。所以他决定放弃 React。

“我认为脸书的条款实际上比许多其他公司可以采取的方法更清晰，脸书一直是那里更好的开源贡献者之一。但是我们有很多问题要解决，让世界相信脸书的专利条款没有问题不是我们的责任。这是他们的战斗，”他写道。

他说，这一决定意味着古腾堡项目将推迟——至少几周——因为它是用不同的库重写的，并指出这可能会将发布推迟到明年。

关于 Calypso 的重写，他只是说这将需要“更长的时间”，并补充说:“automatic 对专利条款仍然没有问题，但与 core 的*长期一致性*比重写对 automatic 的业务造成的*短期打击*更有价值。超过四分之一的网站都有 WordPress 的核心更新，让它们都继承专利条款让我很不舒服。

我们已经联系了脸书进行评论，并将根据任何回应更新这篇文章。

专利条款附带的感知风险取决于如果 React 用户质疑其专利或起诉该公司专利侵权，脸书如何撤销专利许可——这意味着脸书可以对起诉其专利侵权的个人或实体提起专利侵权诉讼。

因此，公司，尤其是那些拥有大量专利组合的公司，很可能会担心他们是否正在使用包含脸书 React 框架的开源软件——即使 Automattic 对自己的帐户感到满意。

专利条款的一些最激烈的批评者将 React 称为“进入开源社区的特洛伊木马”。

在 ASF 的举动之后，开发者 Kevinflo 在[黑客新闻](https://web.archive.org/web/20230306185552/https://news.ycombinator.com/item?id=15051467)上写道:“即使我们尽可能以最有利的方式描绘他们的所有行为，即使该条款像一些人声称的那样是一只纸老虎，也没关系。开源不应该这样工作。我们不应该为了一个项目的许可是否具有放射性而争论多年。尤其是像我这样只想使用一个伟大工具的个人开发者。我们应该能够使用它，因为它是开放的，这就是开放的含义。这比关门糟糕多了。它是封闭的，伪装成开放的。”

弗洛伦萨诺现在正在庆祝离开 React 的决定…

Mullenweg 说，他还没有决定用哪个库来取代 React，但他补充说，这个决定“主要”是技术性的。

“我们会寻找一些具有 React 大部分好处的东西，但没有让许多人感到困惑和威胁的专利条款的包袱，”他说，并补充说:“感谢迄今为止花时间分享他们的想法并就这些问题提供反馈的每个人——我们一直在倾听。”

他博客上的评论大多支持这一举措——一位评论员称之为“艰难但重要的决定”，其他人则称之为“明智”和“好”的决定。

尽管另一个警告不要反应过度，写道:“不要反应过度。在过去的 5、6 年里，WP 生态系统已经有足够多的混乱和变动。Fb 的业务规模和范围使该条款变得可怕。他们最终将不得不放弃。”
