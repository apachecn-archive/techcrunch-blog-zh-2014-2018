原文：https://web.archive.org/web/http://techcrunch.com/2014/10/24/psa-dont-open-random-powerpoint-presentations-from-strangers-right-now/?ncid=rss

现在不要随意打开陌生人的 PowerPoint 演示文稿(或者永远不要，真的)

抬头！感觉像是回到了 90 年代末 2000 年代初，微软在微软 Office 中发现了一个巨大的漏洞。如果执行得当，该漏洞可能会被利用来接管运行任何版本 Windows 的整个系统。

你可以在这里找到微软关于这个 bug 的完整披露，但是这里有你应该知道的大部分内容:

  1. 这个 bug 正在被广泛利用，尽管微软目前只知道“有限的、有针对性的攻击”
  2. 它会影响 Windows Vista、Windows Server 2008、Windows 7、Windows 8、Windows Server 2012 和 Windows rt。XP 没有被提及，因为微软不再支持它——但是的，它可能也会受到影响。
  3. 如果执行得当，该漏洞会让攻击者在您的系统上拥有与您当前登录的任何类型的用户相同的权限。如果你是管理员，这意味着完全的管理权限——代码执行、应用安装等。
  4. 如果你启用了 Window 的用户帐户控制功能，它会弹出一个提示，询问文件是否可以执行。如果你不能 100%确定文件是合法的，避免这样做。
  5. 这个 bug 是 PowerPoint 的 OLE 系统的一部分，它可以让你在演示文稿中嵌入电子表格之类的东西。它被认为是相当好的沙盒；唉，看来有人找到缺口了。
  6. 微软表示，通过电子邮件发送给用户的黑客演示文稿和放在网上的黑客演示文稿都有潜在的危险。简而言之:现在除了最受信任的 PowerPoint 演示之外，不要做任何演示。

那么这只虫子到底有多粗糙呢？微软说(强调我们的):

成功利用此漏洞的攻击者可以获得与当前用户相同的用户权限。如果当前用户以管理用户权限登录，成功利用此漏洞的攻击者可以完全控制受影响的系统。然后，攻击者可以安装程序；查看、更改或删除数据。或者创建具有完全用户权限的新帐户。

但是你应该恐慌吗？

那鸿做一些事情，你应该是安全的:
1)聪明地对待你打开的演示文稿
2)如果你的 Windows 版本有用户帐户控制选项，启用它(在大多数情况下，它应该是默认打开的。)这不会彻底修复 bug，但会抛出一个很大的权限提示，提醒你不要打开神秘文件。查看来自微软的咨询页面，它提供了一个临时补丁，直到微软完成一个安全更新。
