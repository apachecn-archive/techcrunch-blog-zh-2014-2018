# 有人收集了 40000 张 Tinder 自拍照，为人工智能实验制作了一个面部数据集

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/04/28/someone-scraped-40000-tinder-selfies-to-make-a-facial-dataset-for-ai-experiments/>

Tinder 用户有很多动机将自己的肖像上传到约会应用。但是，当他们注册 swipe 时，将面部生物特征贡献给可下载的数据集以训练卷积神经网络可能不是他们的首选。

Kaggle 是一个机器学习和数据科学竞赛平台，最近被谷歌收购，该平台的一名用户上传了一组面部数据集，他说这是利用 Tinder 的 API 从这款约会应用的湾区用户那里收集了 4 万张个人资料照片——每种性别各 2 万张。

这个名为“火绒的 T2 人”的数据集由六个可下载的 zip 文件组成，其中四个包含大约 10，000 张个人资料照片，两个文件包含每个性别大约 500 张照片的样本集。

一些用户的个人资料中有多张照片，所以这里代表的 Tinder 用户可能远少于 4 万人。

数据集的创建者 Stuart Colianni 以 [CC0: Public Domain License](https://web.archive.org/web/20230329081200/https://creativecommons.org/publicdomain/zero/1.0/) 发布了该数据集，并将其 scraper 脚本上传至 [GitHub。](https://web.archive.org/web/20230329081200/https://github.com/scoliann/TinderFaceScraper)

他将其描述为“一个简单的脚本，用于收集 Tinder 个人资料照片，以创建面部数据集，”他说他创建刮刀的灵感来自于对其他面部数据集的失望。他还将 Tinder 描述为提供“几乎无限的访问权限来创建面部数据集”，并表示抓取该应用程序提供了“收集此类数据的极其有效的方式。”

“我经常失望，”他写道其他面部数据集。“数据集在结构上往往非常严格，而且通常太小。Tinder 可以让你接触到你周围几英里内的数千人。为什么不利用 Tinder 建立一个更好、更大的面部数据集呢？”

为什么不呢——也许，除了成千上万个人的隐私，他们的面部生物识别信息被你放在网上的一个海量存储库中，供公众重新利用，完全没有他们的发言权。

浏览其中一个可下载文件中的一些图片，它们看起来肯定像是人们在 Tinder(或者其他在线社交应用)上用于个人资料的准亲密照片——混合了自拍、朋友合照和可爱动物或模因照片等随机内容。如果你只是在寻找人脸，那么这绝不是一个完美的数据集。

反向图片搜索一些照片在网上几乎找不到完全匹配的照片，所以看起来很多照片没有上传到公开的网络上——尽管我能够通过这种方法识别出一张个人资料图片:圣何塞州立大学的一名学生，他在另一个社交资料中使用了相同的图片。

她向 TechCrunch 证实，她“不久前”加入了 Tinder，并表示她实际上不再使用它了。当被问及她是否对自己的数据被重新用于人工智能模型感到高兴时，她告诉我们:“我不喜欢人们用我的照片进行一些悲伤的‘研究’。””在这篇文章中，她不愿透露自己的身份。

Colianni 写道，他计划使用谷歌 tensor flow Inception 的数据集(用于训练图像分类器)，尝试创建一个能够区分男女的卷积神经网络。(我只是希望他先去掉所有的宠物照片，否则他会发现这个任务是一场艰苦的斗争。)

三天前上传到 Kaggle 的数据集(不包括样本文件)目前已经被下载了 300 多次——显然没有办法知道它可能会有什么额外的用途。

这些年来，开发者们做了各种各样怪异、古怪和令人毛骨悚然的事情来玩 Tinder 的(表面上)私有 API，包括[将它黑成自动喜欢每个潜在的约会](https://web.archive.org/web/20230329081200/http://valleywag.gawker.com/techies-are-hacking-tinder-in-a-desperate-attempt-to-ge-1621177524)以节省拇指滑动；为人们提供付费查询服务[来检查他们认识的人是否在使用 Tinder](https://web.archive.org/web/20230329081200/https://qz.com/656947/the-troubling-way-that-anyone-can-spy-on-any-tinder-user/)；甚至建立一个钓鱼系统来诱捕好色的兄弟，让他们[不知不觉地互相调情](https://web.archive.org/web/20230329081200/http://www.theverge.com/2015/3/25/8277743/tinder-hack-bros-swiping-bros)。

因此，你可能会说，任何在 Tinder 上创建个人资料的人都应该做好准备，让他们的数据以各种不同的方式从社区的多孔墙中流出——可以是一张截屏，也可以是通过上述 API 黑客之一。

但是大规模收集数以千计的 Tinder 个人资料照片作为喂养人工智能模型的饲料，确实让人感觉又一条线被跨越了。在[争夺大数据集以推动人工智能应用](https://web.archive.org/web/20230329081200/https://techcrunch.com/2016/07/09/we-need-to-talk-about-ai-and-access-to-publicly-funded-data-sets/)的过程中，显然没有什么是神圣的。

同样值得注意的是，在同意该公司的 [T & Cs](https://web.archive.org/web/20230329081200/https://www.gotinder.com/terms) 条款时，Tinder 用户授予其“全球范围内、可转让、可再许可、免版税、托管、存储、使用、复制、展示、复制、改编、编辑、发布、修改和分发”其内容的权利和许可——尽管尚不清楚这是否适用于第三方开发者抓取 Tinder 数据并根据公共领域许可发布的情况。

在撰写本文的时候，Tinder 还没有回复就其 API 的这种使用发表评论的请求。但由于 Tinder 使其对你的内容的权利可以转让，假设它批准了 Colianni 使用其 API，那么即使这种大规模的数据再利用也完全有可能属于其条款和条件的范围。
