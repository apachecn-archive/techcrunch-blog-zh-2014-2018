优步的 API 联盟项目为开发者推荐的每个用户支付 5 美元

原文：https://web.archive.org/web/https://techcrunch.com/2015/06/30/uber-pays-to-be-everywhere/amp/

优步希望它的“请求搭车”按钮出现在尽可能多的应用程序中，并愿意为此付费。现在的开发者将从每个美国用户那里获得 5 美元，他们通过在自己的应用中集成 API 来注册优步。地图、餐厅、旅游和夜生活应用开发商可能会抓住这个机会增加额外的收入来源。

开发者可以在这里注册优步 API 联盟项目。一旦达到 250 美元，他们就会得到报酬，收入上限为 5000 美元。届时，他们必须加入正式合作伙伴计划才能获得更多收入。通过 API 注册的用户可以免费获得高达 20 美元的首次乘车费。

优步在去年年底发布了它的 API ，但是开发者的采用并不热烈。启动合作伙伴包括 OpenTable、星巴克、猫途鹰和联合航空公司。从那以后，Foursquare、Hinge、微软和 Pebble 也加入了进来。

考虑到优步在每个用户的一生中赚了多少钱，以及它与其他乘车应用的竞争，以这种方式投资用户获取从长远来看可能会带来巨大的回报。优步收取大约 20%的费用，这意味着以这种方式获得用户的 5 美元成本在一次乘坐后可能是盈亏平衡的，假设这次旅行总收入为 25 美元，这是一个不常见的价格。

优步有能力承担该项目的短期成本。该公司是世界上资本最好的公司之一，迄今为止已经筹集了数十亿美元。今天的新闻强调，优步仍然积极部署资本来发展其平台，并推出像 UberPool 这样的新产品，这是一种让用户以更低的价格分享乘车服务的方式。

该公司在国内面临着激烈的竞争，既有类似的服务 Lyft，也有正在努力实现车队现代化的出租车公司，还有迄今为止让优步如此受欢迎的 mimc 部分。

如果该计划在本土市场获得成功，不难猜测优步可能会将其推广到更多市场。
