# 政府不可能建立自己的 5G 网络 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/01/29/theres-no-way-the-government-is-building-its-own-5g-network/>

Axios 本周末的一份报告引用了国家安全委员会内部的文件，描述了由美国政府建设和运营 5G 网络的可能性——不，是不可避免的。官员们给这个想法泼了冷水，事实上，它从来都不可行。

简而言之，Axios 引用的报告表明，真正保护下一代无线网络(无人驾驶汽车等关键基础设施将依赖于此)免受中国和其他国家监视的唯一方法是政府自己建设网络。

这个想法有几个问题。你可能在说出这句话之前就想到了几个。

或许最明显的问题是，为了建设一个新的 5G 网络，政府可能不得不与它将有效取代的公司(如美国电话电报公司和威瑞森)签订合同，或者至少与其密切合作。当然，他们是知道如何去做的人。(披露:威瑞森拥有 Oath，Oath 拥有 TechCrunch。)

这将会很尴尬，因为这些公司以及世界各地的其他公司都已经进入了测试和部署 5G 网络的过程。政府网络独立运作但与商业网络协同运作的想法是站不住脚的(我们以前也考虑过)。

即使尝试了，美国政府也没有办法，即使在最好和最有效的情况下，如果明天就开始两党合作，也不可能在这样一个部署的时间和规模上有任何竞争力。这需要数十亿美元和数年的工作来为这样的事情奠定基础，而其他人有一个巨大的开端。让我们不要忘记，我们正在经历一系列无休止的预算危机，这些危机不会因为提出这种大规模的计划而得到缓解。

联邦通信委员会主席 Ajit Pai 对这一提议毫不留情，这是一个赞成自由市场力量而不是政府干预的人所期望的:

> 我反对联邦政府建设和运营全国性 5G 网络的任何提议。从无线行业过去三十年的发展(包括美国在 4G 领域的领先地位)中汲取的主要教训是，市场，而不是政府，最适合推动创新和投资。政府能够也应该做的是将频谱推向商业市场，并制定规则鼓励私营部门开发和部署下一代基础设施。任何建设国有化 5G 网络的联邦努力都将是昂贵的，并且会适得其反，分散我们帮助美国赢得 5G 未来所需的政策。

克莱伯恩专员对这个想法也同样不友好:

> 美国在 5G 部署方面的领导作用至关重要，必须正确对待。地方可以发挥核心作用；应利用工业界拥有的技术专长；网络安全必须是一个核心考虑因素。我担心，由联邦政府建设的网络没有利用我们国家赢得 5G 竞赛所需的最佳方法。

参议员马克·沃纳(D-VA)加入了合唱，并加入了他自己的一点挖苦:

> 虽然我很高兴特朗普政府认识到维持美国在信息时代的领导地位需要大量的投资承诺，但我担心建设一个国有化的 5G 网络既昂贵又重复，特别是在特朗普政府提议削减对 R&D 的关键联邦投资和对无服务地区的宽带支持的时候。

特朗普政府本身[告诉 Recode](https://web.archive.org/web/20230215105811/https://www.recode.net/2018/1/29/16945452/donald-trump-5g-wireless-network-national-security-council-memo)Axios 看到的文件已经过时，这不是一个经过认真考虑的提议。国家安全委员会并不监督宽带部署，这是不可能通过联邦通信委员会的，正如你在上面看到的，联邦通信委员会可能会联合起来反对这样的策略。

虽然国家安全委员会在建议这条道路时显然考虑到了国家安全，但坦率地说，我不相信政府会建立任何类型的*安全网络，更不用说这么大的网络了。对于一个试图保护其互联网基础设施、同时又试图破坏有效加密的政府来说，这也具有相当大的讽刺意味。*

美国政府很可能会将正在建设的 5G 网络的一部分用于自己的目的，它当然会补贴该技术的推出，以便它可以自己使用它——第一反应网络，军事用品，诸如此类的东西。但认为它会或能够以建议的方式建立一个有竞争力的 5G 网络的想法在现实中没有依据。
