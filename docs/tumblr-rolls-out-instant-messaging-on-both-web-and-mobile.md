# Tumblr 推出网络和移动即时通讯

> 原文：<https://web.archive.org/web/https://techcrunch.com/2015/11/10/tumblr-rolls-out-instant-messaging-on-both-web-and-mobile/>

Tumblr 今天推出了一项用户群期待已久的功能:消息传递。这个博客平台长期以来一直为用户提供收件箱功能和相互提问的方式，但现在它也增加了即时消息。这项功能现在已经在网络上以及 Tumblr 在 iOS 和 Android 上的移动应用程序中推出。

鉴于分阶段推出，并非所有用户都会立即收到 Tumblr 消息。相反，那些看到屏幕上挂着一个“笑脸气球”(如上图)的人会知道他们的帐户已经打开了该功能。此时，如果你选择给另一个 Tumblr 用户发消息，他们也会收到消息功能，如果他们还没有的话。

该公司表示，更广泛的推广将需要几周时间才能覆盖 Tumblr 的所有用户群。

为了在移动设备上使用该功能，您需要更新到 Tumblr 移动应用程序的最新版本。

目前，几乎任何博客都可以使用消息传递，除了私人博客或团体博客。与 Tumblr 的“提问”功能不同，即时消息不能匿名。也就是说，收件人至少能够通过你的 Tumblr 帐户信息来识别你，尽管这并不意味着他们知道你的真实姓名，如果这不是你已经在你的网站上提供的信息。

![tumblr_nxm0z9Cf2M1rdlo21o1_400](img/ff96b7cfe30901e993d72be59d7ac1a5.png)Tumblr 表示，用户可以控制如何通过即时消息联系到他们。虽然任何博客都可以给你发消息，但有一个选项可以将收到的消息仅限于你关注的 Tumblr 用户。当然，你也不能给任何屏蔽你的人发消息。

要在网上使用该功能，您需要点击仪表盘右上角的聊天气泡。在手机上，你也可以点击一个类似的聊天气泡。接下来，你要输入你想发消息的人的博客名，然后输入你的文本。大概就是这么多了。

此外，Tumblr 指出，消息传递的推出将影响其粉丝邮件功能的未来发展。实际上，短信将取代粉丝邮件。这一变化不会影响用户的旧粉丝邮件，但未来任何发出的邮件都必须转移到消息平台上。但是，现在的提问和提交方式不会有任何改变。

虽然看起来是一个小功能版本，但消息传递有可能将 Tumblr 转变为一个比现在更社会化的服务，因为它为平台提供了一个实时、迷人的元素。这不仅在移动设备上运行良好，推送通知可以将用户拉回 Tumblr 应用程序，并让他们与 Tumblr 平台进行更长时间的互动，而且它还为 Tumblr 提供了平台基础，可以随着时间的推移进行改进，以添加更多丰富的媒体功能，类似于其他消息应用程序中的功能。

Tumblr 的高级用户通过流行的 XKit 等第三方扩展利用即时消息已经有一段时间了。这表明 Tumblr 的服务上对此类活动有一些需求。但 Tumblr 的年轻用户群通常对变化反应消极，因此当它首次推出时，可能会有一些反对消息传递的声音。已经有[用户](https://web.archive.org/web/20230326023307/http://sleep-wizard.tumblr.com/post/132946168120/staff-why-did-replies-have-to-go-tho-like-this)因为它取代粉丝邮件的方式而抱怨，而[其他人](https://web.archive.org/web/20230326023307/http://slicies.tumblr.com/post/132946145265/wow-tumblr-is-gonna-have-instant-messaging-soon)则谨慎[乐观](https://web.archive.org/web/20230326023307/http://nihilistgirlfriend.tumblr.com/post/132946122574/didnt-we-ask-for-a-messaging-system-for-like-4)。

例如，与 Facebook Messenger 或 WhatsApp 相比，Tumblr 消息传递的有趣之处在于，它有可能将线下互不相识的人联系起来，这些人不是真实世界的朋友。相反，Tumblr 的与众不同之处在于，它的用户经常围绕兴趣联系在一起——比如粉丝，或者他们不愿意与朋友和家人讨论的各种个人问题。

该公司暗示，今天推出的消息服务只是其围绕即时通信计划的开始。随着时间的推移，该公司将增加对其他对其用户群有意义的东西的支持，包括 gif 和视频。
