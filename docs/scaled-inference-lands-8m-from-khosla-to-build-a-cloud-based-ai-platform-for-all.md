# Scaled Inference 从 Khosla 获得 800 万美元，用于为所有人构建基于云的人工智能平台 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/11/14/scaled-inference-lands-8m-from-khosla-to-build-a-cloud-based-ai-platform-for-all/>

由两名前谷歌员工创建的初创公司，正在为希望使用人工智能和机器学习工具来运行其应用和服务的第三方建立一个基于云的平台。该公司已经筹集了另一轮资金，以继续其开发和招聘，Khosla Ventures 提供了 800 万美元的 A 轮融资。

这使得融资总额达到 1360 万美元，今年早些时候，初创公司[从腾讯和 Felicis 等一些天使和知名投资者那里筹集了种子轮](https://web.archive.org/web/20221206002028/https://beta.techcrunch.com/2014/07/29/scaled-inference-2/)。

虽然 Scaled Inference 尚未推出公共产品，但它预计很快就会有一个工作原型，现在正在召集公司对该平台进行封闭测试。

在筹集种子资金和首轮融资之间的几个月里，Scaled Inference 一直在整合一些将在该平台上运行的首批产品(或服务):模式识别、异常检测、预测和预测排名，开发者可以通过一组 API 访问这些产品。

规模化推理的挑战在于，它不仅要为机器学习智能构建云平台，还要试图找出如何将迄今为止非常具体的应用程序提炼为通用服务。联合创始人 Olcan Sercinoglu(他与 Dmitry Lepikhin 一起开创了这项业务)说，第一套将“作为各种特殊用途服务的基础，我们也计划在未来提供这些服务。”

“这些核心服务都依赖于用户提供的数据集或数据流，这些数据集或数据流在很大程度上定义了潜在的应用，”他解释道。

目前，许多服务都集中在电子商务和相关的金融服务上。

他说，以维萨这样的银行或信用卡公司为例，可以说明模式识别的应用领域。Visa 将为每笔交易提供交易数据，如时间、地点、商家、客户、项目和金额。

“然后，模式识别 API 可以用于检测相关性，例如在特定时间段、特定地点、特定客户、特定项目以及这些属性的特定组合方面的支出增加，”他说。反过来，这种类型的洞察力可以帮助指导重要的业务决策，如分支机构的位置/时间、区域营销策略、交易费用的定价等。

与此同时，异常检测 API 可用于检测意外事件，如咖啡店的大额交易，或挑选出在客户通常不在的地理区域进行的交易。当然，这些是大型银行(或 Visa)已经拥有的服务类型，但现在的想法是以这样一种方式创建这些服务，例如，小型金融初创公司可能能够执行类似的反欺诈数据查询。

预测 API 侧重于根据其他属性预测属性。一个例子是在某一特定时间段内将在特定区域花费的金额。“这种类型的精确预测分析可用于优化重要的前瞻性商业决策，或通过在用户明确要求之前预测他们想要或需要什么来实现智能产品或服务，从而节省用户的时间，”他说。

预测排名 API 与此相关，将让用户根据预测的属性对大量对象进行排名。“例如，它可以用来获得客户在特定时间/地点最有可能购买的商品的排名，或者给定商品或商家的潜在客户的排名，”Sercinoglu 说。除了购物应用之外，这些应用还可以用于分类库存的广告系统。

将这些服务应用于完全在公共云中的电子商务公司的规模推断存在潜在的障碍:这些类型的应用程序基于敏感的个人数据，如财务或健康数据，因此由于政府法规，可能需要私有或内部部署其技术。

Sercinoglu 说，在这方面，这是一个长期愿景，但这是 Scaled Inference 与投资者合作的原因之一。“维诺德·科斯拉是一个长期投资者，是这个行业中目光最长远的思想家之一，我们对人工智能和机器学习有着长期的愿景。这真的引起了他的共鸣。”

他说，从长远来看，“我们寻求为每个人和几乎任何应用程序提供与公共云 API 相同的强大技术；任何东西，从个性化杂志到智能上下文敏感地址簿、应用程序启动器、电子商务网站、颠覆性数据驱动的初创公司(优步、Airbnb、出租跑道等)、广告系统、搜索引擎等等。最终，我们的目标是为世界上最智能、最受欢迎的产品和服务提供动力。”
