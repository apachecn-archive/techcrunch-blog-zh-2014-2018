# 合作竞争的精神在云计算中依然存在

> 原文：<https://web.archive.org/web/http://techcrunch.com/2014/10/17/the-spirit-of-coopetition-is-alive-and-well-in-the-cloud/>

越来越多的与其他公司直接竞争的公司已经宣布了主要的合作伙伴关系，本周我们看到了微软和 Salesforce 以及 IBM 和 SAP 的一些很好的例子。在我看来，它们都与云有关，这并非巧合。这是因为公司可能想要竞争，但客户希望他们的技术供应商只是和睦相处。他们需要产品能够很好地协同工作，而在云中这样做似乎更容易。

我们也看到[整个创业生态系统建立在开源之上](https://web.archive.org/web/20230326225728/https://techcrunch.com/2014/06/27/open-platforms-fuel-startup-ecosystems/)当这种情况发生时，使用公共平台的公司不能让他们的竞争杀死潜在的金鹅。他们知道他们必须一起工作来保持项目的健康发展——并在使他们的产品与众不同的方面展开竞争。

当通常会相互竞争的公司突然开始合作时，他们喜欢称之为合作竞争。你可以称之为敌友关系，但不管你怎么称呼它，不管出于什么原因，这些公司正以看起来更大的数量聚集在一起，而客户是受益者。但毫无疑问，这些公司也希望获得一些东西，否则他们不会这么做。仅仅因为他们合作并不意味着他们没有赢得它。

为什么我们不能好好相处呢？

认为合作的公司彼此相爱是错误的。他们可能不会。他们仍然希望击败竞争对手，并找到占上风的方法，在竞争公司之间的这些协议中，正如你所期望的那样，每个公司肯定都试图在谈判中获得优势，但也有一种理解，即合作有一定的好处。

Box 首席执行官艾伦·列维(Aaron Levie 月份在 BoxWorks 的一次采访中告诉我，如今公司之间的合作越来越多，原因有很多，但很大程度上是因为现在做起来更容易了。“合作的障碍降低了一个数量级。在以前,(公司)必须明智地选择盟友和敌人。今天，您可以将两个 API 插在一起。摩擦已经不存在了。”

他说，客户要求它，因为它是可能的。

Eventbrite 的首席执行官兼联合创始人凯文·哈特说，我们看到这种合作精神还有另一个原因——因为他说，过去公司试图做所有的事情，但他们不可能都做好。他希望自己的公司专注于自己最擅长的领域，而不是试图提供一系列辅助服务。“最好的创业公司专注于核心竞争力，”他告诉我。他补充说，通过合作伙伴推动增长的关系至关重要，他已经与 DoubleDutch 和 Survey Monkey 等合作伙伴公司建立了联系，因为他们拥有他的公司所缺乏的专业知识。这意味着像 Eventbrite 这样的初创公司不需要浪费时间开发这些类型的功能，因为其他公司已经做得很好了。

竞争之火仍在燃烧

仅仅因为有更容易的合作方式，并不意味着竞争已经死亡。远非如此。这些公司，就像运动员一样，仍然是为了竞争和胜利，但是云和开源(以及 API 和标准)都有办法在某种程度上把这些公司聚集在一起，即使他们并不真的想与竞争对手合作。

Couchbase 首席执行官 B ob Wiederhold 表示，这种合作是存在的，因为通常有一个共同的平台，各公司都在这个平台上开发产品，但他表示，这只是稍微推动了竞争，并没有从根本上消除竞争。“有了开源，你的技术栈可能会有合作的部分，但所做的只是将公司的竞争力转移到你与众不同的部分。”他补充说，尽管出于共同利益可能存在一定程度的合作，但市场上仍存在激烈的竞争。

一个很好的例子就是像 [MapR](https://web.archive.org/web/20230326225728/https://www.mapr.com/) 这样的公司，它建立在开源的 Apache Hadoop 项目之上。首席执行官约翰·施罗德不怕承认他喜欢竞争和胜利，但在 Hadoop 的基础上也需要一定程度的合作。“我们的比赛肯定是激烈的。我们喜欢竞争。这是我这么做的原因之一。竞争意味着你在追求一个巨大的成功，你已经有了一些有价值的竞争对手，这意味着这是一个巨大的市场机会。”

也就是说，他说竞争对手明白 Hadoop 项目是一个他们必须共同努力保护和保存的项目，因为它是该领域所有业务的基础。“我们都对 Hadoop 的采用感兴趣，所以我们尽量不让竞争损害采用曲线，”他说。这意味着他们倾向于围绕影响底层平台的问题(如安全性)进行合作，因为这符合每个人的最佳利益。

AgilOne 的首席执行官兼创始人 Omer Artun 说，虽然他看到了合作的好处，但他仍然讨厌他的竞争对手，并希望以最糟糕的方式击败他们。但他仍然承认，社区中初创企业创始人之间的合作精神让他感到惊讶。“我可以拿起电话，他们会帮助我，在这里(硅谷)比其他任何地方都多，”他说。然而，他明确表示，当涉及到竞争对手公司时，合作就停止了。他并不想让他们的生活更轻松。

不断壮大的微软和 SALESFORCE 联盟

然而，尽管渴望竞争和胜利，我们看到像[微软和 Salesforce 这样在同一领域直接竞争的公司走到一起](https://web.archive.org/web/20230326225728/https://techcrunch.com/2014/10/13/microsoft-and-salesforce-promise-windows-office-and-onedrive-integration/)达成协议，因为这符合他们的最佳利益和他们客户的最佳利益。事实上，马克·贝尼奥夫(Marc Benioff)本周在 Dreamforce 的舞台上与负责 Windows 营销的公司副总裁托尼·预言家(Tony Prophet)进行了一次亲密的炉边聊天——那是炉边聊天，所以肯定是温暖而模糊的，对吗？

让我们不要忘记，这两家公司[曾经在法庭上针锋相对，互相起诉对方侵犯专利。微软仍然有一个名为 Dynamics 的 CRM 产品与 Salesforce CRM 直接竞争，但他们打算扩展他们日益增长的联盟，因为这对双方都有意义。](https://web.archive.org/web/20230326225728/https://techcrunch.com/2010/06/25/salesforce-countersues-microsoft-patent-boies/)

这是因为首席执行官塞特亚·纳德拉认为，成为一个值得信赖的云供应商比在 CRM 业务中击败 Salesforce 更重要，因为 CRM 业务只是微软业务的一小部分。对他来说，这是一个明智的战略决策(尽管这显然搞乱了他的 Dynamics 合作伙伴网络)，这让他可以通过 Office 365、OneDrive 和 Outlook.com 接触到一批 Salesforce 客户，他们获得了云，并且不害怕云。它解决了纳德拉与自己的客户群之间的一个问题。他们向云迁移的速度较慢，而他有可能通过这种方式更快地获得大量云客户。

对于贝尼奥夫来说，他可以接触到所有那些微软客户，以及那些可能使用过微软 Dynamics CRM 的公司，现在可能会愿意使用 Salesforce，因为 Office 365 将非常好地集成到生态系统中，包括即将推出的 sales force Outlook 应用程序(许多销售人员喜欢在这里工作)。如果 Salesforce 更顺利地集成到 Microsoft 体验中，它对 Salesforce 有利，因为 Office 系列在企业中仍然非常受欢迎。

IBM 与 SAP 结盟

在另一个经常争夺相同客户的公司案例中，SAP 和 IBM 本周宣布了云合作伙伴关系。这不像微软和 Salesforce 之间的合作那么奇怪，因为这些人已经以各种形式合作了几十年，但他们承认尽管工作关系良好，你也可以称他们为敌友。至少他们从未起诉过对方。SAP 将使用 IBM Softlayer 的 40 个数据中心网络来运行 SAP HANA 的云版本，并在 SAP 自己运行的数据中心的基础上大幅增加。

SAP 大幅扩大了其全球数据中心网络的规模，只需开一张支票，无需自己构建数据中心的麻烦和成本。此外，SAP 的许多客户都有数据主权问题，这意味着数据在某些情况下必须留在国内。拥有遍布世界各地的数据中心网络增加了附近有一个数据中心的可能性，这可能意味着成功与失败的区别。

自去年收购 SoftLayer 以来，IBM 已投资超过 10 亿美元 1B，以将其数据中心数量从 13 个增加到 35 个，到今年年底将有 5 个数据中心上线(总数达到 40 个)。拥有像 SAP 这样的合作伙伴给 IBM 带来了一个巨大的客户，这将有助于开始偿还在新数据中心的一些投资。对于其他大客户来说，这也给了他们一个卖点，即像 SAP 这样的全球性大公司正在使用其基础设施服务。又是一个双赢。

看看当公司表现良好时会发生什么？公司赢了，客户也赢了，最终这就是他们这么做的原因。客户有强烈的需求，开源、云 API 和服务让这些竞争对手更容易合作。他们可能永远不会相爱，但他们明白为了企业的利益，这样做是值得的——这就是为什么你会继续看到公司以这种方式合作。
