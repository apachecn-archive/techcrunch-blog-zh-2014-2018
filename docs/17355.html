如果你现在正在运营一家软件公司，那么几乎可以肯定的是，你的大部分或所有应用程序都将在云中运行。可能是亚马逊或者谷歌的。很难想象情况并非总是如此，但仍有一些后期采用者将他们自己的物理数据中心迁移到托管数据中心。而且，就像所有的技术趋势一样，这也将会过去。就在您对容器和自动伸缩感到舒适的时候，一种新的架构出现了，将钟摆摆回到真正的分布式世界。

云怎么了？

一辆典型的自动驾驶汽车每秒钟从摄像头、激光雷达、加速度计和车载计算机的组合中产生高达 100MB 的数据。这些数据需要被即时处理以保证汽车在路上行驶。有这么多数据要筛选，当前一代的蜂窝网络跟不上。当数据到达云端时，一切都太晚了。相反，数据需要在尽可能靠近传感器的地方处理，直接在网络的边缘，在汽车上处理。

我们大多数人还没有制造或驾驶自动驾驶汽车，但我们很有可能已经每天都在与边缘计算互动。几乎4000 万美国家庭的智能扬声器中的神经网络正在监听像“Alexa”、“Siri”或“Google”这样的词，根据 Statista 的数据，每天有 30 亿次 Snapchats 被扫描人脸，以便添加令人上瘾的人脸过滤器。到今年年底，全球 20%的智能手机将具备硬件加速的机器学习能力。

我们是怎么到这里的？

所有这些应用和设备都是由两大趋势促成的:帮助计算机看、听和理解的深度学习算法的进步，以及 GPU 和 TPU 等专门处理器的激增，这些处理器可以有效地运行这些算法，甚至在移动环境中。

神经网络和深度学习并不新鲜。事实上，第一个人工神经网络是在 20 世纪 50 年代创建的，从那以后有过多次失败。然而，这一次，大量的标记训练数据和计算能力使得训练这些大型模型变得可行。虽然人工智能研究仍在以极快的速度进行，但像计算机视觉这样的领域已经开始成熟。开发人员可以从各种标准化的模型架构、公开可用的训练数据集和工具中进行选择。你不再需要一个博士学位就可以开始了。技术正在被大众化。

工具和硬件发展如此之快，很难跟上。

硬件正在快速赶上。像神经网络这样的机器学习算法实际上只是矩阵乘法的长序列。GPU 等专用处理器和更新的神经处理单元，如苹果 A11 仿生芯片和谷歌张量处理单元(TPU)中的处理器，正是针对这些数学运算进行了优化，比传统 CPU 提供了 10-100 倍的加速，同时整体功耗更低。随着主要芯片制造商推出移动就绪的机器学习加速器，每台设备都将很快拥有运行最新人工智能模型的能力。

优势=新机遇

过去十年来，大数据、数据科学、机器学习以及现在的深度学习一直在慢慢地融入产品和公司。大多数时候，这发生在幕后，在云上。数据仓库和分析管道一起处理记录。最终用户可以通过 API 和数据库查询访问结果。这不会消失，但边缘提供了一个新的机会，可以更快地使用机器学习模型的预测能力。

现在，算法转向数据。一旦传感器捕捉到信息，就会对其进行实时处理，并立即获得结果。在这个无延迟的世界里，全新的用户体验是可能的。你的手机屏幕成为了增强现实世界的入口。产品可以针对单个用户进行个性化，而私人数据不会离开设备。应用程序变得无环境、无摩擦，在你提问之前预测问题并回答它们。

不需要博士学位

如果做得好，人工智能和边缘计算带来的体验就像魔术一样，但构建它们非常复杂。用于在云中训练和部署机器学习模型的技术堆栈与用于为智能手机和物联网等边缘设备构建应用程序的技术堆栈之间存在分歧。神经网络可以取代数千行程序代码，但会以意想不到的无声方式失败，需要进行不同的测试。性能问题可以通过简单地从近乎无限的云中添加更多计算或内存来解决，但当这些问题出现在我们无法控制的边缘设备上时，就需要进行专门的优化。即使是云首选的编程语言也不同于那些在移动设备上运行应用程序的语言。

这种情况正在开始改变。工具和硬件发展如此之快，很难跟上。苹果和谷歌等重量级公司已经将移动机器学习框架(分别为 Core ML 和 TensorFlow Lite)作为其最新开发者产品的核心。AWS 的 SageMaker、Azure 的 ML Studio 和 IBM 的 Watson Studio weekly 等工具正在增加更多的导出选项和更好的互操作性。

是时候开始思考如何利用机器学习和边缘计算来改进应用了。它不再需要人工智能博士学位或多年的经验来开始——如果你不迅速行动，你就有被落在后面的风险。
