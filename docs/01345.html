PayPal 通过新的 PayPal Here SDK TechCrunch 进一步扩展到线下零售和移动支付领域

原文：https://web.archive.org/web/http://techcrunch.com/2014/09/04/paypal-expands-further-into-offline-retail-and-mobile-payments-with-new-paypal-here-sdk/?ncid=rss&mc_cid=b8fbcc3005&mc_eid=84947498a2

PayPal 今天上午推出了一款新产品，旨在帮助其业务进一步进入线下零售。该公司正在推出 PayPal Here SDK(软件开发工具包)，这将帮助在线商家离线销售他们的产品，同时也为实体商家提供了一个进入移动商务世界的新入口。

PayPal 这里的，你可能还记得，是几年前推出的，作为 Square 等其他移动支付解决方案的竞争对手。这项服务包括一个三角形的加密狗，作为插入商家智能手机的信用卡读卡器，允许他们随时接受支付。

然而，随着 PayPal Here SDK 的推出，PayPal 正在采取基于合作伙伴的方式在离线世界中站稳脚跟，而不是依赖商家采用 PayPal 加密狗解决方案本身。该 SDK 一经推出，就被已经为 PayPal 所追逐的商家提供服务的合作伙伴所采用——go daddy 和 Ecwid，不久还会有更多。SDK 支持与“行业规模”的第三方信用卡读卡器(如 MagTek 的读卡器)集成，这意味着商家可以继续使用他们已经拥有的硬件。

贝宝表示，通过帮助商家建立在线店面的 Ecwid，商家将能够将其店内销售与其在线销售数据和库存联系起来。

除了支付之外，Ecwid 与 PayPal 的集成还利用了 PayPal 在实体店的“签到”功能，因此使用 PayPal 应用程序的消费者可以签到，以通知正在移动或在实体店销售的 Ecwid 零售商，然后使用他们的 PayPal 移动钱包支付。该公司解释说，零售商通过他们的 Ecwid 商家应用程序收到了顾客“在商店”的警报。

与此同时，GoDaddy 的商业应用 SVP Steven Aldrich 表示，PayPal Here SKD 允许 GoDaddy 将 SDK 提供的支付处理和轻松加入集成到自己的 Get Paid 应用程序中，该应用程序允许小企业管理客户工作流，包括提案和评估、发票和收据、持续提醒和感谢，并跟踪计费时间和费用。

奥尔德里奇说:“它允许企业主亲自和在线获得付款，并在一个地方跟踪他们所有的应收款。”“此外，由于我们的客户使用 Go Daddy 来创建他们的数字身份，他们可以出示印有他们自己的 URL 和徽标的发票。”

这种基于合作伙伴的方法是 PayPal Here 未来战略的关键，与其他合作伙伴的讨论已经在进行中。这些未来的合作伙伴可能会为他们的商家提供不同的功能集，例如涉及预订系统或时钟管理的功能集。

“人们知道 PayPal 是我们的在线业务，但如果你想一想我们是如何在在线渠道中增长如此之快的，我们确实是通过合作伙伴驱动的战略做到的。我们的重点是为商家和消费者提供更好的支付方式，我们已经在我们的在线业务中做到了这一点，”PayPal 零售副总裁兼总经理 Brad Brodigan 解释道。“PayPal SDK 让我们感到兴奋，因为它是我们将支付平台转移到买家和卖家希望见面的地方的战略的延伸，”他补充道。

与合作伙伴合作的策略对 PayPal 来说是有意义的，因为这一直是它做生意最舒适的方式，而且这也是一种与竞争对手 Square 截然不同的方法，后者最初采取了更加以消费者为先的道路(尽管现在这种情况正在发生变化，因为公司越来越重视中小企业及其需求。)

PayPal Here SDK 今天向其他感兴趣的合作伙伴免费开放。在 DL-PayPal-Here-SDK@ebay.com，企业被要求通过电子邮件注册。
