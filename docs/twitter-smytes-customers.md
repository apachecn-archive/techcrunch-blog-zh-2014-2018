# Twitter 的“百万”客户 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/06/21/twitter-smytes-customers/>

Twitter 今天[宣布收购](https://web.archive.org/web/20230117112402/https://techcrunch.com/2018/06/21/twitter-acquires-anti-abuse-technology-provider-smyte/)“信任和安全即服务”初创公司 [Smyte](https://web.archive.org/web/20230117112402/https://www.smyte.com/solutions) ，以帮助其更好地解决与其平台上的在线滥用、骚扰、垃圾邮件和安全相关的问题。但它也决定在没有警告的情况下立即关闭对 Smyte API 的访问，让 Smyte 的现有客户没有时间过渡到新的服务提供商。

这一变化让 Smyte 目前的客户群陷入困境，生产问题与他们自己平台的安全性有关。

不用说，许多人对这种情况不满意，并在 Twitter 上登记他们的投诉。

根据 Smyte 的网站，其客户包括 Indiegogo、GoFundMe、npm、Musical.ly、TaskRabbit、Meetup、OLX、ThredUp、YouNow、99 Designs、Carousell 和 Zendesk，这些知名品牌以各种方式使用 Smyte 的功能集来打击欺诈、滥用、骚扰、诈骗、垃圾邮件和其他安全问题。

虽然 Twitter 早些时候告诉 TechCrunch，它将“逐步减少”Smyte 与现有客户的业务，但这显然意味着它将宣布收购，然后实际上关闭 Smyte 的灯光，让所有人陷入困境。

根据那些受影响的人的报告，Smyte 禁止访问其 API，几乎没有警告客户，也没有给他们时间准备。客户接到一个电话，然后——嘣——服务就没了。在某些情况下，客户有多年合同。

再次重申，Smyte 是反滥用和反欺诈保护的提供商，不是任何企业可以在一夜之间关闭的。

在 npm 的案例中，它甚至导致了生产中断。

Twitter 拒绝置评，但我们知道它今天正在给受影响的 Smyte 客户打电话，为他们匹配新的服务提供商。
