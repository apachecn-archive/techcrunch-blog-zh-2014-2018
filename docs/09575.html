Stripe 推出雷达，通过机器学习解决电子商务欺诈技术危机

原文：https://web.archive.org/web/https://techcrunch.com/2016/10/19/stripe-launches-radar-to-tackle-e-commerce-fraud-with-machine-learning/

Stripe 是一家让网站和移动应用通过其应用编程接口和几行代码实现支付服务的初创公司，随着它继续用更多工具构建平台，它今天又增加了另一项新功能。现在，它将通过一项名为雷达的新服务，帮助阻止 Stripe 交易中的欺诈行为。

雷达正在全球范围内推广，作为 Stripe 主要支付服务的一部分，这意味着使用 Stripe API 进行支付的公司不需要支付额外费用或做任何特别的事情来打开它。

如果 Stripe(现在已经为 T4 筹集了约 3 亿美元，价值约 50 亿美元)开始增加更多功能并决定单独将该服务货币化，这种情况可能会改变。(其目前的基本支付费率是，在美国，每张成功卡收取 2.9%的佣金外加 30 美分，其他市场的费用各不相同。)

“这是一个积极发展的领域，有一长串我们想做的事情，”Stripe 的联合创始人兼总裁约翰·科利森在接受采访时说。“我们还没有排除(将它作为一项独立服务推出)的可能性，但希望看到人们如何使用它，以及哪些先用，哪些先不用。”

我已经在一些领域看到 Stripe 可能会在欺诈检测和预防之外添加更多功能。例如，目前 Stripe 不提供任何种类的保险或保护服务以及欺诈预防，尽管科利森告诉我这是它考虑过的一个领域。

在雷达中，Stripe 正在解决在线商务世界中的一个非常大的问题。根据 eMarketer 的估算，仅零售电子商务(不包括可能通过 Stripe 系统进行的其他类型的交易)今年在全球就将价值近 2 万亿美元，在未来两年内，它将继续以 20%以上的速度增长，并在所有商业(包括线下)交易中占据越来越大的比例。

但随着在线商务的持续增长，电子商务欺诈事件也在增加。Pymts 和 Forter 进行的一项调查发现，在 2015 年第四季度，每 1000 笔交易中有 27 起欺诈企图，在 2016 年 Q1，每 100 美元的交易中约有 4.79 美元存在风险(一年前为 2.90 美元)。

科利森不愿透露这些年来有多少欺诈通过 Stripe 自己的渠道进行，但指出了电子商务中存在大量欺诈的事实。

Stripe 在解决这个问题上处于一个有趣的位置。该公司表示，目前它与数十万家企业合作——它从未精确统计过具体有多少——并且由于这些客户的规模和范围(他们包括GoFundMe、Lyft 和 Best Buy 等)，它估计它已经通过数十万笔交易见过全球五分之四的所有信用卡，并且它平均见过每张卡六次。

这与它神秘地称之为来自其金融合作伙伴的“信号”(它与它活跃的国家的许多主要信贷提供商合作)相结合，本质上给了该公司一个关于使用和用户的行为数据宝库，它在后端通过其机器学习算法运行这些数据，以便能够检测购买模式中的不一致。

Stripe with Radar 面临的一个挑战是，市场上已经有许多其他防欺诈服务，包括 Trustev、Kount、Forter、Signifyd 等等。Stripe 的一些客户已经在使用它们了。

Stripe 认为，它的机器学习和对自己平台上交易数据的密切监控，使它在许多第三方产品中具有优势。科利森指出，一个方面是 Stripe 如何跟踪 IP 地址，他声称这个细节并不总是评估过程的一部分。

对于企业，它提供了标记费用的优先列表，无需任何代码或工程工作即可预览和设置自定义规则的能力，以及“传统欺诈检查”，如卡验证码和地址验证。

Stripe 企业可能对 Radar 感兴趣的另一个领域与它的构建方式有关:前提是 Radar 的工作方式就像 Stripe 的其他服务一样，是一个简单的 API，不需要大量集成或单独的欺诈或数据科学团队来监控和运行它。不过，一家公司也可以选择定制它，这样 Radar 确实会阻止一些销售，但也会标记其他看起来可疑的交易，以便人们在阻止之前进行评估。

“在国际象棋中，最好的棋手不是计算机也不是人，而是与计算机一起工作的人，”科利森打了个比方。“有点像那么回事。我们拥有所有的机器智能，构建有助于您运营业务的逻辑。”

有趣的是，他告诉我这一点和简单性规则是 Stripe 下一步计划解决的模板的一部分。科利森没有透露该产品将会是什么，只是指出在不久的将来还会有更多的产品发布。

他说，公司的使命是“消除业务增长的复杂性”..我们将把这视为 Stripe 未来产品供应的主要部分。这是我们看待世界的方式。是什么让企业日子不好过，为什么它们没有更快地发展？我们该如何改变这种状况？”
