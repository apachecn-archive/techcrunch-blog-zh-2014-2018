英国政府对 Twitter 的监控 API 镇压感到愤怒

原文：https://web.archive.org/web/https://techcrunch.com/2017/04/26/uk-government-irate-at-twitters-surveillance-api-crackdown/?ncid=rss

英国政府对推特的监控 API 镇压感到愤怒

据《每日电讯报》报道，英国政府批评推特限制警察和情报机构获取数据，称这些数据可能被用来识别恐怖分子的阴谋。

该报援引首相特里萨·梅的发言人的话说:“我们抗议这一决定。我们正在与 Twitter 就获取这些数据进行谈判。”这位发言人还呼吁社交媒体公司在政府打击恐怖主义的斗争中发挥作用。

TechCrunch 明白，有争议的数据与 Twitter 出售给私营部门客户和广告公司的数据相同。

《每日电讯报》报道称，政府已经向一家第三方公司支付了一项服务，该服务跟踪与潜在恐怖袭击相关的术语——在此之前，Twitter 切断了对其 API 的访问。

在12 月，在美国公民自由联盟的压力下，Twitter 类似地阻止美国执法机构通过与 Dataminr 的协议获取实时数据。《每日电讯报》暗示，在访问被切断之前，英国内政部也一直在与 Dataminr 合作。(让事情更复杂的是，Twitter 部分拥有 Dataminr。)

Twitter 的一位发言人拒绝置评——但指出了 11 月份的一篇博文,其中讨论了对其 API 被用于创建产品以“跟踪或描述抗议者和活动人士”的担忧，包括执法机构。

推特当时写道:

我们禁止使用公共 API 和 Gnip 数据产品的开发者允许执法部门或任何其他实体出于监视目的使用 Twitter 数据。句号。我们的公共 API 和 Gnip 数据产品提供人们选择公开分享的信息，这一事实不会改变我们在这一领域的政策。如果开发者违反我们的政策，我们将采取适当的行动，这可能包括暂停和终止访问 Twitter 的公共 API 和数据产品。

当新开发人员加入时，我们有一个内部流程来审查 Gnip 数据产品的用例，并且在适当的情况下，我们可能会拒绝全部或部分请求的用例。在接下来的几个月里，你会看到我们采取更多的执法和合规措施，包括增加更多的资源来快速调查和处理关于滥用 Twitter 公共 API 和 Gnip 数据产品的投诉。

在此之前，去年 10 月，Twitter 暂停了 Geofeedia 的数据访问，这是一个基于位置的社交媒体监控系统，被政府办公室、私人安全公司、营销人员和其他人使用。

TechCrunch 了解到，英国政府和 Twitter 正在进行讨论，试图解决这个问题。据知情人士透露，要求提供的数据将被匿名化。

社交媒体公司在英国受到越来越大的政治压力，要求他们做更多的工作来帮助打击恐怖主义。内政大臣 Amber Rudd 在三月与 Twitter 和其他主要的技术平台举行会谈,推动他们开发工具来解决恐怖主义宣传在其平台上的传播。
