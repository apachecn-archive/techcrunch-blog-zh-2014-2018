# SAP 以 3.5 亿美元收购客户身份管理公司 Gigya 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/09/24/sap-is-buying-identity-management-firm-gigya-for-350m/>

德国企业软件巨头 SAP 今天宣布了一项收购，以加强其 [hybris](https://web.archive.org/web/20230326023436/http://hybris.com/) 电子商务部门。T4 收购了 Gigya T5，这是一家帮助在线房产管理客户身份和资料的公司。交易条款尚未正式披露，但我们的消息来源告诉我们，这是 3.5 亿美元。

这与昨天报道的数字相同，当时新闻[作为谣言在以色列媒体](https://web.archive.org/web/20230326023436/https://www.themarker.com/technation/1.4459713)中泄露(Gigya 总部位于加州山景城，但其研发总部位于以色列，其创始人是以色列人)。分析公司 [Zirra](https://web.archive.org/web/20230326023436/http://zirra.com/) 还告诉 TechCrunch，该公司最近一次估值约为 2.5 亿美元，当时该公司[在 2014 年筹集了由英特尔投资领导的 3500 万美元](https://web.archive.org/web/20230326023436/https://techcrunch.com/2014/11/04/gigya-raises-from-intel-capital/)。

Gigya 目前管理着数百个站点的约 13 亿个客户身份，现有业务将纳入 SAP 旗下。这个想法是将这些功能整合到 SAP 更广泛的电子商务运营中，以扩大其向现有客户提供的服务种类，并帮助向 Gigya 的基础销售更多的电子商务服务。

SAP Hybris 总裁兼联合创始人 Carsten Thoma 在一份声明中表示:“Gigya 带来了丰富的技能和专业知识，将显著增强 SAP Hybris Profile 解决方案，使我们能够在新兴的客户身份和访问管理市场中占据领先地位。“消费者信任是客户驱动型组织取得成功的主要因素。这就是 Gigya 为人所知和认可的地方。”

Gigya 最初是一个“社交”登录平台，它帮助在线酒店管理客户资料，这些资料与他们在脸书等网站上的资料相关联。但几年前，该公司开始拓宽其对身份管理目的的看法，以及如何将其用于电子商务和其他网站的更广泛的客户管理功能。

2014 年，帕特里克·塞尔(Patrick Salyer)告诉 TechCrunch，“我们实际上认为正在发生相当大的转变。“我们正处于第三方身份与社交登录有关的阶段，但我们看到身份的定义正在发生变化……包括支付、安全，甚至利用硬件。在这个新世界中，谁是重要的玩家？除了脸书，还有亚马逊和贝宝。”

这一转变是一个明智之举:由于一系列高调的违规事件、将更多服务转移到云的趋势以及恶意黑客越来越老练，网络安全已经成为一个问题。

Gigya 本身也未能幸免于其中一些问题:早在 2014 年，该公司的平台就被叙利亚电子军用作黑客网站的后门(该漏洞随后被修复)。

随着违规事件的激增，我们看到了对更好、更严密的身份管理的关注。电子商务公司——就像它们在交易和消费者信任方面所做的那样——注定会在所有在线垂直行业中损失最大。

结合这一点，他们只是试图创建更好、更智能的身份管理系统，使他们的用户体验更加友好。

Gigya 当前产品的另一个重要细节是其产品(它以“注册即服务”的形式销售)提供本地化功能，这在电子商务中非常重要，因为不同地区在数据存储位置和数据保护领域都有特定的合规要求和法规。

“将 SAP Hybris Profile 的数据匹配和丰富功能与 Gigya 基于同意的身份数据和访问管理平台相结合，将使我们能够跨渠道识别消费者，并提供一个强大的单一消费者档案，”Salyer 在一份声明中说。

“这是企业数字化的重要一步，因为公司需要能够跨所有渠道无缝地得出准确的结论，包括网络、移动、店内或连接设备以及物联网，并收集关于消费者偏好的数据。我们共同定位于通过数据推动更有效的营销、销售和服务，同时客户保持对共享多少数据的控制。”

Gigya 有 300 名员工，他们都将加入这笔交易，交易预计将在 2017 年第四季度完成。
