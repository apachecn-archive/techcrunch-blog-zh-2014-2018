新的 Cloud Foundry 项目帮助其开发人员在笔记本电脑上运行云应用

为大型基于云的平台编写应用程序通常意味着，作为开发人员，您编写代码，将其推到云中的开发环境中，测试它，然后在迭代过程中从头再来。当然，如果您已经部署的一些代码开始出现问题，您也不能在自己的机器上轻松地排除故障。

随着CF Local,Cloud Foundry项目今天接受了其最新项目，由 Pivotal 的 Stephen Levine 撰写。CF Local 背后的想法非常简单:允许开发人员在他们的笔记本电脑上运行 Cloud Foundry 的本地版本——在 Docker 容器的帮助下——允许他们更快地迭代，而不必将更新推送到云。

CF Local 是现有 Cloud Foundry 命令行工具的插件，充当 Docker 和 Cloud Foundry 之间的转换层。正如 Levine 告诉我的那样，它不会消耗大量的本地资源，并且几乎 100%兼容所有其他 Cloud Foundry 安装(区别在于 Docker 和 Cloud Foundry 如何处理内存限制，尽管这通常不是一个问题)。他还指出，为了确保本地系统完全复制远程设置，CF Local 允许通过下载必要的凭证并将其保存在内存中，通过 SSH 隧道轻松访问第三方服务，如数据库。

正如 Cloud Foundry CTO Chip Childers 所强调的，能够在本地迭代代码也意味着迭代循环明显更快，这使得更容易保持专注。Levine 对此表示赞同，并指出，在很多情况下，您只需进行系统可以监视并在本地推送的迭代更改，而无需重新发布整个应用程序。

值得注意的是，CF Local 并不是一个帮助快速体验 Cloud Foundry 的系统。不过，Childers 暗示，这样的项目实际上会非常有趣，社区可能会对此感兴趣。CF Local 可能是这个项目的核心，因为项目成员希望将它变成现实。
