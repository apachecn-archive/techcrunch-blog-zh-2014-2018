# Accel 向 Semmle 投资 800 万美元以发展其软件开发分析平台 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/09/16/semmle-series-a/>

Semmle 是一家 b2b 初创公司，拥有一个商业分析平台，旨在通过分析开发人员编写的代码质量以及查看企业在特定软件开发项目上的支出等因素来优化其他公司的 IT 项目，该公司已经完成了新一轮融资——筹集了 800 万美元的首轮融资，由 Accel Partners 牵头。

硅谷现有的天使投资者也参与了这轮融资，他们在 2011 年投资了 Semmle 的 200 万美元种子资金。新的资金将用于扩大销售和营销，以扩大其客户群，包括建立一个永久性的美国办事处。它还计划投资于产品开发，计划推出一款云产品，向更广泛的公司开放其平台。

Semmle 目前还没有透露客户数量，但是它的用户包括花旗、瑞士瑞信银行、戴尔、Murex、Nasa 和 Trafigura。总的来说，它的客户已经使用它的平台来管理“数以万计”的开发人员，并解析“数十亿”行代码。

联合创始人 Oege de Moor 表示，Semmle 未来 12 个月的重点将是扩大客户群，更广泛地说，是将业务本身从纯粹的工程实体发展成为更全面的实体，从而更好地反映它所创造的产品。

“我们现在几乎完全是一家工程公司。当我试图筹集资金时，我寻找的不仅仅是钱——关于如何发展业务的建议，”德穆尔告诉 TechCrunch。“因此，Accel 是显而易见的选择，因为他们在开发工具(他们是 Atlassian 的主要支持者)和商业智能方面都有经验。他们是 QlikTech 的支持者。实际上，我们公司介于这两种类型之间。

“一方面，这是非常技术性的…包括代码分析等等。但这也是商业智能，试图让整个组织，包括 C 级高管，了解工程过程中发生的事情。”

de Moor 补充说，Semmle 还将花钱加强其工程团队，以便能够将更多的数据源集成到产品中。

Semmle 的技术分析软件的版本变化，以确定代码的好坏，并标记出可能导致问题的坏代码。

“我们所做的是寻找糟糕或次优编码的模式。每种语言的模式集是不同的。我们有一个知识库，我们存储源代码，我们寻找人们在不同语言中犯的特别常见的错误，”他说。

“我们与 Nasa 等客户合作。在那里，他们联系了我们，提出了一个他们发现的特别糟糕的问题，并说如果你们的分析能在未来任何时候发现这个问题，那就太好了。因此，我们创建了一个新的分析来准确地做到这一点。我们和许多客户都是这样工作的。”

“开箱即用，我们带来了数百个我们寻找的特定问题，但这在很大程度上是一个与我们的客户互动的过程，以不断改进和完善这些标准，”他补充说。

系统可以识别的常见编码问题包括标记为无状态的 JavaBeans 违例，以及 web 应用程序中潜在的跨站点脚本攻击等安全问题。

除了特定的代码分析之外，该产品还引入了其他信号，因此它可以在软件开发时提供更全面的业务有效性分析——查看预算、团队位置和收到的任何 it 票证等方面。

“我们的产品查看版本库，并依次分析每个更改，因此您可以获得非常精细的信息——不仅是当前代码的质量，还有它是如何变成那样的，”de Moor 说。

“因为你分别分析了每一项贡献，所以你实际上知道是谁做出了改变。您可以按个人、团队、地点，甚至是贡献者工作的公司来划分数据。因为我们的许多客户有成千上万的内部软件开发人员，但更多的软件开发人员从世界各地的外包提供商那里为他们的系统做出贡献，在这种情况下，了解谁在提供物有所值的服务尤为重要。”

他补充说，Semmle 的方法使该公司有别于纯代码分析领域的其他公司，如 Coverity 和 Cast。“我认为我们在采取这种整体方法方面是独一无二的…他们纯粹看源代码，他们不看其他数据——例如，当你按外包提供商划分、按位置划分、或关联预算数据时，诸如此类的事情。”

当被问及该产品会扰乱哪些业务角色时，他补充道:“我们帮助组织中各个层面的人。通常，我们的购买者是 C 级主管，他们希望有一个高层次的概述，但是项目经理会得到非常详细的信息来优化他的团队的工作，个人开发人员会在他们的桌面上得到具体的建议，告诉他们应该修复什么，有趣的是，他们通常会做错什么。事实证明，每个软件开发人员都有自己独特的编码风格。

"只要看看我们发现的坏模式，你就能猜到是谁写的代码."
