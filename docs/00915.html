当一个网站无法在你的设备上运行时，手机上的谷歌搜索会发出警告

原文：https://web.archive.org/web/https://techcrunch.com/2014/07/15/google-search-on-mobile-now-warns-when-your-device-wont-support-a-site/?ncid=rss

谷歌今年有望超越桌面搜索引擎，今天它又向前迈出了一步，它将代表用户控制这种体验——无论是好是坏。公司已经宣布当搜索结果中列出的网站可能含有不会在用户设备上显示的元素时，它将会向用户发出标记。如今，不受支持的代码通常会导致缺少元素和空白屏幕。

需要澄清的是，谷歌并没有而不是说它会将这些网站完全排除在搜索结果之外，或者以任何方式禁止你访问它们。但是我怀疑许多用不可支持的代码编写的网站(即使是部分)会因为这些警告而导致流量下降。

鉴于其他一些事情，这对谷歌来说是一个有趣的发展。

首先是审美方面。

在 6 月的 I/O 开发者活动期间，该公司大力宣传其设计语言和精神，最简洁地体现在其“材料设计”指南中，但也渗透到活动之前、期间和之后的其他产品发布中。

谷歌本身并不想成为苹果，但它正在努力呈现一个更加统一和可控的生态系统和体验。更好的体验是这种更加定制的、针对用户的网络搜索完美契合的地方。

这也符合谷歌对移动搜索体验做出的其他改变。例如，6 月份，它还开始标记“错误重定向”——表明搜索链接可能会把你带到一个“智能手机优化”的主页，而不是你想要的页面。

同样，搜索结果中的这些指示可以用截然相反的方式来看待。一些人可能很高兴谷歌减少了浪费的点击时间。但有些人会对谷歌如何引导你感到不快。

有些人认为这只是一个更大问题的部分进展:

为了保持更强大和更统一的体验，谷歌还利用这一公告推广更多使用 HMTL5，它在所有设备上得到普遍支持，响应式设计，以及两个自己的新网站建设工具: Web Fundamentals :“现代最佳实践的精选资源”；和 Web Starter Kit :“支持 Web 基础最佳实践的现成入门框架。”

这也是谷歌用来确保开发者不会试图退出谷歌搜索算法的时刻。

软件工程师 Keita Oda 和谷歌英国网站管理员趋势分析师 Pierre Far 在博客中写道:“确保不要阻止使用 robots.txt 或其他方式抓取任何谷歌机器人的页面资产(CSS、JavaScript 和图像)。”

“能够完全访问这些外部文件有助于我们的算法检测您网站的响应性 web 设计配置，并对其进行适当处理。你可以使用网站管理员工具中的获取和渲染为谷歌功能来测试我们的索引算法如何查看你的网站。”

第二个是更直接的谷歌主要业务，搜索。

就在上周，我们公布了一份从 Yelp 泄露的文件，强调谷歌正在改变搜索结果的呈现方式，转向以谷歌为中心的内容，而不是其竞争对手的内容。

当然，这是一个由来已久的争论，但在移动领域，竞争似乎更加激烈。最近发表的一项研究发现，基于浏览器的搜索(谷歌)的最大竞争对手是特定应用程序(如 Yelp)的崛起。让浏览器体验更加有效，减少垃圾内容，这可能是谷歌在中国留住用户的一种方式。
