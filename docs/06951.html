面向司机的Waze集成在Lyft的移动应用TechCrunch中向全国推广

原文：https://web.archive.org/web/https://techcrunch.com/2016/03/25/waze-integration-for-drivers-rolls-out-nationwide-in-lyfts-mobile-app/

今年早些时候，按需叫车服务 Lyft宣布计划与Waze的软件整合，允许其司机自动接收更新的路线，以更高效、更快速地完成收费。现在，这种集成已经上线——根据 Lyft移动应用程序的更新文本，Waze集成已经到来，并“在全国推广”

这一整合是Waze更大举措的一部分，该举措旨在为按需服务和运输企业提供对其数据的访问，供他们在自己的应用程序中使用，以实现路线选择和指导的目的。通过推出新的“ Waze Transport SDK ”，这些公司能够利用各种Waze功能，包括旅行时间日期、路线信息、ETA计算等等。

该公司当时解释说，合作伙伴可以选择他们想要利用的Waze功能，以便根据自己的需求定制他们的集成。

除了Lyft，Waze的新SDK也正在被其他人使用，包括Genesis Group(英国紧急调度技术公司)、JustPark Parking(英国的停车公司)、Cabify(西班牙、智利、秘鲁、墨西哥和哥伦比亚的拼车服务)、99Taxis(巴西的拼车服务)和Cornershop(墨西哥和智利的送餐服务)。)

然而，就美国市场而言，Lyft是Waze最大的合作伙伴。

据这些公司称，Lyft正在使用Waze软件在Waze中实时更新路线，并让司机更容易在应用程序之间移动；Waze也正在成为司机的默认导航应用。

正如Lyft之前解释的那样，这种整合意味着Lyft将能够在Waze中实时更新司机的路线。这将允许更有效的Lyft线路匹配，因为该应用程序将能够在司机前往下一站的途中添加乘客接站。最终结果是缩短了客户的乘车等待时间，减少了弯路。

此外，当使用Waze导航时，Lyft司机可以按下“返回Lyft”按钮，立即重新启动Lyft应用程序。这两个应用程序之间的无缝切换解决了司机的一个痛点，过去，他们不得不在手机上的应用程序之间摸索，这使得他们更难专注于道路。

虽然Waze受益于让大合作伙伴进入其平台，然后向其提供更多关于交通状况的数据，但Lyft使用Waze软件也可能提高其底线。拥有更高效路线的司机可以更快地完成车费，乘客可以更快地到达目的地，司机可以在轮班期间处理更多的车费。
