# TechCrunch 扰乱黑客马拉松开发者在压力下学会简化

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/09/07/techcrunch-disrupt-hackathon-developers-learn-to-simplify-under-pressure/>

在 TechCrunch Disrupt 黑客马拉松的压力下，一个团队发现自己陷入了困境。产品太复杂，无法按时完成。团队成员感到疲惫和沮丧，准备放弃，直到团队的一名成员，软件工程师亚历山德罗·迪奥戈·布鲁克海默想出了一个简化他们目标的想法，应用程序 *[我应该骑自行车吗？](https://web.archive.org/web/20221006183646/http://challengepost.com/software/should-i-use-my-bike)* 诞生。

该应用程序旨在帮助用户决定他们是否应该骑自行车。团队成员 Alan Rafael Fachini 说，一些团队成员来自巴西，天气可能相当不稳定。他告诉我，早上醒来，阳光明媚，但四个小时后可能会有严重的雷暴和洪水泛滥。如果你有你的自行车，这不是一个愉快的经历。

为了缓解这个问题，该团队连接了地理数据、天气数据和短信，每天早上，该应用程序会为你检查天气，并给你一个简单的信息:要么拿走你的自行车，要么把它留在家里。

产品设计师/前端开发人员 Elle Sakamoto 说，为了让它更有趣，这条信息包括一幅有趣的漫画，以说明你是要把自行车带走还是留在家里。

队员们昨天才见到坂本。他们有一个开发团队，但没有 UX 设计师。他们在黑客马拉松上建立了联系，团队也就形成了。

这款应用的魅力在于它的简单。它做一件事，而且做得很好，这对于一个黑客马拉松来说是一个合理且更容易达到的目标。

据法基尼说，在他们简化之前，他们有一个更广阔的视野。他们想到了一个界面，你可以输入起点和终点，程序会分析一整天整个旅程的天气。事实证明，这太复杂，太难实现。

该团队利用几个 API 工具来制作最终产品。首先，天气信息来自 Weather Underground API。SMS piece call 使用 Twilio API 构建，他们使用 esri 和 ArcGIS 获取地理位置信息。

“我对结果和我们提出的新想法感到高兴，”坂本说。

在这一点上，他们不确定他们是否会完善它并使之成为一个产品，因为他们不确定如何组织接下来的步骤，但在应用程序变得越来越简单的时代-想想应用程序的娱乐化-这个团队提出了一个简单可行的想法。

“我们没有离开，”法奇尼说，“而是决定专注于我们能够交付的东西，我们能够做到这一点。”让我们面对现实吧，许多软件开发人员可以从他们的经历中吸取教训。不要追求整个想法，而是想出一些你可以交付的东西，然后把产品推出去。

团队的最后一名成员是 27 岁的软件工程师伊万德罗·路易斯·杜特拉。
