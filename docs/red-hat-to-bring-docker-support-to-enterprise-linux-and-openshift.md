# Red Hat 将 Docker 支持引入企业 Linux 和 open shift

# Red Hat 为 Enterprise Linux 和 OpenShift 带来 Docker 支持

Red Hat 和 [Docker.io](https://web.archive.org/web/20221007054820/https://www.docker.io/) 今天宣布扩大合作，将 Docker 的容器技术引入 Red Hat 的 Enterprise Linux high-touch[beta program](https://web.archive.org/web/20221007054820/https://access.redhat.com/site/products/Red_Hat_Enterprise_Linux/Get-Beta)及其 [OpenShift](https://web.archive.org/web/20221007054820/https://www.openshift.com/) 平台即服务产品。

Docker 的最初发布是在大约一年前，但在这么短的时间内，它已经迅速成为一个非常受欢迎的工具，供那些希望使用软件容器而不是更传统的虚拟化技术的开发人员使用。Docker.io 是指导 Docker 项目的商业实体。

Red Hat Enterprise Linux 7 目前处于测试阶段，使用 Docker 作为其主要支持的容器格式。Docker 还宣布了一个新的 JumpStart 计划，用于组织创建基于 Red Hat Enterprise Linux 和 OpenShift 的试点计划。该计划将为 Docker Registry 提供培训和安装支持，以及 Docker 商业支持。

“RedHat在开发、投资和培育创新技术方面有着悠久的历史，包括 Linux Containers，这是一种开源方法，多年来，RedHat做出了许多根本性的贡献，”Red Hat 的执行副总裁兼首席技术官 Brian Stevens 在今天 的一份声明中表示。“Docker 技术有助于消除企业采用容器所面临的障碍——易用性、应用程序打包和基础设施集成——这让我们非常兴奋。”

这并不是红帽和 Docker 第一次合作。正如两个组织[去年宣布的](https://web.archive.org/web/20221007054820/http://www.redhat.com/about/news/press-archive/2013/9/red-hat-and-dotcloud-collaborate-on-docker-to-bring-next-generation-linux-container-enhancements-to-openshift)一样，他们也在为 Red Hat 的开源 Fedora 项目 Linux 发行版打包 Docker，以及在文件系统依赖和容器供应方面进行合作。上个月，红帽公司[宣布](https://web.archive.org/web/20221007054820/http://www.redhat.com/about/news/press-archive/2014/3/red-hat-announces-certification-for-containerized-applications-extends-customer-confidence-and-trust-to-the-cloud)扩展其应用程序认证计划，将容器化应用程序纳入其中。该公司当时表示，这确保了使用 Red Hat Enterprise Linux 构建的容器化应用程序可以跨认证的容器主机工作。
