在进入流行科技词汇十多年后，人们对“云”有了一些了解，但大多数人可能只了解一点点。

这是因为云不是一个单一的具体事物，而是一个包含许多技术的概念。理解这些片段以及它们是如何交织在一起的，是完全掌握其背后思想的秘密。

从最基本的角度来说，云就是把你电脑上所有的技术都拿走，然后让别人替你保管。理解这一点的最简单方法是考虑从 Dropbox、谷歌、亚马逊和微软这样的公司租用这些资源(服务器、硬盘、网络设备),这些公司在自己的数据中心维护这些资源，而不是拥有它们。在某种程度上，这类似于租一套公寓，而不是拥有自己的房子。然而，在这种情况下，你租用的是计算机资源，而不是你的屋顶——就像你租的公寓一样，你的房东负责维护。

因此，您不必担心处理在其上运行的所有硬件和软件，只需做好自己的工作，运营自己的企业。

软件为您服务

在智能手机、平板电脑、笔记本电脑和云计算出现之前，我们的桌子下面有一个米色的大盒子(或者说很多盒子放在数据中心里)。我们购买软件并将其安装到硬盘上，并且通常多年来一直使用旧版本，因为升级既昂贵又耗时，还可能导致兼容性问题。云改变了这一切。

照片:罗恩·米勒

软件即服务——或 SaaS——是你在云中访问的软件的正式名称。销售力量。com (于 1999 年 1 月推出)可能是这类软件中最著名的例子，因为它最先出现。如今，你接触的每一款软件都有云版本。纯云公司包括 Zendesk、Box、Slack、DocuSign、AppDynamics(其中被思科收购)，还有这样的公司不胜枚举。对于消费者来说，云服务的例子包括 Gmail(实际上还有 Google Calendar、Drive 等。)、Dropbox、Spotify 和苹果 iCloud(自动将你所有的苹果设备备份到云端。

无论是对消费者还是对企业而言，软件即服务的美妙之处在于，您永远不必手动更新它，因为云供应商正在为您管理所有这一切。云公司会定期(通常是每天)自动为您更新，而不是等待数年的下一次更新。

更重要的是，你不用支付昂贵的单一版本许可证，而是购买订阅——按月或按年付费(或者在某些情况下免费)。卖方为你处理所有的维护和保养。您可以随时随地从任何计算机或设备上使用它，并且总是使用最新版本。

那确实能计算

假设你不只是想租软件，而是想在云端编写和运行自己的程序？事实上，许多企业，尤其是初创企业，都想这么做，但他们需要计算机/CPU 能力、存储和内存——那个旧米色盒子的内脏——来运行该软件。这就是基础设施即服务或 IaaS 的用武之地。它可以让你根据需要使用任意多的计算能力，只需为你使用的计算能力付费，并在你用完后关闭它。

就个人而言，基础设施可以像使用 Dropbox 或 Google Drive 存储文件一样简单。在这种情况下，这些公司就相当于一个在线硬盘。当文件存储在云中时，你可以从任何设备访问它们，而不是回到过去，那时你需要通过电子邮件发送文件或使用 USB 驱动器移动文件。现在，你的文件总是可以从任何电脑、手机或平板电脑上获得，因为存储是在线的，而不是在你的设备上。

也许这一领域最著名的商业供应商和第一个向市场推广面向大众的云基础设施的供应商是 T2 的亚马逊网络服务 T3。今天，谷歌、微软、IBM 和许多其他公司都加入了这个竞争激烈的市场，但 AWS 仍然主导着这个市场，至少现在是这样。

在云出现之前，如果企业需要为黑色星期五这样的使用量激增做好准备，并且知道这将需要比他们拥有的更多的服务器，他们就会受到这些资源的限制。IT 部门不可能为短期需求投资新的服务器，然后将它们装箱并存放起来，直到下一次使用。云可以让你根据需要购买任意数量的计算机资源，并在使用完毕后关闭它们。

这使得公司可以采取更加灵活的方法来满足他们的计算需求(尽管他们确实为这种灵活性付出了一些代价)。这有助于他们应对短期峰值，但也推动了实验和创新，因为基本上没有前期成本。如果每次你想尝试某样东西的时候都必须通过一个正式的采购过程，那很可能就不会发生，或者要花上几个月的时间。IaaS 让即使是最大的企业内部的开发人员也能更自由地进行实验，而不会产生巨大的成本。

更重要的是，将基础设施作为云服务提供给初创公司，使其能够开展新业务，而不必在硬件上进行巨额投资和建设数据中心来实现这一目标。这使得今天的许多创业公司得以起步。

为我搭建一个平台

最后一点，也可能是三个组件中最不为人所知的一点，是平台的概念，或者——等等——平台即服务(PaaS)。

假设你是一名程序员，你想开发一些东西，而不必深究如何管理数据库的细节，或者如何随着需求的增减而增加或减少服务器。如果您使用 AWS 这样的基础设施服务，您不需要负责管理单个服务器，但是您仍然需要设置它们并确保它们能够协同工作。借助平台即服务，您可以利用大量现有组件，如安全性、存储、数据库、网络等。

在这种方法之前，程序员必须手动编写每个部分的程序，这个过程可能需要几周或几个月。平台降低了准入门槛，为编程提供了公平的竞争环境，并为程序员提供了一套广泛的工具，这些工具很难或根本不可能自行开发。

这种类型很难理解，因为有时基础设施和软件供应商也提供平台服务。Salesforce 就是一个很好的例子。它首先是一个 SaaS 供应商，但它也是一个平台。许多独立和成功的公司都建立在 Salesforce 之上，包括 Apttus、FinancialForce、Veeva 和 ServiceMax。他们可以利用 Salesforce 的所有编程知识，并简单地利用他们创建的软件服务。像 Google、Microsoft、IBM 和 AWS 这样的 IaaS 供应商也为他们的客户构建了这些类型的服务。还有像 OpenStack 和 Cloud Foundry 这样的开源产品。

总的来说，重要的是要理解这些工具，无论是建立在软件服务上，独立的，开源的还是基础设施服务的一部分，都是为了帮助程序员更快更有效地构建软件。

推动创业生态系统

如今的创业生态系统、消费者和各类企业都欠了云一大笔债。想想今天最受欢迎的应用和服务，如优步、Waze、Spotify 甚至 Pokémon GO，它们都需要云来运行。

以优步为例。它使用地理定位、地图、短信和支付，所有这些都需要云连接才能正常工作，并允许用户请求搭车，司机找到人，以便双方在需要时进行交流并促进支付。如果没有云，对于一家初创公司来说，建立一个具有这种复杂功能组合的应用程序，并拥有实现它的计算资源，将是极其昂贵的。

云提供了前所未有的访问级别，我们今天认为这是理所当然的，但在它出现之前，这在财务上或逻辑上都是不现实的。今天，通过刷信用卡(甚至在许多情况下是免费的)，我们可以访问几乎任何类型的软件、无尽的计算资源和大量的编程工具——这一切都归功于云技术。
