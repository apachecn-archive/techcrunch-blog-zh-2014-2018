# Skype 增加了由信号协议 TechCrunch 支持的 e2e 加密“私人聊天”

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/01/11/skype-adds-e2e-encrypted-private-chats-powered-by-signal-protocol/>

微软旗下的网络电话和消息平台 Skype 长期以来一直因缺乏端到端加密而受到批评。这意味着 Skype 通信不受零访问架构的保护，例如，该公司可以向当局提供你的解密内容数据。之前[也展示过](https://web.archive.org/web/20230215191435/https://arstechnica.com/information-technology/2013/05/think-your-skype-messages-get-end-to-end-encryption-think-again/)访问通过平台发送的链接。

但它现在部分纠正了这种情况——通过推出一个选择加入的 e2e 加密功能，利用行业标准信号协议，该协议也支持 WhatsApp 的 e2e 加密，并由[自封的非营利项目](https://web.archive.org/web/20230215191435/https://news.ycombinator.com/item?id=7701666)Open Whisper Systems 构建。

微软[宣布](https://web.archive.org/web/20230215191435/https://answers.microsoft.com/en-us/skype/forum/skype_insiderms-skype_insnewsms/skype-insider-preview-private-conversations/01616ac9-2171-4151-b9a2-c77761c0fbf8?auth=1&ranMID=24542&ranEAID=je6NUbpObpQ&ranSiteID=je6NUbpObpQ-EK7eEoD4j5uKKZv9qoHGdA&tduid=(097dcf9a7eecaad2d3546f7276d9e48f)(256380)(2459594)(je6NUbpObpQ-EK7eEoD4j5uKKZv9qoHGdA)())今天推出“私人对话”功能，最初通过 [Skype Insider](https://web.archive.org/web/20230215191435/https://www.skype.com/en/insider/) 程序提供预览。它还没有给出更大范围推广的确切日期。

该功能支持对 Skype 音频通话以及发送消息和媒体(如文件)进行 e2e 加密。

它写道:

> 在私人通话中，您可以使用 Open Whisper Systems 的行业标准信号协议，进行端到端的加密 Skype 音频通话，发送文本消息或图像、音频或视频等文件。这些对话的内容将隐藏在聊天列表和通知中，以保护您共享的信息的隐私。
> 
> 尝试从撰写菜单或收件人的个人资料中选择“新的私人对话”。收件人接受您的邀请后，该对话中的所有通话和消息都将被端到端加密，直到您选择结束该对话。您一次只能从一台设备参与私人对话。您可以将对话切换到您的任何设备，但您发送和接收的信息将与您当时使用的设备相关联。

Signal [在其博客上讨论了这一合作，并补充道](https://web.archive.org/web/20230215191435/https://signal.org/blog/skype-partnership/):“我们的目标是让私人交流变得简单且无处不在。Skype 拥有数亿活跃用户，是世界上最受欢迎的应用程序之一，我们很高兴 Skype 中的私人通话将允许更多用户利用信号协议强大的加密特性进行安全通信。”

那么，一些选择加入的 e2e 加密比没有 e2e 加密好吗？有些人会说“勉强”。尤其是视频聊天 Skype 的旗舰功能——不包括在内。

也可能会有争论，即该功能是否会让 Skype 用户感到困惑，他们可能会错误地认为他们在 Skype 上的所有通信都将被高度加密——而事实上他们每次想使用 e2e 加密时都必须主动选择它。这显然造成了隐私摩擦。也让 Skype 用户认为隐私是一种例外，而不是标准规则。

消息平台 Telegram 还在一个没有 e2e 加密的通信平台上提供了一个 e2e 加密的“[秘密聊天](https://web.archive.org/web/20230215191435/https://core.telegram.org/api/end-to-end)功能，例如，经常因为采取这种局部立场而受到攻击。(尽管它也因使用自己的专有密码而受到批评，而不是采用受尊重的标准。)

另一方面，考虑到 Skype 是一个多么主流的通信平台，那么即使是部分 e2e 加密也可以被视为一种进步，也许最特别的是因为 Skype 正在采用信号协议进行加密。

也有可能——信号是[希望](https://web.archive.org/web/20230215191435/https://twitter.com/signalapp/status/951501788443635712)——微软会及时扩大对 Skype 更多功能的支持。
