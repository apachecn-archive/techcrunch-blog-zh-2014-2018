# 英国新网络安全中心揭穿恐吓战术并以身作则

> 原文：<https://web.archive.org/web/https://techcrunch.com/2016/10/21/uks-new-cyber-security-centre-to-debunk-scare-tactics-and-lead-by-example/>

英国政府已经受够了陈词滥调的网络摄魂怪形象、听起来吓人的行业言论以及普通消费者没有希望遵循的不可能的安全建议。

它希望通过采取一种不那么夸张的、数据驱动的方法来解决网络安全问题，可以鼓励行业效仿，并专注于持续和大量的安全问题，其首要目标是减少规模伤害，提高消费者对数字经济的信任。

英国新的[国家网络安全中心](https://web.archive.org/web/20230316161016/https://www.gov.uk/government/news/new-national-cyber-security-centre-set-to-bring-uk-expertise-together) ( [NCSC](https://web.archive.org/web/20230316161016/https://www.ncsc.gov.uk/) )背后的理念是由该中心的技术总监伊恩·利维博士昨天在伦敦举行的[有线安全](https://web.archive.org/web/20230316161016/http://www.wired.co.uk/event/wired-security-2016)会议上提出的。

据利维称，一份完整的国家网络安全战略即将公布，但他透露了政府在这方面的想法。利维已从英国情报机构 GCHQ 的网络安全技术总监一职调任 NCSC，该机构将于本月正式成立。

**将安全威胁置于环境中**

“我们未来面临的最大威胁是继续像今天这样谈论网络安全，”他认为。“没有哪一项公共政策的叙事是由一群被大量误导的人设定的。”

他说，该中心的核心理念是为“一致、连贯的建议”提供一站式服务，并“公开、透明地”这样做——这显然不是他曾经工作过的情报机构所能提供的自由(在他称之为“[象牙甜甜圈](https://web.archive.org/web/20230316161016/https://upload.wikimedia.org/wikipedia/commons/7/72/GCHQ-aerial.jpg)”中，对他们“象牙塔”中的学者进行即兴表演)。例如，NSCS 将公布其学习数据。虽然它也将向 GCHQ 报告，但显然不是所有的讨论都将对公众开放。

“一个什么都可以去的地方，”利维说，描述了该中心将提供什么。“从一开始，当您需要威胁信息并了解如何设计系统时，通过构建、操作系统，到您遇到问题时，我们如何提供帮助。”

在一次直截了当的演讲中，他对安全行业当前的做法进行了不少讽刺，他抨击了一些语言和媒体对关键和零日安全漏洞的关注，例如 2014 年出现的 heart bleed 加密漏洞，认为这种“末日场景”的报道在公众中造成了混乱和恐慌，并将注意力(和资源)从解决更平凡但持久的安全威胁上转移开来，这些威胁给网络用户带来了持续的问题。

“今天，缓冲区溢出仍然是导致安全漏洞的最普遍的软件缺陷之一，”他说。“四十多年的缓冲区溢出。我们必须开始记住，软件漏洞是不言自明的。你将永远拥有它们，因为软件是由人编写的——问题是你如何管理它们。”

“2015 年五十四零天。让我们把它放在背景中…在同一年[搜索[国家漏洞数据库](https://web.archive.org/web/20230316161016/https://nvd.nist.gov/) ]有 6488 个其他漏洞。我可能应该更关心这些——因为它们太多了，”他补充道。

去年，英国政府宣布其打算将网络安全专业知识聚集在一个屋檐下[——当时它将网络安全列为优先领域，并表示到 2020 年将在该领域的支出增加近一倍，达到 19 亿英镑。](https://web.archive.org/web/20230316161016/https://techcrunch.com/2015/11/18/uk-gov-to-invest-in-security-startups/)

上个月，NCSC 首席执行官 Ciaran Martin 在华盛顿的一次安全会议上发表了他的第一次公开演讲，他还告诉代表们，该中心的目标是让政府在制定积极主动的安全措施方面发挥领导作用，并通过这种推动来鼓励行业提高水平。

“就网络安全而言，市场上有些东西还不太管用，”他认为。“有伟大的公司，伟大的人，有伟大的创新，信息共享的障碍正在被打破。但是考虑到过去几年的记录，很难说我们已经战胜了威胁。

“如果我们要保持对数字经济的信心，我们必须解决这个问题。我相信政府在带头方面有一个合法的角色…至少是暂时的。这是我们战略背后的思考。”

昨日，利维更进一步，将网络安全问题部分表述为“情境感知”问题之一。

“你谈论某件事的方式从根本上改变了你对它的风险评估方式，”他说。“你判断事物的背景也决定了你如何解读它。因此，如果你被告知网络安全攻击据称是由长着翅膀的忍者网络猴子发起的，这些猴子住在外国，只要想想就能危及你的机器，你就会产生恐惧反应。这就是我们今天的处境。”

他的前提是“恐惧反应”会压倒对安全问题更理性的分析，并导致无效或错误的解决方案。因此，政府的希望是重新设定安全叙事，使其更接近现实。

> 安保公司受到激励，让它听起来尽可能可怕，因为他们希望你买他们的神奇护身符。

“安全公司受到激励，让它听起来尽可能可怕，因为他们希望你买他们的神奇护身符，”利维补充说。“这就是我们今天要做的。你买了一个网络安全产品，却把它扔向问题，因为你已经不知道问题到底是什么了。

“如果我们如实谈论事情，我们会有不同的反应。”

解决长期存在的棘手问题

在谈到 NCSC 一直在进行的早期项目之一时，他说，在 gov.uk 域名开通的第一天，欺骗电子邮件被转移到 NCSC(而不是他们的目标受害者)，大约收到了 5 万封电子邮件。

但几天后，在攻击者大概意识到他们的网络钓鱼企图被从根源上切断后，这些电子邮件就永远停止了。

“每一次网络攻击——无论是犯罪、污损还是民族国家，都是基于投资回报的计算。风险计算。但是通过这样做，我们可以改变攻击者的投资回报率，”Levy 说。

在这次试验之后，他说 NCSC 现在打算将 DMARC 放在每个政府域名上——全部大约 5700 个——以终止网络钓鱼者将来从任何 gov.uk 地址欺骗电子邮件的能力。

接下来的一步将是向工业界施加压力——举例来说——重点关注商业部门的其他“高价值”领域，以促使它们遵循政府的领导。

“我要指出并嘲笑每个不这么做的人——公开地，”利维说。“因为再也没有借口不在高价值领域进行 DMARC 了。

“因此，当你收到来自 gov.uk 的电子邮件时，它将来自 gov.uk。当你收到来自(英国零售商)约翰·刘易斯的邮件时，我希望它肯定来自约翰·刘易斯。然后我们也会有接受域。”

此外，由于 NCSC 现在收到了大量的网络钓鱼电子邮件数据，利维指出，它还可以分析攻击者试图将人发送到的域，并计划利用网络钓鱼网站英特尔的缓存来构建“公共部门的大规模递归 DNS 服务器”。

“我将成为公共部门的域名服务提供商。因此，中央政府、地方政府，或许还有医疗、或许还有教育——我们拭目以待，”他继续说道。“同样，首先为政府做这件事，然后去和互联网服务提供商交谈，说‘嘿，伙计们，让你的客户在不知情的情况下受到伤害可能是不合适的。默认情况下，你为英国公民做些类似的事情怎么样

> 默认情况下，我希望互联网服务提供商承担一些责任，不要让他们的客户在不知情的情况下伤害自己。

“默认情况下，我希望互联网服务提供商承担一些责任，不要让他们的客户在不知情的情况下伤害自己。”

根据利维的计划，安全研究人员可以选择退出域名系统过滤，但其想法是，由于互联网服务提供商采取类似的主动行动来阻止对数据窃取网站的访问，普通公众——通常受到网络钓鱼者的伤害最大——将得到更好的保护。

“默认咱们保护的人。因为我奶奶不知道到底发生了什么，我想保护她，”他补充道。

利维表示，他的团队已经取得进展的其他领域是缩短网络钓鱼网站上线的时间窗口——通过 NCSC 联系互联网服务提供商，让他们关闭网络钓鱼网站——用他的话说，是“一种要求托管公司将其删除的非常复杂的网络方法”。

“世界上任何地方假装是英国政府品牌的网络钓鱼时间都从 49 小时减少到了 5 小时，”他补充道。

他还抨击了给网络用户的“愚蠢的建议”——比如告诉他们应该阅读完整的邮件标题，以确定邮件是否真实；或者要求他们记住“相当于每个月 600 个数字”,因为密码要求为他们使用的每项服务创建一个唯一的、多字符、多字符串的密码，并且经常更改它们——这对普通网络用户来说是不合适的。

“我们告诉人们做一些他们不可能做的事情，”他认为，并补充道，该中心将“提供愚蠢的建议”——并旨在改变系统“让它对人们更好，而不是极客”。

“让我们在公共场合做这件事，让我们透明地做这件事，让我们公布数据，让我们公布我们已经做了什么，产生了什么影响，以及成本，”他补充道。“我希望人们能够真正理解网络安全威胁的图景。他们真正的风险是什么，以及如何更好地保护自己。”
