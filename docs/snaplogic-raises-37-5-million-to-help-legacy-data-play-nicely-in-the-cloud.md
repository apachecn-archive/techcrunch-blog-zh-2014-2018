# SnapLogic 筹集了 3750 万美元来帮助遗留数据在云中很好地运行

SnapLogic 是一家帮助将传统应用程序的数据连接到云或集中式内部数据湖的公司，今天宣布了一轮 3750 万美元的融资。

投资者包括微软和银湖的增长资本部门银湖沃特曼，以及现有的投资者安德森霍洛维茨，点火合作伙伴和三角峰合作伙伴。今天的投资总额达到 9630 万美元。

SnapLogic 首席执行官 Gaurav Dhillon 解释说，SnapLogic 本质上是作为数据(或数据流)转移到云或企业内部数据湖的翻译器。“我们有超过 400 个 snap——各种系统的连接器或适配器，如 Workday、Concur、SAP、Twitter、Tableau 和机器协议，”他解释道。

“这使得企业能够更好地将消费者应用程序连接到传统应用程序，并在任何地方处理信息流，”迪隆说。

随着越来越多的客户寻求迁移到云，他们发现很难将数据从遗留系统链接到 Workday 或 ServiceNow 等更新的云服务。SnapLogic 希望帮助客户在无需寻求 IT 帮助的情况下，将数据导入和导出云服务。

他以一个人力资源入职流程为例。这种类型的操作可能涉及四五个内部系统。如何让这些系统相互交流，并对用户隐藏这些交互的复杂性？这就是 SnapLogic 工具的目标。

另一个目标是帮助将数据流引入所谓的数据湖，这是一个集中的地方，公司可以在这里汇集各种数据源。随着时间的推移，SnapLogic 意识到，他们用于连接到云的相同技术也可以在这种情况下工作。

虽然这两个用例明显相关，但它们具有不同的特征，与 Dhillon 所说的“数据质量和数据速度”有关。“有了云连接，数据就没那么多了，但移动速度非常快。大量数据正在移入数据湖，但移动速度没有那么快。”Dhillon 解释说，这是一个连续体，但公司本质上正在解决同样的问题。

值得注意的是，微软不仅是投资者，也是合作伙伴。SnapLogic 最近宣布支持[微软 Cortana 分析套件](https://web.archive.org/web/20230326110034/https://techcrunch.com/2015/07/13/microsoft-unifies-big-data-and-analytics-in-newly-launched-suite/#.v3ewwm:Cg6d)。其他新的合作伙伴包括 Okta 和 Workday。该公司报告称，每月通过其平台处理超过 550 亿份文档。

客户包括 Adobe、Box、阿斯利康、CapitalOne 和威瑞森(TechCrunch 的所有者)。
