# MySQL 创始人尝试新的软件许可模式 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2016/08/19/mysql-founder-tries-a-new-software-licensing-model/?ref=webdesignernews.com>

当谈到开源许可时，开发者有他们公平的选择份额(GPL，BSD，MIT，Apache，等等。)，都各有利弊。商业许可证也是如此。然而，MySQL 创始人[迈克尔“蒙蒂”维德纽斯](https://web.archive.org/web/20221007233751/https://en.wikipedia.org/wiki/Michael_Widenius)和他的联合创始人[大卫阿克马克](https://web.archive.org/web/20221007233751/https://en.wikipedia.org/wiki/David_Axmark)在几年前提出了一个不同的模式:T4 商业来源许可证 (BSL)。

这种新的许可证为许多初创公司为他们的软件选择的闭源和开放核心许可证提供了一种替代方案，Widenius 的新公司 [MariaDB](https://web.archive.org/web/20221007233751/https://www.crunchbase.com/organization/skysql) 现在[将它](https://web.archive.org/web/20221007233751/http://monty-says.blogspot.com/2016/08/applying-business-source-licensing-bsl.html)用于其一款产品。

在某些方面，BSL 类似于软件许可的免费增值模式(有点开源的味道)。正如 Widenius 向我解释的那样，BSL 允许开发人员设置服务器/CPU/等的数量限制。他们的软件可以在生产环境中运行(对于测试环境没有使用限制，只有生产使用限制)。使用量超过该值会产生许可费。

这听起来像是一个非常标准的商业许可证，但是这里的问题是所有的源代码在任何时候都是可用的，并且 BSL 许可证有一个截止日期。经过一段时间(比如三年)后，许可证到期，并恢复为开源许可证，如 GPL 或开发人员选择的任何其他许可证。

“这可以创造一个全新的生态系统，”维德纽斯告诉我。“即使你不能马上获得开源，我们也会在未来创造更多的开源应用。”那些是强烈的言辞；鉴于他在开源世界的经验，我们有必要仔细看看他和 Axmark 是如何以及为什么提出 BSL 的。

> 这可以创造一个全新的生态系统。即使你不能马上获得开源，我们也会在未来创造更多的开源应用。蒙蒂·威德尼斯

Widenius 在许可方面有相当多的经验，在很大程度上，他的财富来自于他为 MySQL 做出的许可选择。“对于像 MySQL 这样的产品，GPL 实际上是完美的许可证，因为 MySQL 是公司想要集成到自己产品中的东西，”他解释道。要将 GPL 许可的产品集成到你自己的产品中，你也必须开源你的软件。Widenius 和 Axmark 创办的 MySQL AB 公司为这些用户提供了商业许可。

当他们在 2008 年将 MySQL AB 出售给 Sun 时，该公司 70%的收入来自许可证。“这就是 MySQL 估值如此之高的原因，”维德纽斯说。“我们是一家产品公司，在某些情况下，人们必须为此付费。”

Widenius 实际上更早就想使用 BSL 的变体，“但那时，管理团队没有我们现在的团队有远见，所以他们决定使用封闭源代码。”然后，几年前，他注意到很多创业公司来到他的[开放海洋](https://web.archive.org/web/20221007233751/http://openoceancapital.com/#team)风险投资公司，他们想为最终用户产品做开源。对于这些公司来说，在 MySQL 上运行良好的双许可模式是行不通的，因为这些用户不会将软件嵌入到他们自己的产品中，因此没有理由为许可付费。

在这种情况下，大多数想做开源的公司都试图围绕这些开源工具提供服务来建立他们的业务。维德纽斯不相信这种模式(尽管他承认这种模式对一些公司有效)。“这对支持一个项目的公司是有效的——人们支持 Ubuntu 并从中赚钱，”他说。“但是没有许可证的公司，他们几乎永远也不能生产产品。”为什么？因为如果你从一个好的支持人员那里得到 10%的利润，你需要 10 个支持人员来支付一个开发者。在他看来，这种模式没有规模。

所以在 MariaDB，团队决定现在在 BSL 下许可其 MaxScale 数据库代理的[最新版本(MariaDB 本身是 MySQL 的一个分支，所以它永远绑定到 MySQL 许可下的开源 GPL 许可)。](https://web.archive.org/web/20221007233751/https://mariadb.com/blog/introducing-maxscale-20-beta-release)

据 Widenius 所知，其他两三家公司也已经使用了该许可证，但他认为还有许多公司在观望，等待一家大公司采取行动。该团队还在致力于为开发人员提供一个将他们的软件迁移到 BSL 的框架的文档。

希望在新项目中采用 BSL 的开发人员只需填写四行:产品名称、限制用户付费时间的限制、许可转换为开源许可的变更数据以及转换为哪个许可。

因为您最终会随着每次更新而更新许可日期，所以开发人员将有动力保持他们的软件是最新的并进行创新。但是如果他们不这样做——或者如果用户乐于使用旧的产品——那么一旦变更数据到达，新的许可就会生效。这也意味着当开发者停业时，软件将在变更日期后成为开源的，社区可以接手工作。

“很多人会以错误的理由批评这一点，”维德纽斯告诉我。“但我认为这是一个通过生产更多开源软件来改善开源未来的机会——即使会有一点点时间延迟。”
