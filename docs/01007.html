桌面生产力应用 TechCrunch 的回归

原文：https://web.archive.org/web/https://techcrunch.com/2014/07/27/the-return-of-the-desktop-productivity-app/

日出。Evernote。Feedly。神童。邮箱。*

这些应用程序都是我个人生产力套件不可或缺的一部分，是我日常使用的工具。如果没有桌面版本，这些移动客户端是有用的，但不是变革性的。但随着多设备接入，它们变得难以替代。

我不是唯一一个看到他用于个人生产力的工具从网络和移动设备的奇怪组合转向越来越多的连接应用的用户，这些应用存在于我的智能手机、平板电脑上，现在也存在于我的桌面上。这改变了开发者对桌面对他们的分销策略有多重要的看法。

网络->移动->桌面

对于许多用户来说，桌面生产力应用程序可能永远不会完全消失。任何使用 Microsoft Exchange 处理电子邮件或日历，或使用 Microsoft Word 作为文字处理器的人都可以告诉你，这些应用程序在许多组织中仍然无处不在。

然而，改变的是用户最初如何找到并连接他们喜爱的应用程序。我们越来越多地看到用户在安装桌面版本之前，先在移动设备上使用应用程序。

有趣的是，这些新的桌面应用程序的兴起在一定程度上得益于将基于云的服务作为个人和企业生产力工具的底层基础设施的趋势。

过去几年中，Google Apps 在企业中的兴起意味着许多用户开始依赖 web 浏览器中的应用程序来与他人交流(Gmail)、创建文档和协作(Google Docs/Drive)以及安排日程(Google Calendar)。但这些服务的开放性也催生了一个应用生态系统，提供了比浏览器更强大、更灵活的功能。

“我认为我们已经回到了一个应用程序的世界，在这个世界里，用户和开发者都更希望应用程序完成任务，而不是通过浏览器标签来完成一切，”格雷洛克的合伙人 T2 在一封电子邮件中写道。“在第一种情况下，现在为 iOS 开发的开发者也更愿意为桌面开发——从 99 年到 2009 年有一段时间，大多数开发者根本不开发客户端应用。”

与此同时，我们看到越来越多的网络应用支持通信、协作和日程安排，我们也看到在移动设备上涌现的生产力应用的数量也在增长。有一段时间，这些应用程序主要是对人们已经在浏览器中使用的工具的补充。

“我不认为桌面生产力工具的使用消失了，但它被利用移动连接和数据优势的移动生产力工具所补充/部分取代，”家酿合伙人萨蒂亚·帕特尔写道。“然而，所有这些移动优先的公司都开始意识到，粘性参与的关键是无处不在，对于大多数知识工作者来说，这意味着连接到桌面。所以旧的都是新的。”

粘性 FTW

移动应用开发者发现，仅仅在智能手机上是不够的。今天，用户通过各种设备和屏幕连接到电子邮件和日历等服务，应用程序开发者越来越希望在所有这些设备上无缝工作。

“通过同时在桌面和移动设备上参与，创业公司获得了更多的关注，变得更容易防守，”Product Hunt 创始人瑞安·胡佛写道。“当人们同时使用移动应用和桌面应用时，要从 Sunrise 切换过来需要更多的努力。这对于那些创造更多易于替换的‘商品化’应用的人来说尤其重要。”

胡佛使用日历应用 Sunrise 作为一个例子。移动电子邮件、日历和待办事项客户端本身很容易被替换，但是在多个屏幕上拥有一致的体验使得它们不太可能被替换。

“当然，当我们谈论移动时代时，我们指的是多屏时代，”邮箱创始人 Gentry Underwood 写道。“我们希望我们的信息和我们使用的工具的体验能够无缝地跨越这些屏幕，不受操作系统的影响。在我们的手机上，设计良好的单一用途应用程序胜出，因为它们最大限度地减少了摩擦，帮助我们在做其他事情的同时快速获得我们需要的东西。”

Mailbox 是一个移动优先应用程序走向多屏的完美例子。它始于 iPhone，但很快也发布了一款 iPad 应用，因为它知道用户也在使用平板电脑提高工作效率。现在，该公司正在开发一个原生桌面应用程序，可以与其现有的移动应用程序无缝连接。

“对于像邮件这样的东西，效率和体验的一致性非常重要，”安德伍德写道。“原生应用程序让您在各方面都拥有一流的体验:dock 上带有徽章的图标、根据应用程序的用途量身定制的专用用户界面、可以轻松交互的多个窗口，以及不必通过 html 在网络上提供的界面速度。”

一种新的销售方式

虽然 web 应用程序非常适合分发，但由于无需安装或定期更新——用户界面的所有变化都发生在后端，并显示在浏览器中——它们缺乏用户在移动设备上习惯的原生应用程序的速度和功能。

与此同时，分配手段也发生了变化。过去，如果你想要一个特定的桌面应用程序，你要么需要从开发者自己的网站上找到并下载，要么可能通过像 Download.com 这样的应用程序聚合器。在那之前，你可能不得不*气喘* 去实体店买一个盒子，里面有你可以从光盘安装的软件。

但随着桌面应用商店的出现，分发问题正在得到解决，这些商店模仿了用户在移动设备上发现和安装软件的方式。最好的例子是苹果的 Mac App Store，它是 OS X 最新版本的关键组件。但用户也可以在 Chrome 网络商店上找到外观、感觉和反应都像原生应用的应用。

“通过桌面应用商店将可下载、可执行的客户端转移到“应用”的隐喻中会带来一系列的不同，主要是在安全、信任和便利方面，”彭博贝塔合伙人罗伊巴哈特写道。"它还为所有这些应用程序提供了一套一致的权限(尽管相对于野外下载来说是有限的). "

此外，随着时间的推移，移动和桌面连接的融合似乎只会越来越强。苹果和谷歌都一直致力于将跨平台功能构建到他们的应用平台中。对谷歌来说，这意味着移动和网络应用之间的无缝连接，而对苹果来说，这意味着在其 OS X 和 iOS 开发平台之间建立“连续性”。

正如 Product Hunt 的胡佛指出的那样，“iOS 8 + Maverick 的跨平台推送通知将为移动和桌面捆绑在一起带来新的机遇。”

随着桌面、web 和移动开发平台之间的连接不断融合，我们很可能会看到更多的移动开发人员将他们的应用程序带到桌面上。这些应用程序将会变得越来越有用，它们在更多的屏幕上可用。
