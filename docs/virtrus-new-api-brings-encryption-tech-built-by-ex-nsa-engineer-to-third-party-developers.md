# Virtru 的新 API 将前 NSA 工程师构建的加密技术带给第三方开发者

Virtru 的联合创始人 Will Ackerly 在美国国家安全局担任工程师时开发了该公司的底层加密技术，因此可以说他对该主题略知一二。该公司几年来一直在为传输中的电子邮件和文件提供加密产品，主要是通过与[谷歌 GMail](https://web.archive.org/web/20230225042337/https://www.virtru.com/gmail-security/) 和[微软 Office 365](https://web.archive.org/web/20230225042337/https://www.virtru.com/microsoft-office-365-email-encryption/) 的合作。今天，它宣布将通过 Virtru 数据保护平台向第三方开发者开放该技术。

在这些主要平台上使用 Virtru 的工具来加密您的电子邮件和文件固然很好，但开发人员直到现在都无法利用这种加密技术。该平台允许企业和政府机构将 Virtru 加密嵌入任何程序。

这意味着，如果您的应用程序包括一个文件存储，您可以在文件运动时(或静止时)对其进行加密。你可以在文件上附加一个截止日期，立即取消对任何文件的访问，甚至是已经发送的文件。您还可以控制复制和粘贴文件的能力，或者在发送文件之前或之后设置水印。

例如，您可以构建一个程序，使用[Box storage service](https://web.archive.org/web/20230225042337/https://developer.box.com/)作为您的存储组件，并且您可以使用 Virtru 来保护这些文件，所有这些都只需要几行代码。如今，许多软件公司以开放 API 的形式向开发人员公开他们的核心服务。每次程序访问服务时，这些公司都会赚钱。

谷歌和微软目前的系统可能涵盖了许多企业用例，但正如首席执行官兼联合创始人约翰·阿克利(Will 的兄弟)指出的那样，加密传统上对大多数用户来说太难了。他说，以他公司的 Gmail 产品为例，“它实际上是 GMail 中的一个按钮。”如果您愿意，还可以在后端设置策略，以便默认设置加密。

他说，通过向第三方开放这些功能，Virtru 正在将这种能力扩展到他们向任何组织或开发人员提供产品的标准方式之外，通过几行代码将这种能力构建到他们的程序中。

Virtru 由威尔和约翰·阿克利兄弟于 2011 年创立。该公司已经筹集了近 4000 万美元。最近一轮是 2016 年由 Bessemer Venture Partners 牵头的 2900 万美元 A 轮融资。
