随着 Kubernetes 升级到 1.0，谷歌向新成立的云计算基金会捐赠技术

原文：https://web.archive.org/web/https://techcrunch.com/2015/07/21/as-kubernetes-hits-1-0-google-donates-technology-to-newly-formed-cloud-native-computing-foundation-with-ibm-intel-twitter-and-others/

谷歌去年二月推出的开源容器管理工具Kubernetes，今天发布了 1.0 版本。有了这次更新，Google 现在认为 Kubernetes 已经可以投入生产了。然而，更重要的是，谷歌也放弃了对 Kubernetes 的控制权，并将其捐赠给一个新成立的基金会——云原生计算基金会(CNCF)，该基金会将由 Linux 基金会在运营。新基金会的其他合作伙伴包括 AT & T、Box、思科、Cloud Foundry Foundation、CoreOS、Cycle Computing、Docker、易贝、高盛、华为、IBM、英特尔、Joyent、Kismatic、Mesosphere、Red Hat、Switch SUPERNAP、Twitter、Univa、VMware 和 Weaveworks。

Linux 基金会执行董事 Jim Zemlin 今天写道，这个新基金会的使命是“帮助促进开发者和运营商在部署云原生应用和服务的通用技术上的合作”。

如果这一切听起来有点熟悉，那可能是因为仅仅几周前，一个非常相似的公司集团，包括 Docker、Google、IBM、Intel、Mesosphere、VMware 和容器生态系统中的一系列其他参与者，也聚集在一起启动了开放容器项目。这个项目也将由 Linux 基金会管理，它将指导一个通用容器标准的开发。然而，与这个 CNCF 不同的是，这个团队还包括谷歌的竞争对手微软和亚马逊，这两家公司都明显缺席了这个最新项目。

正如谷歌高级产品经理 Craig McLuckie 告诉我的那样，该公司决定为 Kubernetes 寻找一个新家，因为它已经准备好投入生产。“目的是利用我们的技术，让它尽可能无处不在，”当我问他谷歌放弃对 Kubernetes 开发的控制权的动机时，他说。“我们希望人们能够选择一朵云。我们的大多数客户在混合云环境中工作，但希望获得这种云原生计算模式的优势。”

他还指出，谷歌将在该项目中保持非常积极的态度，该公司希望取得成功。有趣的是，他指出，谷歌甚至希望 Kubernetes 能够解决谷歌自己的容器内部工具的一些缺点。

McLuckie 特别指出，虽然 Kubernetes 目前非常适合数百个节点规模的小型集群，但许多客户现在希望能够管理数千个节点。谷歌的团队还希望创建更好的集成，以支持不同类型的工作负载，如批处理。

值得注意的是，Kubernetes 只是 CNCF 管理下的第一个产品。这里的目标远不止于此，重要的是要注意，该基金会并不仅限于 Kubernetes。相反，正如 Joyent 首席技术官布莱恩·坎特里尔(Bryan Cantrill)也今天提到的，该基金会的真正使命是“推进构成现代弹性计算的关键开源技术”

看来，围绕 CNCF 治理模式的一些细节仍有待解决。然而，正如 Linux 基金会主席吉姆·泽姆林告诉我的，这不是一个付费游戏计划，任何人都可以参与(这与其他 Linux 基金会项目非常一致)。基本的想法是，贡献重要技术的实体将在会议桌上占有一席之地。Zemlin 还指出:“引领这一技术变革的个人开发者应该得到一个真正有意义的席位。”“核心开发人员至关重要。”

在容器生态系统中，仍然有一些大型的参与者不在基础之内(微软、亚马逊、Pivotal 等)。)，但泽姆林认为，许多落伍者将在晚些时候加入。“我预测这项技术会好到让人无法抗拒，”他说。"今天不参加的人以后会改变主意的。"
