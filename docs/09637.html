Mirai 未来组合僵尸网络的互联网攻击为攻击者和防御者打开了一个新的市场

原文：https://web.archive.org/web/https://techcrunch.com/2016/10/25/the-mirai-botnets-internet-takedown-opens-up-a-new-market-for-attackers-and-defenders/

当亚马逊、推特、贝宝、Spotify 和其他主要网站在周五变得无法访问时，成千上万的美国人第一手了解了 DDoS 攻击的感觉。

分布式拒绝服务(DDoS)是一种简单的攻击形式，它用垃圾邮件流量淹没了网站，使合法用户无法通过。分布式拒绝服务是一场经济战争:无论是防御者还是攻击者，谁拥有最多的计算能力，谁通常都会赢。

这使得 DDoS 成为对中小型出版商进行审查的有用工具，但是大型网站通常都有防御措施，并且不容易受到这些攻击。然而，周五并不像往常一样正常。摧毁 Dyn 的系列攻击，Dyn是为许多主要网站提供主干的域名系统服务，部分是由被黑客入侵的视频录像机和网络摄像头组成的僵尸网络驱动的，被称为 Mirai 未来组合。几周前，在针对布莱恩·克雷布斯(Brian Krebs)的一次 DDO 中，Mirai 未来组合首次露面。布莱恩·克雷布斯是一名网络安全记者，经营着自己的出版物《T4》、《KrebsOnSecurity.com》、《T5》。

针对 Krebs 的 DDoS 攻击、随后的保护争夺战以及周五的大规模攻击标志着 DDoS 的新篇章。越来越多的网站被迫躲在越来越少的强大的 DDO 保护供应商后面。但这种集中化意味着，随着像 Mirai 未来组合这样的强大僵尸网络变得越来越强大，互联网的更大部分在攻击过程中可能会被离线。

Mirai 未来组合令美国互联网用户恼火，他们周五无法访问自己最喜欢的网站，也令现在被迫召回易受黑客攻击的物联网设备的公司头疼不已——但僵尸网络也在影响 DDoS 保护市场。

克雷布斯袭击

9 月下旬，Krebs 的网站遭到了规模空前的 DDoS 攻击。内容传递网络 Akamai 在四年时间里保护 KrebsOnSecurity 免受 250 多次 DDoS 攻击，但它难以抵御这一破纪录的虚假流量攻击，并且在几次攻击后，将 Krebs 从其服务中踢出。

“这是我们见过的最大的攻击。Akamai 安全倡导者马丁·麦克凯(Martin McKeay)告诉 TechCrunch 说:“我们在免费保护某人，这占用了大量资源。“当时我们做出了决定——这是一个我们出于营销原因保护的客户，但这花费了我们太多的资源，无法使这成为一件长期可行的事情。他调查做 DDoS，梳理和撇脂的罪犯。对我们来说，保护一个对坏人嗤之以鼻的人是不值得的。”

但是 Akamai 的决定不仅仅是为了保护它的人力和技术资源。McKeay 说，DDoS 攻击是如此之大，以至于使周围的互联网基础设施超载。这些攻击有可能导致该公司付费客户的速度变慢。以前攻击 Krebs 网站的许多攻击每秒发送 3-4 千兆比特，但 Mirai 未来组合的攻击在 500-600 千兆比特范围内。

尽管如此，Akamai 还是试图坚持使用 Krebs。该公司要求一些付费的 Prolexic 客户暂时关闭该服务，以便在 Krebs 战斗中提供更多带宽，并避免他们自己的网站出现问题。Akamai 说，这种请求经常出现在维护期间，不应该引起恐慌。但是 TechCrunch 意识到至少有一位 Akamai 的客户被这一请求所震惊——该公司几天后联系了一家竞争对手 DDoS 保护服务公司。

然而，Akamai 表示，它没有因为 Krebs DDoS 而失去客户。Akamai 发言人 Jeff Young 告诉 TechCrunch:“在攻击期间成功保护了 Krebs 的网站后，他的网站是唯一一个被转移到另一个解决方案的网站。每当我们察觉到意外交通流量的可能性时，主动通知客户是很常见的。发送的绝大多数通知都受到客户的欢迎。"

当 Krebs 离开 Akamai 的消息传出后，其他 DDoS 保护服务蜂拥而至，提供他们的服务。随着大规模 DDoS 攻击成为头条新闻，Krebs 没有保护，这是另一个提供商成名的绝佳机会。

DDoS 保护服务可以抵御在线攻击，这些攻击使用大型计算机网络向网站发送垃圾流量，最终导致网站离线。声称 Krebs 是一家 DDoS 保护提供商的客户将是一个强有力的营销时刻，使该公司能够说他们抵抗了互联网上有史以来最强的 DDoS 攻击。在 Krebs 网站遭到攻击后的几天里，一些免费和付费的 DDoS 服务找到了他。

最后，Krebs 去了 Project Shield，这是一个谷歌支持的 DDoS 保护服务，专门与记者、人权组织和选举监督网站合作。Cloudflare 的项目 Galileo 保护以政治或艺术内容为特色的公共利益网站，也试图保护 Krebs。据克雷布斯报道，一家不知名的公司提供了两周的免费保护，随后是每年 15 万到 20 万美元的服务。

Krebs 写道 Akamai 只给了他两个小时的时间来迁移出它的网络，Akamai 后来告诉波士顿环球报这次攻击可能会给公司造成数百万美元的损失。“我们做出了一个商业决定，不再将这个客户留在我们的平台上，并将我们的资源优先用于我们的付费客户，”杨告诉《环球时报》。

克雷布斯写了:

我不责怪 Akamai 的决定。我从一开始就是一个无偿服务的客户，在过去的四年里，Akamai 和它的姐妹公司 Prolexic 在无数次攻击中支持了我。碰巧的是，这最后一次围攻的规模几乎是他们所见过的下一次最大规模攻击的两倍。当攻击开始给公司的付费用户带来问题时，他们解释说放弃我的网站是一个商业决定，简单明了。

保护克雷布斯是谷歌人道主义智囊团运作的项目“盾牌计划”的一大胜利。在 DDoS 军备竞赛中，Project Shield 有谷歌庞大网络做后盾的优势。“盾牌是一个反向代理，”项目负责人乔治·科纳德解释说。"我们在谷歌云平台上运行虚拟机，这些虚拟机正在进行反向代理."

为了赢得 Krebs 的业务，Project Shield 不得不与 Cloudflare 的 Project Galileo 竞争。与 Shield 一样，Galileo 也是一项免费的 DDoS 保护服务，旨在保护敏感内容免受离线攻击。

然而，Cloudflare 首席执行官马修·普林斯(Matthew Prince)表示，让克雷布斯参与伽利略项目一开始就希望渺茫。普林斯和克雷布斯在 2013 年的黑帽安全会议上针锋相对，当时克雷布斯批评 Cloudflare 允许 DDoS 攻击者在该公司付费的 DDoS 保护服务背后避难。Krebs 认为，如果 Cloudflare 把他们踢出去，他们自己也会被 DDoS 攻击。观众席上的普林斯跳上舞台为他的公司辩护。接下来的辩论看起来非常尴尬，很可能让普林斯失去了在大规模攻击后接管 Krebs 的 DDoS 防御的机会。

普林斯表示，他向克雷布斯提供了伽利略的保护，但克雷布斯拒绝了，声称在批评他们之后接受 Cloudflare 的帮助会感觉很虚伪。(Krebs 没有回应 TechCrunch 的置评请求。)“在最初的 24 小时里，谷歌很难保持下去，”普林斯在谈到克雷布斯的网站时说。“我们肯定会阻止它。”

竞争感是双向的。当被问及伽利略项目时，一名 Jigsaw 员工反驳道，“我相信你知道，竞争推动创新。”

“我们希望在建立服务的人方面有一点竞争，因为很多人都在想如何关闭网站，”这位员工补充道。

保护有价值的人

因为 DDoS 是一种简单而有效的审查手段，像克雷布斯这样的独立记者经常成为引人注目的目标。像谷歌、Cloudflare 和 Akamai 这样的公司有保护他们的善意利益，但是保护像 Krebs 这样的人也是很好的营销。信息是:如果我们能处理对这个有价值的出版商的大规模攻击，我们就能处理互联网向你的企业抛出的任何可怕的混乱。

对于盾牌项目来说，微积分有点不同。与 Cloudflare 和 Akamai 不同，谷歌不出售 DDoS 保护。

但谷歌正在努力推销自己作为一个中立政党的声誉，这一点可以在新闻中得到信任。当谷歌将其加速移动页面(AMP)与脸书的即时文章针锋相对时，它需要证明自己是一家站在出版商一边反对审查制度的公司，并避免脸书因趋势话题和删除有新闻价值的内容而犯的错误。

谷歌首席执行官桑德尔·皮帅似乎在 2 月份的巴黎演讲中承认了这一点，当时他宣布与 AMP 联手正式启动项目 Shield。皮查伊说:“有时候，新闻内容是无法获取的，不是因为页面加载缓慢，而是因为你受到了攻击。”他解释说，Shield 将“提供一个更可持续的新闻生态系统。”

谷歌组织世界信息并使其易于获取的使命与新闻业的使命基本一致。“我们在同一个行业，谷歌和新闻。新闻机构越多，互联网就越好，”Jigsaw 的一位发言人告诉 TechCrunch。

保护克雷布斯的机会对盾牌项目来说来得正是时候，该项目今年努力工作以重新获得国外记者和人权维护者的信任。Shield 必须克服谷歌是美国监控国家的一部分的观念。

“我们肯定遇到过这种情况，”拼图公司的发言人谈到这种不信任时说。“我们的理念是，对于我们收集的数据、我们从未收集的数据、我们收集的原因、何时实施以及如何实施，保持透明。我们让他们探索这是否符合他们的最佳利益。对于我们来说，没有什么比加入神盾更符合他们的利益了。”

自从几年前推出测试版以来，Project Shield 一直致力于扩大其影响力。这项服务在皮查伊的演讲后正式在欧洲推出，在那里它保护了几百个网站，大约一周前在拉丁美洲推出。“这是一个需要解释的工具，”科纳德说。“我们已经学会了如何最好地接触那些对我们至关重要的人。我们对这些组织所面临的威胁有了更多的了解，他们被丢弃的频率以及他们知道原因或来源的频率。”

这种扩张带来了建立信任的机会——Project Shield 现在保护约翰-艾伦·那木，他是一名肯尼亚调查记者，在非洲未经审查工作，项目工程师在 2013 年选举期间花时间在肯尼亚了解记者和选举监督员的 DDoS 保护需求。Jigsaw 员工还强调，当 Project Shield 团队收集其保护的网站的信息时，这些数据不会在谷歌内部共享。

然而，Project Shield 在 2014 年明显失策，当时它与 Cloudflare 合作向香港亲民主网站 PopVote 提供 DDoS 保护，但在一次重要的公投前 24 小时退出了这笔交易。除了记者之外，Shield 还保护选举监控网站，而面临政府 DDoS 攻击的 PopVote 似乎是完美的候选人。

对于谷歌和其他提供 DDoS 服务的公司来说，决定谁应该和不应该得到 DDoS 保护是一个困难的过程。对 Akamai 来说，Krebs 是有价值的，直到他花了太多时间“用拇指抠鼻子”,变得太昂贵而无法保护。Cloudflare 为其付费服务提供的全包式方法是 Krebs 和 Prince 之间争论的部分原因，当涉及到伽利略项目时，该公司同样不插手。Cloudflare 选择完全不衡量伽利略项目候选人的价值，而是将决策外包给外部人权和自由新闻组织。

普林斯解释说:“我们对好的和坏的内容的突发奇想不能进来，这真的很重要。”。“我们不挑选赢家和输家。”

虽然谷歌放弃了 PopVote 项目，但 Project Shield 目前正试图采取一种更具包容性的方法。Conard 表示，Project Shield 并不基于内容歧视媒体，指出它在克里米亚冲突期间保护了亲俄罗斯和亲乌克兰的出版商。“杀死某人从来都不是一件合法或合理的事情。我很乐意保护我们不同意的人，”科纳德告诉 TechCrunch。“总的来说，因为我们不认为 DDoS 有合法的使用案例，所以我觉得在这里采取包容性的方法并保持信息的可访问性相对较好。”

想要将自己的网站置于 Project Shield 保护之下的个人需要通过申请程序，并接受 Jigsaw 团队的审查。Jigsaw 通过内部筛选程序管理寻求保护的出版商和组织，包括对照恐怖分子观察名单检查它们，并表示它不会保护像伊斯兰国出版的在线杂志 Dabiq 这样的出版物。“这是一个法律不允许我们提供服务的组织，”康纳德说。

随着巨大的 DDoS 攻击变得越来越普遍，慈善保护项目将面临挑战性的问题，即谁值得保护免受审查，谁不值得。虽然伊斯兰国的宣传部门显然符合美国对恐怖组织的定义，但中国政府可能会以类似的方式将 PopVote 这样的组织归类，而不愿引起争议的科技公司将不得不决定，他们是否要对抗那些杀害自己公民的政府。

DDoS 即服务

Mirai 未来组合的攻击不仅仅是保护提供商扬名立万、扩大客户名单的机会——攻击中也有线索表明，Mirai 未来组合对 DDoS 业务有利。

对 Krebs 网站的攻击很可能是一种审查行为，意在压制他的报道。但 Cloudflare 的首席执行官普林斯认为，Krebs 也可能成为目标，因为关闭他的网站是为新的 DDoS 服务做广告的好方法。有什么比关闭 DDoS 攻击网站更好的方式来宣传你的僵尸网络呢？Mirai 未来组合看起来确实是招人喜欢的。普林斯解释说:“目标确实是随机的。“攻击长度以五分钟为增量，”他补充说，这表明攻击是以分层的方式定价的。

当 Mirai 未来组合的源代码公开后，Cloudflare 对其进行了分析，发现 5%的代码是为了绕过该公司的防御而编写的——这表明僵尸网络的设计不仅仅是为了 Krebs。

但是阿卡迈的安全倡导者麦凯驳斥了对克雷布斯的攻击是为了 Mirai 未来组合的广告的观点。“我认为这不是一个可靠的理论，”他解释道。“你本可以在攻击中安静得多，而不会像对布莱恩网站的攻击那样引起执法部门的注意。您必须知道，如果您正在进行有史以来最大的攻击，您的僵尸网络将无法幸免。它将在全球范围内引起执法行动。”

一种更安全的分布式拒绝服务商业模式是基于勒索的攻击，而不是公开推倒主要网站并占据头条，这种攻击在过去一年里有所上升，攻击者威胁要对一个网站进行分布式拒绝服务，除非他们收到付款。

但是 Mirai 未来组合袭击是巨大的和公开的，使得袭击者的意图不清楚。“你不能发动这么大的袭击而不被逮捕，”普林斯说。

这些攻击有可能是为了向 DDoS 的维护者本身传递一个信息。“这将使提供保护的公司三思，”麦凯说。“如果您是一家在全球范围内提供万亿字节保护的公司，那么在提供 DDoS 保护时您确实需要三思。如果是二级产品线，他们将不得不考虑大规模攻击。这只是我们在未来几年将会看到的情况的前兆。您必须考虑在基础设施上投资更多，或者决定不值得投资，并摆脱 DDO。许多较小的供应商将会看到这些攻击的到来，并退出业务。”

随着小型供应商的关闭，这为像 Cloudflare、Akamai 和谷歌这样的大型供应商扫清了道路。但是，正如 Dyn 攻击所表明的，集中化可能是危险的。
