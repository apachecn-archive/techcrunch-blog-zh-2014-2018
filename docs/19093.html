在 Twitter 的 API 改变之前，Tweetbot 失去了几个关键功能

Twitter 的 API 变化要到明天才会出来，但它的影响已经开始显现。Tapbots 今天发布了对 iOS 版 tweet bot 的更新，该更新失去了许多 Twitter 客户端最受欢迎或最重要的功能。它还移除了苹果手表应用。在 Tweetbot 的应用商店发布说明中，Tapbots 解释说“8 月 16 日，Twitter 将禁用我们在 Tweetbot 中使用的部分公共界面。因为 Twitter 选择不提供这些界面的替代品，我们被迫禁用或降级某些功能。我们对此感到抱歉，但不幸的是，这完全超出了我们的控制范围。”

这些变化意味着 Tweetbot 的时间线流现在被禁用，因此时间线将改为每一到两分钟刷新一次——这对希望实时看到新推文的人来说是一个损失。提及和直接消息的推送通知也将延迟几分钟，而喜欢、转发、关注和引用的推送通知已被完全禁用(Tapbots 的发布说明称，他们正在研究如何在未来恢复其中一些通知)。Tweetbot 的活动和统计标签已经被删除。

作为加强控制第三方开发者如何使用其服务的努力的一部分，Twitter 在 2017 年 4 月宣布，将关闭用户流、网站流和其他 API，为其新账户活动 API 和其他产品的到来做准备。

其他可能会受到 API 变化影响的第三方 Twitter 客户端包括 Twitterific、Tweetings 和 Talon，它们与 Tweetbot 一起在 4 月抗议说，他们没有得到足够的时间或信息来准备原定于 6 月 19 日的发布。作为回应，Twitter 将截止日期延长至 8 月 16 日。其他已经受到影响的应用包括 Favstar，，由于 API 的变化，该应用于 6 月份下线。
