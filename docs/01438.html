编者按: Steven Hillion 是 Alpine Data Labs 的联合创始人，他在那里领导高级分析企业平台的开发。在加入 Alpine 之前，他曾在 Siebel 和 Greenplum 等公司管理数据科学家和工程师团队。

数据比以往任何时候都更加多样化和快速移动，现在有效地分析数据需要高度复杂的软件和机器。但是，大数据分析把传统的直觉留在了哪里？如果数据告诉一个业务经理“跳”，但她的直觉说“留”怎么办？

这听起来可能会让我感到惊讶——毕竟我是一个数学和技术专家——但我坚信，浸淫于数据和商业头脑中的直觉必须引导分析，才能产生真正的价值。

有一种观点认为，你只需要对数据集应用足够的数学和机器能力，就能获得最好的模型。但是，认为仅仅是数字运算就能提供企业发展所需的答案是愚蠢的。在数据科学中，直觉和分析协同工作，相互促进。

首先，直觉引导分析。分析见解很少凭空出现。它们是应用数值方法来测试直觉和观察产生的假设和想法的结果。直觉也指导着研究者用来检验这些假设的方法。哪些数据是相关的？哪些变量和变换有意义？原因和结果之间可能有什么关系？哪些型号合适？

第二，分析传达直觉。无监督建模技术可以辨别数据中的关系和模式，这些关系和模式在表面视图或人类大小的数据样本中是不明显的。简而言之，分析可以建议探索的途径，这些途径不会被观察发现，甚至可能是反直觉的。

如果数据和业务两方面都没有明智的飞行员来指导数据分析过程，并根据专业经验和知识平衡直觉，肯定会出现问题。

让我举几个我想到的例子。

一个消费者银行团队希望我们创建一个客户流失模型，以帮助银行预测哪些客户可能会取消他们的账户。这些数据产生了一幅暗淡的画面。事实证明，当涉及到储蓄、贷款和信用卡时，没有明确的触发因素来揭示客户何时会跳槽。在做出决定和创建新账户之前，支出和支付模式基本保持不变。

然而，随着银行家更仔细地检查数据，审查团队创建的一组客户细分，一位分析师利用她的直觉突然注意到一个有价值的新见解。她意识到，某个客户群表现出异常高的贷款价值、长期客户价值和其他几个异常因素，可能主要属于小企业主。对个人账户的审查证实了她的怀疑。

她猜测，那些伪装成常客的企业主没有意识到，可能有比使用信用卡或普通贷款账户更好的融资方式。项目目标转移到识别这些高价值客户，并为他们提供更合适的产品。然后，银行团队更进一步，要求数据根据历史用户行为识别向其他客户群推荐的产品。这些数据使他们能够开始向客户提供定制的产品，增加终身价值。

仅仅依靠数据是不太可能提供这种关键见解的。这种商业洞察力加上分析是无价的。(实际上，我们的分析确实计算了产品推荐的价格和利润，但你明白我的意思了。)

考虑到直觉在数据分析中的重要性，为什么业务方面经常被忽略，直到最后。相反，业务分析师应该在早期被邀请到这个过程中来进行合作。我已经改变了流程，将整个团队带入最初的模型评审，甚至更早，带入原始数据的评审。

在另一个例子中，我们的一个客户，一家大型饮料公司，想要预测未来在日本的销售额。我们建立了一个模型，研究未来一年销售如何应对不同的市场和定价压力。客户告诉我们，他们认为销售直接受到经济的影响。他们认为，如果日本经济缓慢复苏，软饮料消费将会增加。

他们要求我们在模型中使用日经指数作为一种趋势变量。该指数起初提高了模型的准确性——至少看起来是这样。但是过了一年，这个模型开始做出疯狂的预测。经济已经开始反弹，但现在日经指数超出了训练数据的范围，原始模型可能“过度拟合”。

一个更有经验的建模者可能会拒绝引入这个变量。有时候直觉是有道理的，但在这里，数据科学专家建议谨慎，并意识到建模过程的局限性和陷阱。在这种情况下，我们引入了一种转换来抑制股票市场指数的影响，并且这些模型在指导新营销计划的开发和预测其效果方面表现得非常好。

数据科学家和企业之间经常存在紧张关系——特别是当数据似乎与直觉相矛盾，而引人注目的新活动的影响似乎可以忽略不计时。通常我们会坐在桌子对面，营销人员会问‘这个数字是从哪里来的？’和处于守势的数据科学家。

但我相信这场脑力战是积极的。数学和科学应该经得起质疑。有时这会导致数据推翻直觉。其他时候，那些基于深刻经验的直觉可以发现过程中的缺陷。理想情况下，每个人都受益。
