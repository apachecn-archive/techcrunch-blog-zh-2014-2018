# 为什么 Dropbox 决定放弃 AWS，建立自己的基础设施和网络

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/09/15/why-dropbox-decided-to-drop-aws-and-build-its-own-infrastructure-and-network/>

无论需求如何，公司内部对于是建造还是购买总是存在矛盾。几年前 [Dropbox](https://web.archive.org/web/20230329031226/http://dropbox.com/) 决定将其大部分基础设施需求从 [AWS](https://web.archive.org/web/20230329031226/https://aws.amazon.com/) 转移到自己的数据中心。你可以想象，这需要巨大的努力，但该公司认为，控制自己命运的优势值得他们面对所有的挑战。

首先，像 Dropbox 这样的公司正在处理大量存储着大量数据的客户。最新数字是 5 亿用户和 20 万企业客户。当他们进行转换时，他们不得不移动一直在 AWS 服务器上的 500 后跟 17 个 0)。(他们仍然使用 AWS 处理一些工作负载。)

第一步是建立基础设施来取代它。我们谈论的是一家拥有 1500 名员工的公司，基础架构团队只有十几个人。这并不是一个巨大的操作，但是他们试图做的是在网络规模上自己建立一些东西，只有少数拥有更大团队的公司尝试过这一点。

这包括建造和装备三个美国数据中心。这还意味着建设网络主干，这是一种促进美国数据中心和他们在世界各地的其他设施之间的连接的基础设施。当你打开 Dropbox 并请求一个文件时，你希望你的文件几乎即时下载而没有延迟，这取决于团队来确保这一点，同时尝试在旧系统和新系统之间导航。

Dropbox 生产工程负责人丹·威廉姆斯(Dan Williams)告诉 TechCrunch:“主干的巧妙之处在于，它类似于你可能在谷歌或脸书找到的东西，但我们是用一个相对较小的团队建造的。”。这个小团队必须构建主干网并移动所有内容，同时保持服务正常运行。

虽然最终该公司愿意进行这一大规模的迁移，因为它渴望控制其基础设施。威廉姆斯承认，当做出这样的决定时，总会有让步，但最终对他们来说，这种权衡是值得的。“对我们来说，这关乎质量、控制和管理。我们知道有可靠的第三方拥有[高]质量和性能，但我们认为我们的可以与之相当，甚至更好，因为我们非常了解这个系统。”

威廉姆斯说，对于 Dropbox 来说，建立网络是一个商业决定，它对整个业务产生了积极的影响。威廉姆斯解释说:“我认为，事实上可以说，任何建立了这样一个相当规模的网络的人都以积极的方式对业务产生了一些影响，实际上是为用户建立了信任，并让更多的用户基于服务质量来采用产品或服务。”。

新系统无疑也对 Dropbox 在企业 IT 领域的声誉产生了积极影响。过去，Dropbox 经常因为在大型组织内部未经授权的使用而名声不佳。今天，Dropbox 业务系列产品与这种内部基础设施和网络相结合，创造了一种前所未有的信任水平。威廉姆斯指出，他们对网络运营有如此详细的了解，他们可以将这些数据用作销售驱动力，尽管他们最终没有对这种变化收费。

"我们的核心信念之一是提供高性能、低成本的产品."他认为这就是为什么他们能够通过这种转变留住老客户，同时随着时间的推移扩大用户群。

无论实际拥有成本是否下降，该公司相信它通过自己构建实现了目标。“虽然成本一直是我们考虑的因素，但我们网络扩张的目标是为我们的用户提高性能、可靠性、灵活性和控制力——我们已经成功做到了这一点，”公司发言人告诉 TechCrunch。

构建还是购买的决定从来都不是一个容易的决定，尤其是对于 Dropbox 这样规模的公司来说，在过渡期间，它必须在两个系统之间游走，但今天它似乎为他们带来了巨大的回报。
