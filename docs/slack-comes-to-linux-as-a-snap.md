# Slack 迅速进入 Linux 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/01/18/slack-comes-to-linux-as-a-snap/>

Slack 的雄心是成为员工相互聊天并链接到数百个其他应用程序以完成工作的默认去处，今天通过在一个新平台上可用，它又向前迈进了一步。从今天开始，Slack 将会以 Snap 的形式[提供，这是一个可以在一系列基于开源的 Linux 环境中使用的应用程序包。](https://web.archive.org/web/20221205125106/https://snapcraft.io/slack/)

这使得这款应用面向更广泛的新人群:Canonical 工程、设备和物联网副总裁杰米·贝内特(Jamie Bennett)表示，在多个 Linux 发行版中有“数千万”桌面用户。(作为对比，Slack 在【2017 年 9 月表示，其日活跃用户超过 600 万，其中 200 万是付费用户。)

除了 Ubuntu(由 Canonical 开发)之外，这将向基于 Linux 的服务用户开放 Slack，如 ArchLinux、OpenSUSE、Solus、Debian、KDE Neon 等。

你可能知道，Slack 最初的构想是为 TinySpeck 的开发者提供服务，这家初创公司最终成为了 Slack。尽管过去一年[一直在努力扩大用户数量，向更大的企业领域推进，但与使用 Linux 的开发人员社区结盟是一种潜在的强大和自然的契合。](https://web.archive.org/web/20221205125106/https://beta.techcrunch.com/2017/01/31/slack-enterprise-grid/)

Bennett 在接受电子邮件采访时表示:“由于 snap 旨在跨桌面、物联网设备和云工作，snap 在所有或部分这些设备上应用的潜力甚至更大，并且通过简化流程和只需要创建一个版本，使开发人员的生活变得更容易。”

Bennett 说，Slack 使用 Canonical 的 Snapcraft 工具自己构建了快照，这是所有开发人员用来创建快照的标准工具。当被问及谁联系了谁时，他说这款应用是 Slack 自己开发的，以响应 Linux 社区的请求。“尽管 Slack 年前才推出，但他们发展非常迅速，已经看到了包括 Linux 用户在内的所有平台的需求，”他说。

“因此，为 Slack 提供选择并迎合所有用户是很重要的，这也是为什么构建 Linux 对他们来说很重要，并且他们经常参与更广泛的 Linux 社区。”
