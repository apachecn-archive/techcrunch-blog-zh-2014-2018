Pulumi 希望让您使用 code TechCrunch 管理您的基础设施

原文：https://web.archive.org/web/https://techcrunch.com/2018/06/18/pulumi-wants-to-let-you-manage-your-infrastructure-with-code/amp/

总部位于西雅图的初创公司 Pulumi 希望开发者和运营团队能够更容易地通过编写代码来定义他们的基础设施。该服务的工具不使用特定于云的配置语言，而是允许开发人员使用他们已经用于应用程序的相同编程语言来为他们的应用程序定义基础架构。

这项服务得到了 Madrona Venture Group 和 Tola Capital 的支持，Madrona 的 S. Somasegar 加入了其董事会。

有趣的是，无论基础架构是容器、虚拟机还是无服务器功能，或者是在私有云还是主要公共云中运行，都无关紧要。目前支持的语言包括 JavaScript、TypeScript、Python 和 Go，支持。NET 和 Java，C#紧随其后。

Pulumi 首席执行官 Joe Duffy 拥有丰富的开源经验(他在微软建立了一个团队。NET 开源)，所以 Pulumi 也有许多开源组件也就不足为奇了。除此之外，该服务还提供了一个托管服务，用于在部署 Pulumi 堆栈之后管理它们，以及用于协作和将服务集成到现有工作流中的工具。

正如达菲和他的联合创始人兼执行主席埃里克·鲁德尔(Eric Rudder)在我今天发布之前会见团队时所说的那样，今天特定于供应商的模板语言只会导致“配置蔓延”，并导致模板将代码和配置混合成无人愿意接触的不可读文件。Rudder 和 Duffy 认为，这种方法也导致了开发人员和试图支持他们的运营团队之间更大的差距。例如，团队告诉我，他们成功地将 1000 多行 Helm 代码减少到不到 200 行 Pulumi 代码——而这 200 行代码中的大部分是可重用的。

Somasegar 说:“世界各地的企业客户正在大规模转向云计算。“随着这一趋势继续发展壮大，随着客户真正开始利用云原生功能，需要新的变革性技术。这种转变突飞猛进，无服务器计算开始成为下一个前沿领域，实现由微服务和事件驱动功能支持的真正分布式应用和服务。”在他看来，Pulumi 的代码优先方法将有助于推动这一运动向前发展。

Pulumi 现在是预览版——它所有的定价方案都是免费的。
