# 优步启动漏洞奖励计划，奖励黑客发现安全问题 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2016/03/22/uber-launches-bug-bounty-program-that-pays-hackers-to-find-security-issues/>

[优步](https://web.archive.org/web/20221207094210/https://www.uber.com/)今天宣布，经过一年的测试后，它将在 [HackerOne](https://web.archive.org/web/20221207094210/https://hackerone.com/) bug bounty 平台上[开放其首个 bug bounty 程序](https://web.archive.org/web/20221207094210/https://hackerone.com/uber)。

Bug bounty 计划对于那些不熟悉行话的人来说，付钱给黑客或研究人员，因为他们更喜欢称之为寻找软件平台上的安全漏洞。本质上，这是通过支付他们的努力，把那些曾经利用这些问题的人变成你的朋友。

大多数大公司如微软、谷歌和脸书都有这种类型的程序。事实上，帮助优步开发这个程序的 Collin Greene 之前曾在脸书与 HackerOne 的 CTO Alex Rice 一起帮助建立了一个类似的程序——毕竟这是一个小世界。

优步的计划有几个独特的组成部分。首先，当涉及到基本规则和报酬时，它试图尽可能直接地与研究人员沟通。格林说，研究人员/黑客对这些程序的一个问题是，支付系统可能是反复无常的。有人发现了一个 bug，并就其价值展开了谈判。

他说，这个项目将会非常清楚优步将会支付多少钱，为一个关键的错误提供高达 10，000 美元。

其次，该公司希望奖励忠诚的研究人员，他们报告了大量的错误，所以他们正在建立一个忠诚计划。“实际上只有一小部分(合格的研究人员)能够在这些应用程序中找到漏洞，这一比例很小，你希望抓住他们的注意力并留住他们，”Greene 解释道。

从本质上讲，优步把寻找漏洞游戏化，以吸引最优秀的研究人员。第一个忠诚度计划于 5 月 1 日启动，持续 90 天。如果研究人员在这段时间内发现了 4 个错误，他或她在报告第五个和任何后续错误时会获得奖金。据优步称，奖金相当于该时段所有其他股票平均奖金的 10%。

最后，优步提供了一份他们称之为“藏宝图”的文件，给了研究人员一点先机虽然这是一个有趣的名字(增加了程序的游戏化方面)，但它提供了关于寻找错误和漏洞的最佳位置的有价值的信息。

“我们看代码，像黑客一样思考，寻找安全漏洞。[参与者]获得我们积累的关于代码库和最有可能发现 bug 的领域的智慧，”Greene 说。

该公司从 200 名研究人员的测试版开始，并在去年完善了该计划，以包括今天宣布的元素。HackerOne 的首席技术官亚历克斯赖斯(Alex Rice)表示，这是他们第一次看到一家公司在发布前以这种方式收集反馈。

“优步像所有的 HackerOne 客户一样开始运营一个私人飞行员，但他们的项目很独特，因为他们特别强调从黑客那里收集反馈，告诉他们如何最好地组织他们的项目以使其有效。从这里开始，他们在 HackerOne 与我们合作，创建运行忠诚度计划所需的功能，”赖斯告诉 TechCrunch。

希望随着时间的推移，这些漏洞会越来越难被发现，它们会根据从赏金计划参与者那里获得的信息，越来越多地填补代码库中的漏洞。当这种情况发生时，付款将随着难度的增加而增加，每个人都是赢家。黑客们的辛勤工作得到了回报，优步建立了一个更安全的平台。
