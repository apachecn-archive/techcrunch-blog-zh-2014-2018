# 设计优秀企业软件的经验

随着人们对工作场所的高质量用户体验有了新的期望，提供消费级体验对所有商业软件来说都变得至关重要，也是软件制造商在市场中竞争的基本方式。虽然大多数软件制造商越来越意识到这个事实，但今天的挑战是如何有效地过渡到为企业构建消费者质量的软件。

**没有工程师的设计师是美术馆，没有设计师的工程师是停车场**

平衡设计师和工程师的观点，甚至设计师和产品经理的观点是至关重要的。不要让设计师在项目开始或结束时仓促设计，他们需要在整个过程中与工程师密切合作，以适当地调整设计和工程能力。

首席设计师与产品经理共同承担产品定义的角色，这创造了一种健康的张力，有助于在多人从两个不同但同样重要的角度定义一个项目时提供制衡:

1.  设计师:设计视角，关注可用性、工作流程和唤起客户良好的情感反应

2.产品经理:商业视角，专注于为公司增加利润和市场份额

我们发现，与工程团队的其他成员一起组织产品设计的轴辐式模型运行良好。我们作为一个单一的设计团队来处理问题和进行评审:这就是中心。但我们也有代言人:每个项目的首席设计师，完全融入并专注于项目团队的成员。拥有嵌入式设计人员可确保我们始终有一名代表作为单点联系人，与工程团队在其开发生命周期中处理的日常问题保持联系。

**激发情感联系**

执行用户研究、利用正确的设计模式、反复测试和应用可靠的交互设计来打造最能支持客户目标和工作流程的产品体验是至关重要的。让它易于使用，这样人们就不会犯导致沮丧的错误。我们将产品流程决策建立在研究的基础上，这些研究让我们了解客户的目标，以便我们能够专注于解决正确的问题。

这导致伟大的功能性产品设计，但要真正培养信任和忠诚度，必须与客户建立情感联系。用一个巧妙的图表动画来取悦人们，一个在图表加载时有用的指示器，一个菜单如何快速返回原位的巧妙手势。这种设计有助于人们建立情感联系。他们可能不会有意识地注意到它，但潜意识里它可以帮助建立信任，增加产品的成瘾性。

我们在许多消费产品中都看到了这种情感化的设计，例如 iPhones 上令人愉悦的拖动和捕捉动画，当你打开设备时，设备会发出声音，或者当我将我的 Misfit Shine 可穿戴设备与移动应用程序同步时，会出现脉动动画。

**在 beta 测试之前**

对于数据和信息密集的 UI，您需要让人们看到他们自己的数据和信息来测试软件。正确评估我们的 UI/设计过程需要用真实数据进行实验和原型制作，然后才是修复错误和对体验进行轻微打磨的测试过程。

因此，我们重视在设计过程的早期让客户用他们自己的数据和信息来测试产品。在 beta 测试期间等待这样做是有风险的，因为你通常已经在开发中走得太远了，以至于无法在不招致重新开始的巨大成本的情况下对你的概念中的重大问题做出反应。

在每个开发冲刺阶段，我们通常会有一些客户随时待命，作为一个探测板来审查产品并帮助评估可用性。在过程的早期，甚至在产品被完全定义之前，我们和他们一起尝试不同的原型，以确定他们对哪些原型反应更好，帮助他们解决对他们来说重要的问题。

当然，大多数软件制造商试图这样做，但是对于处理数据的企业产品和无数种表示数据的方式来说，这可能是额外的挑战。预测客户将会问的问题(即使他们还不知道他们想问什么问题)的最佳方式是在整个过程中获得实时反馈。*询问*顾客想要什么和看着他们实际使用某样东西是有很大区别的。你需要两者都做。

**平衡反馈并保持专注**

也许最大的挑战是让用户界面对那些身兼多职的初创公司来说足够简单，对那些要求更多控制和灵活性的企业客户来说又足够强大。为了保持这种平衡，你必须仔细权衡你得到的反馈，有时还要避开看似合理的建议。

例如，当我们几年前第一次建造[新遗迹服务器](https://web.archive.org/web/20221209072304/http://newrelic.com/server-monitoring)时，我们开始提供非常简约的服务，以确保我们不会给人们带来过多的信息。我们为客户提供了几种方法来查看他们的服务器在做什么。但是有一个客户想要一个非常不同的东西:一个由 30 个图表组成的网格，包含你在主屏幕上能想到的所有指标。

对于像这样的客户，当他们需要更深入了解时，我们让他们更容易深入了解。但是我们没有在第一个屏幕上包括所有的内容，因为我们不想让大多数客户对每一个可能的指标都一视同仁。这没有带来正确的价值。

遵循最佳设计团队结构、早期调整和与客户持续实时协作的简单最佳实践，创造惊喜和愉悦的用户体验，并了解将哪些反馈纳入设计将有助于企业构建软件，在日益软件驱动的世界中成为竞争优势。
