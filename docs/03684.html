Mozilla 推出不支持 DRM 的新 Firefox 版本

差不多一年前，Mozilla 宣布将(非常不情愿地)在 Firefox 中实现 HTML5 DRM 规范。今天，随着 Firefox 38 的发布，该组织正式推出了 HTML5 DRM 支持。然而，除此之外，Mozilla 还宣布推出一个单独的 Firefox 下载，它不会自动安装 Adobe 的技术，用于在浏览器中播放 DRM 包装的内容。

随着今天火狐 38 的发布，默认版本的浏览器现在支持Windows 桌面(Vista+)上的加密媒体扩展 API，并自动下载 Adobe 内容解密模块(CDM)。

这里的论点是，这将允许用户观看来自网飞和类似服务的内容，而不必处理像微软的 Silverlight(即将过时)和 Adobe 的 Flash 这样的插件。浏览器中内置的 DRM 解决方案可以很好地符合 HTML5 标准，这意味着用户可以观看视频，而不必考虑插件和随之而来的安全问题。网飞已经在测试火狐的数字版权管理解决方案，以向其用户提供流媒体内容。

不出所料，封闭源代码的 DRM 解决方案和免费开放的 Firefox 浏览器之间的冲突——以及 Mozilla 的总体使命——并没有让 Firefox 和开源的长期支持者满意。他们现在可以下载无 CDM 版本的 Firefox。

当 Mozilla 决定在其浏览器中采用 DRM 时，它处于一个艰难的境地。T2 已经很难留住现有用户，大多数用户可能并不太关心 DRM，但他们会关心网飞是否突然停止工作。最终，该组织做出了支持这项技术的务实决定，但它显然不喜欢这样做。
