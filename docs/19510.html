Cloudflare 新的“一键式”DNSSEC 设置将使欺骗网站变得更加困难

原文：https://web.archive.org/web/https://techcrunch.com/2018/09/18/cloudflare-dnssec-one-click-securing-internet/

先说坏消息:互联网断了一段时间。好消息是，Cloudflare 认为它可以使它稍微不那么破碎。

这家网络巨头周二表示，只需“点击一个按钮”，用户就可以在他们的仪表板上打开 DNSSEC。通过这样做，Cloudflare 希望它能够消除采用 web 安全标准的一个主要痛点，许多人还没有建立 web 安全标准，要么是因为它太复杂和艰巨，要么是因为太昂贵。

这是这家总部位于旧金山的网络巨头努力使互联网管道更加安全的一部分——即使是你看不到的东西。

多年来，你可以打开一个网站，并认为它的即时可用性是理所当然的。将网址转换为计算机可读 IP 地址的 DNS 一直受到漏洞的困扰，很容易劫持该过程的任何一步，秘密地将用户发送到虚假或恶意的网站。

以去年发生的两起事件为例——亚马逊和 T2【谷歌、脸书、苹果和微软的流量被劫持，每次改道几分钟到几小时。万亿字节的互联网流量在俄罗斯被吸走，原因尚不清楚。俄罗斯政府至少在理论上可以读取任何未加密的信息。可疑？的确是。

这就是以安全为重点的 DNS 演进——DNSSEC——的意义所在。它类似于 DNS，但它通过加密签名数据来保护从计算机或移动设备到您试图访问的网站的 web 服务器的端到端请求，因此更难以欺骗(如果不是不可能的话)。

但是 DNSSEC 的采用率低得可怜。《财富》1000 强中只有 3%的网站签署了他们的主域名，主要是因为域名所有者不愿意被打扰，但也因为他们的 DNS 运营商要么不支持它，要么收取过高的特权费。

Cloudflare 现在希望为受支持的注册服务商的客户设置这些关键的 DS 记录，这是设置 DNSSEC 的必要组成部分。传统上，创造一个 DS 记录是出了名的困难，通常是因为注册商自己也有问题。

自发布之日起，甘地将成为第一个支持一键式 DNSSEC 设置的注册服务商，预计会有更多的注册服务商跟进。

该公司认为，支持此举的注册商越多，实现更安全互联网的障碍就越少。目前，该公司表示，用户应该考虑从提供商那里转移的服务不支持 DNSSEC，并“让他们知道这是转移的原因。”

就像 HTTPS 在过去几年里缓慢采用——但最终在 2015 年起飞——DNSSEC 有望遵循同样的命运。越多的公司采用这项技术将有助于最终用户在互联网上不容易受到 DNS 攻击。

除了黑客，谁不想这样呢？
