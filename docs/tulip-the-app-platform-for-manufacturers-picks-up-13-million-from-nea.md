# 面向制造商的应用平台 Tulip 从 NEA TechCrunch 获得 1300 万美元

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/06/06/tulip-the-app-platform-for-manufacturers-picks-up-13-million-from-nea/>

虽然大型原始设备制造商的车间正在拥抱机器人和自动化时代的开端，但在这些车间工作的实际人类大多已被技术遗忘。但是为商店工程师、操作员和经理提供的软件平台 Tulip T1 正在寻求改变这一切。

为此，郁金香已经在首轮融资中筹集了 1300 万美元，由 NEA 牵头，Pitango Venture Capital 和其他现有投资者参与。作为交易的一部分，NEA 的合伙人戴娜·格雷森将加入公司董事会。

尽管机器人和自动化的兴起将改变制造商生产商品的方式，但在这些工厂工作的人们通常使用最基本的工具来完善流程、调试等。这包括剪贴板、秒表和 Excel 电子表格，它们远非实时，只是收集大量手动输入的数据，而不是自动提取见解。

这就是郁金香的由来。

Tulip 提供了一个软件平台，让传感器、摄像头和其他物联网硬件在一个中央后端系统中相互通信，而不管协议如何。不熟悉这类软件的人(如车间操作员和工程师)能够即插即用各种“如果/那么”的情况，以鸟瞰地板上的问题区域并立即修复它们。

通过 Tulip，制造商可以实施交互式工作指令、自动数据收集、质量控制、审计、机器监控和培训。

例如，Tulip 使现场经理能够专注于数据和见解，而不是徘徊在展示现场，站在操作员和工程师的肩膀上。特定工作台上的传感器、特定工具上的传感器或装配线上的摄像头可以自动检测整个过程中发生的情况，帮助管理人员优化过程，而无需手动收集和输入数据。

“在某种程度上，我们正在使这些操作员、工程师和经理能够做信息员工认为理所当然的事情，”Tulip 首席执行官兼创始人 Natan Linder 说，他创建了 Tulip，作为他在麻省理工学院围绕接口进行的研究的副产品。“除此之外，Tulip 将工作变成了一种游戏，具有排行榜的机制，并产生了一种让员工了解自己表现的动力。”

Tulip 表示，德勤的分析发现，在全球制造商 Jabil 运营的前四周，Tulip 的产量增加了 10%以上，人工装配质量问题减少了 60%。

尽管该公司没有披露其全部定价，但 Tulip 通过一种 SaaS 模式向其客户收取每条长凳的费用。

郁金香计划利用新的投资来发展产品、设计和研究团队，并扩大客户支持。
