# 用户抗议促使一加停止过度的数据收集

本周早些时候，独立手机制造商一加被披露正在从运行 OxygenOS 的手机上收集各种信息——当然，没有告诉用户。被当场抓住，该公司正在退出选择退出数据收集计划，让用户提前选择，而不是埋没在选项中。

这一令人不快的遥测技术是在本周早些时候被发现的，当时软件工程师克里斯托弗·摩尔[碰巧在一次黑客挑战中窥探了他手机的流量](https://web.archive.org/web/20230316161016/https://www.chrisdcmoore.co.uk/post/oneplus-analytics/)。他注意到，该设备在崩溃时正在给一加的家打电话——这是意料之中的，也是良性的——但每次电话被唤醒或进入睡眠时也是如此——这很奇怪，也很烦人。

仔细观察，他发现该设备还在重复发送其 IMEI、电话号码、序列号、wi-fi 网络和 MAC 地址以及许多其他指标。可以选择将这些信息与错误报告一起发送，这是可以理解的，但是每次应用程序启动时都要发送这些信息。

一加当时表示，这些数据是为了“根据用户行为微调我们的软件”，并“提供更好的售后支持”它可以在高级设置中部分关闭，或者用命令行工具完全删除。

当然，在所有手机制造商中，一加可能拥有最有可能为这种东西四处窥探的用户，所以采用这种明显侵入性的指标很奇怪。用户显然被困扰了，所以昨天一加在其支持论坛上提供了一个更实质性的回应[。](https://web.archive.org/web/20230316161016/https://forums.oneplus.net/threads/lets-talk-about-oxygenos-analytics.654820/)

在标准的“我们非常认真地对待我们的用户及其数据隐私”样板文件并向人们保证这完全是一个大误会之后，一加联合创始人 Carl Pei 解释了该公司正在采取的实际步骤:

> 到 10 月底，所有运行 OxygenOS 的一加手机都会在设置向导中出现一个提示，询问用户是否想加入我们的用户体验计划。设置向导将明确指出该计划收集使用分析。此外，我们将包括服务协议条款，进一步解释我们的分析收集。我们还想告诉大家，我们将不再收集电话号码、MAC 地址和 WiFi 信息。

他还指出，该公司从未将这些信息发送给任何第三方，这很好。但是选择退出“用户体验计划”似乎并不能阻止遥测数据的发送——它只是意味着“你的使用分析将不会与你的设备信息绑定。”用户可能更愿意知道他们的数据根本没有被收集，但是现在这个选项似乎仅限于以前的命令行工具。
