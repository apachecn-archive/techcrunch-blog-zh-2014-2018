微软收购图像灾难恢复服务

微软今天宣布已经收购了in image，这是一项业务连续性服务，帮助企业在公共云和私有云之间迁移数据，在云中复制内部资产，并在出现问题时恢复数据。收购的财务细节没有披露。

微软显然有兴趣让 Azure 对企业客户更有吸引力。该公司表示，此次收购将有助于其为任何客户 it 环境提供混合云业务连续性解决方案，“无论是 Windows 还是 Linux，是基于 Hyper-V、VMware 或其他平台的物理或虚拟化环境。”

然而，具体来说，该公司负责云和企业营销的企业副总裁 Takeshi Numoto 认为，这次收购将有助于使 Azure 成为“世界上几乎所有企业服务器灾难恢复的理想目的地。”因此，他还设法深入了解了 VMware，因为在他看来，这也将有助于 VMware 的客户“探索将其应用程序永久迁移到云的选项，这也将提供一个巨大的商机。”

InMage 现有的 Scout 服务将被整合到 Azure Site Recovery 服务中，InMage 将支持使用 Scout 将数据迁移到 Azure。现有客户将能够继续使用这项服务，但未来，新客户将通过 Azure 的站点恢复服务获得 Scout。目前还不清楚 InMage 的其他产品会发生什么，包括其 InMage-4000 设备，但根据 InMage 网站上的消息，所有现有产品将继续像以前一样工作。

总部位于圣何塞的 InMage 成立于 2001 年，之前已经从 Hummer Winblad、Amidzad Partners 和 Intel Capital 等公司筹集了至少 3600 万美元的风险投资。

微软经常说，它认为企业对混合云解决方案最感兴趣，InMage 的技术正好符合这一战略。
