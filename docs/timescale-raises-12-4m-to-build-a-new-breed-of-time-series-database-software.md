# 时标筹集了 1240 万美元来开发一种新型的时间序列数据库软件

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/01/24/timescale-raises-12-4m-to-build-a-new-breed-of-time-series-database-software/>

无论是制造商还是银行，企业正在收集比以往任何时候都多的信息，对于首席执行官 Ajay Kulkarni 来说，现有的数据库软件很快将无法提供他们所需的马力。

这就是为什么他和首席技术官迈克·弗里德曼(Mike Freedman)创办了[时标](https://web.archive.org/web/20230306054831/https://www.timescale.com/)，一个新的开源时间序列数据库软件。这个想法是，企业越来越需要从他们收集的数据中获得洞察力的方法，无论是从电厂的传感器到金融交易，他们都需要实时完成，没有出错的余地。时标希望开发出一种新的数据库软件，让开发者感到熟悉，但提供这种能力，然后在此基础上建立一个企业。为了实现这一目标，时标在新一轮融资中筹集了 1240 万美元，由 Benchmark Capital 牵头，NEA 和两家适马风险投资公司参与其中，Benchmark 的彼得·芬顿加入了这家初创公司的董事会。

“开发人员有两种选择:SQL 和关系数据库，它们易于使用且可靠，但难以扩展，”Kulkarni 说。“他们还有 NoSQL，规模有所扩大，但[不太可靠]。当我们审视机器数据的本质时，我们意识到有一种方法可以两全其美。但也意识到机器数据本质上主要是时间序列。每个机器记录都是带有时间戳和一些元数据的度量。我们所做的有点不同和异端，我们在 Postgres 的基础上建立了它，通过这样做，我们继承了 Postgres 的所有操作工具，但我们最终使它可以扩展到时间序列数据。”

时标位于 Postgres 之上，Postgres 是当今开发人员使用的最流行的开源数据库工具之一。Kulkarni 表示，随着企业开始收集越来越多的数据，Postgres 的使用正在复苏，这有助于吸引大量开发人员寻找更有效的实时方式来访问这些数据。Kulkarni 说，开发人员已经下载了超过 10 万次的时间刻度。

时标是基于这样一种想法，即公司正在向数据库中插入越来越多的信息，这些信息通常是按顺序发生的——因此按某种时间序列排序是有意义的。这个想法是，一旦数据在那里，除非有错误，它很少更新，而且它已经是自然有序的，所以除了正常的分区以外，以这种方式对数据进行分区是有意义的。当企业查询数据时，他们可能会在相同的时间间隔内进行查询，并且弄清楚如何将数据分块并使其更容易访问(如在内存和磁盘之间操作)是有好处的。

随着开发人员将这些信息添加到他们的数据库中，这些信息会自动按时间进行分区，他们通过一个抽象层(初创公司称之为“超表”)像访问普通表一样访问这些信息因此，在大多数情况下，开发人员看到的是他们通常会看到的那种界面，时间刻度在幕后操作，以使事情更有效、更快。

Kulkarni 说:“即使在磁盘上，你通常也是在同一时间间隔查询数据——也许是昨天的数据。”“我们让将这部分数据加载到内存中变得很容易。我们设计的数据库针对这种数据进行了优化。你最终可能会有成千上万个分区，我们会把它们藏在[超表]后面，不让用户看到。它给人一种跨越时空的连续表的错觉。我们优化查询并呈现数据。如果你要插入，你只需插入到超表中，我们就会创建新的块。”

像许多在开源软件基础上发展起来的创业公司一样，Kulkarni 希望这家创业公司能够在最后一英里工具中找到一些利基，这将有助于它成为一家功能齐全的企业。像这样的公司面临的最大挑战之一是找到利用其软件专业知识的方法，为需要更专业的部署的企业接管一些最复杂的操作，但希望将这一责任交给拥有工具核心知识的其他人。

当然，正如许多初创公司看到的那样，这也带来了自身的挑战。MongoDB 去年凭借建立在开源软件基础上的业务上市，自上市以来一直有一个非常艰难的开始，自上市以来下跌了 10%。时标也不是唯一一个利用这一趋势的公司，像 InfluxData 这样的初创公司也在筹集资金。当然，时标必须让开发者和企业相信这些工具更好更快。但 Kulkarni 说，证据在于使用，因为它开始吸引新的企业，这些企业已经寻求部署某种比现有技术更快的技术。

“时间序列已经成为增长最快的类别，”库尔卡尼说。“当我们创办这家公司时，我们都在挠自己的痒痒。但是回顾过去的一年，我们碰巧处于这两个主要趋势的交汇点 Postgres 的复兴和时间序列数据的出现。对我们来说，第一年真的是建立社区并获得一些客户的支持。第二年和第三年将真正扩展业务模式和企业部署。”
