Mesosphere为其数据中心操作系统| TechCrunch增加了Kubernetes支持

原文：https://web.archive.org/web/https://techcrunch.com/2017/09/06/mesosphere-says-its-not-bowing-to-kubernetes/?utm_source=shootr&utm_medium=referral

毫无疑问， Kubernetes 是目前容器编排服务的主流。Mesosphere是最早采用容器的公司之一，专注于允许企业在云中运行大数据和分析工作负载，今天宣布，它现在也支持Kubernetes在其【today操作系统平台上运行云中的大数据应用程序。考虑到Mesosphere长期以来一直以马拉松的形式为DC/操作系统和Apache Mesos提供自己的容器编排工具，这一宣布对许多人来说将是一个惊喜。

Kubernetes将作为计划于9月11日发布的DC/OS 1.10版本的一部分进行测试。

今天早上早些时候披露这个故事的信息认为这意味着中间层正在向Kubernetes“低头”。这是我今天早些时候与Mesosphere的联合创始人兼首席执行官Florian Leibert和Mesosphere的CEO彼得·瓜根蒂谈话时强烈否认的事情。他们都强调，这里的想法是给Mesosphere的用户——往往是大型企业——更多的选择。“我们的客户往往是大型公司的基础设施和运营专业人士，他们为一个组织中的数百或数千名开发人员提供服务，”Guagenti说。"对他们来说，他们提供的最重要的东西是选择的自由."

在Leibert看来，提供对Kubernetes作为编排引擎的支持与提供对多种数据服务、持续集成平台和网络工具的支持没有什么不同。Guagenti后来还强调，重要的是要记住，对于Mesosphere的客户来说，该平台不是关于容器的，而是关于部署和管理数据密集型应用程序的。

Leibert还指出，Marathon和Kubernetes有不同的用例，因为Marathon也可以用来运行没有容器技术的遗留应用程序，而Kubernetes显然只专注于容器。“所以对我们来说，支持这两者是很自然的，”莱博特说。“这些技术中有很多真的就像是分层蛋糕。Kubernetes和Mesos可以很好地合作。Kubernetes接管了容器工作流，但它无法处理通常不会在Hadoop等容器上运行的工作流。”

Guagenti还指出，他认为Mesosphere目前是容器领域的领导者，无论是就其用户在生产中运行的容器数量还是就收入而言(尽管该公司遗憾地没有透露任何数字)。

Leibert和Guagenti都强调，Mesosphere将一如既往地继续投资马拉松。

今后，开发人员将能够使用DC/OS直接从DC/OS并在运行其余容器部署的同一基础设施上设置和管理他们基于Kubernetes的容器部署(包括不同版本的Kubernetes)。Mesosphere在这个项目上与Google合作，Mesosphere将为其用户提供一个纯上游版本的Kubernetes，没有任何特定于供应商的更改，以确保没有兼容性问题。

“通过将Kubernetes引入DC/OS，Mesosphere为客户提供了一个强大的平台，用于在您的数据中心和公共云上构建、部署和运行数据丰富的容器化应用程序，”负责Kubernetes和谷歌容器引擎的谷歌产品经理Allan Naim说。“像Kubernetes for containers和tensor flow for machine intelligence这样的项目都在我们的平台上运行，Mesosphere DC/OS和谷歌云平台共同提供了一个引人注目的开放式混合云平台。我们很高兴能继续与Mesosphere和社区合作。”

最终，Mesosphere认为它只是给了客户更多的选择，尽管很明显这是Kubernetes生态系统的又一次胜利，这对Mesosphere来说并不是太大的威胁，因为它早就找到了自己的位置，但对Docker来说更是如此，尽管它启动了容器运动，但Docker有落后的风险。事实上，我不禁认为，Mesosphere的这一举动实际上将使Docker更难找到自己的位置。
