苹果公司正着眼于面部识别来简化照片分享

原文：https://web.archive.org/web/https://techcrunch.com/2015/08/13/apple-photo-sharing-patent/

根据一份新的专利申请(由Apple insider发现)，苹果公司可能会效仿脸书的剧本，寻找自动与被拍照者分享照片的方法。这种努力的驱动力是智能手机用户现在拍摄的大量照片——其中许多照片无疑在手机相机胶卷上无人问津。

与今年夏天推出的脸书时刻应用程序非常相似，它使用面部识别技术来帮助将照片分发给照片中的人，苹果的专利——其标题为“发送数字图像的系统和方法”,于 2014 年 2 月提交，但于本周公布——描述了各种通过将面部与联系人数据关联来简化照片共享的方法，也利用了面部识别技术。

正如 AppleInsider 指出的，苹果已经使用面部识别技术来帮助用户识别和分类他们的照片，比如在它的 Mac 照片应用中。这项专利以此为基础，描述了将联系数据与已识别的面部联系起来的方法，并为用户提供了不同的选择来与照片中的人分享照片——例如通过电子邮件或短信。

该专利涵盖了多种联系和照片共享方法，包括让用户手动将联系信息与面孔相关联，并发布到社交网络作为另一种分发照片的方式。

从文件来看:

面部识别算法可以在数字图像中识别一个或多个人的面部。多种类型的通信可用于数字图像中的不同人。可以呈现用户界面，指示所识别的面部以及相应人的可用通信形式。可以呈现使用每种通信形式可与之通信的总人数的指示。用户可以选择一种或多种通信形式，使得使用所选择的通信形式将数字图像发送给接收者。个人可能已经向服务提供了用于个人面部识别的信息。基于该信息，该服务可以识别该个人在上传的照片中，并将数字图像发送到该个人的用户账户。

值得注意的是，谷歌最近对其自己的照片应用进行了彻底改革，应用机器学习算法对用户的相机胶卷进行自动分类——允许人们使用自然语言查询来搜索他们的照片。当然，让谷歌的算法爬满你的相机胶卷的另一面是，它的基于它可以从你的照片中收集的情报，吸取各种个人数据。

相比之下，苹果公司一直在大声疾呼其商业模式不依赖于用户数据的积累——强调这是与谷歌和脸书等广告驱动的商业模式的区别。

今年夏天早些时候，苹果在 WWDC 发布了 Siri 语音助手的更新版本，名为 Proactive，提供了一些类似 Google Now 的预测功能。然而，与这项服务相关的数据处理是在用户的设备上本地完成的——而不是通过云处理——在苹果和用户的个人数据之间划清了界限，以加强隐私。

回到面部识别照片共享专利，苹果指出，将面部识别信息与个人联系方式联系起来的相关数据库可以存储在本地或云中:

面部的面部识别数据可以与面部的姓名和地址相关联地存储。例如，可以创建或更新数据库中具有面部识别数据字段、姓名字段和地址字段的记录。可以存在更多或更少的字段。数据库可以存储在客户机 110 或 112 上，或者存储在服务器 118 上。在一些示例实施例中，客户端机器 110 或 112 访问服务器 118，以检索用于客户端机器 110 或 112 上的面部识别的面部识别数据。在其他示例实施例中，客户端机器 110 或 112 访问服务器 118 以在服务器 118 上传输用于面部识别的图像。

如果 Cupertino 确实为照片分发实施了面部识别系统，鉴于其强烈的支持隐私的轨迹，很难想象它能够实施基于云的处理或存储这种敏感的用户信息。本地处理将更符合其当前支持隐私的立场。

也就是说，这项专利涵盖了所有基础——例如，描述了一个场景，涉及通过“中央服务器系统”将之前训练的面部识别数据(来自桌面图片组织工具，比如可能是苹果自己的 Mac Photos 工具)同步到移动设备:

一旦同步，移动设备就可以利用预训练的面部识别数据来识别新拍摄的照片中的面部。随后，根据本文讨论的示例，与预训练的面部识别数据相关联的联系信息可以用于与已知用户共享新捕捉的照片。

在描述的另一种方法中，如果用户选择自动上传，系统甚至可以自动将照片分发给照片中识别的人。

在推动服务于便利之神的过程中，云变得越来越重要，它允许在多种设备上轻松访问数字内容。但隐私问题也是如此。因此，苹果将在谈判如何将更多人工智能和机器学习功能应用到其软件中时做出一些关键选择。

当云同步和自动上传在后台进行时，锁定用户隐私肯定是一个棘手的问题——正如一位谷歌照片的用户最近发现的那样。因此，如果苹果真的决定将这项专利转化为具体的消费产品，那么看看它如何实现面部识别照片分发功能将是一件有趣的事情。
