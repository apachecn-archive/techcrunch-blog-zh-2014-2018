# 参议院希望通过网飞、Spotify 等发布紧急警报。

紧急警报响起，试图让你知道即将到来的坏消息——导弹、海啸或其他可怕的东西。你的电话开始喊…但是在楼下。如果你在看有线电视，电视上会弹出一个警告提示……但你的眼睛却一直盯着网飞、Hulu 或其他在线流媒体服务。

随着这些服务越来越多地占据我们的屏幕时间，它们应该准备好广播这些警告吗？

夏威夷州和南达科他州的参议员认为是这样的，他们刚刚提出了一项法案，将“探索”向“在线流媒体服务，如网飞和 Spotify”广播警报，以及对紧急警报系统的其他改变。

“夏威夷？这个州的紧急警报系统不是有一个[非常公开的假警报](https://web.archive.org/web/20221025222233/https://techcrunch.com/2018/01/30/details-and-solutions-emerge-for-missile-threat-false-alarm-in-hawaii/)吗？”

没错。但似乎在调查出了什么问题时，该州发现了现有的老化警报系统中大量长期存在的缺点。

**该法案涉及的其他一些事情:**

*   许多手机用户目前可以禁用联邦警报；他们想摆脱这种选择
*   建立一个更好的系统来报告假警报并弄清楚发生了什么
*   更新系统以更好地防止假警报，并在真的发生时更好地撤销它们

向网飞等发送紧急警报的想法。在这一点上似乎有点显而易见——见鬼，我在 2011 年的 TechCrunch 上仔细考虑过这个问题[,甚至在当时也似乎有点显而易见。](https://web.archive.org/web/20221025222233/https://techcrunch.com/2011/11/09/streaming-emergency-alert-system/)

话虽如此，我仍有当时同样的犹豫。在最近的假警报和随之而来的恐慌之后，很明显，从安全角度来看，任何这样的系统都需要坚如磐石——一个遗漏的漏洞或漏洞，半个国家都在为不存在的来袭导弹而恐慌，而他们只想看橙色是新的黑色。如果做得对，这似乎是一个合理的想法。
