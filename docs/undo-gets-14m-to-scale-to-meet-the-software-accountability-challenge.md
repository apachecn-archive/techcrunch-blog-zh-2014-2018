# Undo 获得 1400 万美元来应对软件责任挑战 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/07/16/undo-gets-14m-to-scale-to-meet-the-software-accountability-challenge/>

调试工具领域的长期参与者 Undo T1 提供程序执行捕获和重放技术，以帮助其他人诊断软件故障，该公司已经完成了由总部位于英国剑桥的科技和医疗保健公司建筑商[剑桥创新资本](https://web.archive.org/web/20221208034100/https://techcrunch.com/2013/10/11/cambridge-innovation-capital/)牵头的 1400 万美元的 B 轮融资。

这家成立于 2005 年的初创公司最初是自举式的(从创始人格雷格·劳(Greg Law)的花园小屋开始)，已经走过了漫长的道路，现在已经有 30 多个付费客户，他们称之为“记录、倒带和重放”调试技术，包括 SAP HANA、Mentor Graphics、Cadence 和 Micro Focus 等。

一段简短的历史:2012 年，劳辞去工作，开始全职运营 Undo，筹集了少量天使投资，然后在 2014 年获得了 125 万澳元的种子投资，随后在 2016 年获得了 330 万美元的首轮融资。

B 轮新投资者包括日本风险投资基金环球大脑公司；和专注于英国的 Parkwalk Advisors，而所有 Undo 的现有投资者团体也参与其中——包括 Rockspring 马特莱；彼得·迈克尔爵士(Quantel、Classic FM、加州彼得·迈克尔酒庄创始人)；剑桥天使集团和 Jaan Tallinn(Skype 和 Kazaa 的联合创始人)。

B 系列将用于扩大 Undo 的软件开发团队，加速产品开发，并扩大其美国业务。Undo 称其目前最好的市场是电子设计自动化(EDA)；数据库制造商/数据管理；和网络。

“这笔资金将用于显著提高性能，作为 Undo 的*永远在线录制*愿景的一部分，同时还将加速我们的产品路线图，并将技术扩展到编译代码以外，以便它可以用于 Java 和其他基于 VM 的语言，”它告诉我们。

“我们的主要竞争对手是现状——不与时俱进的工程组织。老式的调试技术(例如 printf、日志、核心转储分析)已经存在了几十年。2000 年的主题是静态分析。2010 年是动态分析，2020 年将是通过捕获和重放技术‘实时’捕获软件故障。”

Undo 辩称，其 Live Recorder 技术提供了“一种在开发和生产过程中诊断软件故障的全新方式”——辩称其方法优于 printf、日志记录、核心转储分析等传统调试技术，这些技术“用途广泛，提供的信息有限”，同时表示静态和动态分析“很深入，但只能查看特定的错误实例”——而它声称其技术“可以捕获整个范围内的故障实例，因此填补了其他人尚未填补的空白”。

这家英国公司还认为，鉴于日益复杂和日益自主的软件有变得不可问责的风险，如果它在人们不知道如何和为什么的情况下做出决策，它的方法也有越来越大的机会。因此，撤销的更广泛的愿景不仅仅是更快地修复错误，而是满足软件制造商日益增长的需求，即能够阐明——并说明——他们的程序在任何给定时刻都在做什么。

Law 说:“从长远来看，这是迈向软件责任的旅程。”。“软件责任是一个相当广泛的东西——它实际上意味着能够确切地知道一些软件在运行时做了什么。今天，这就是程序员对他们的程序做了什么的理解。但实际上，需要理解软件的不仅仅是程序员——尤其是当我们进入信息革命的第二章时，计算机开始做出影响我们生活和生计的决定。我指的是社交媒体、脸书和其他东西，西方民主！软件行为背后的问责能力将变得非常重要。这是我们正在进行的进步之旅。

“所以问题是这个软件实际上做了什么？随着我们的成长和时间的推移，我们将在越来越大的背景下回答这个问题。”
