亚马逊为任何电视提供商或流媒体服务提供了一种构建基于 Alexa 的视频应用的方法 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2017/06/08/amazon-introduces-a-way-for-any-tv-provider-or-streaming-service-to-build-alexa-powered-video-apps/

Alexa 已经可以控制你的 Fire TV 来帮助你启动网飞或找到你最喜欢的节目，等等，但今天亚马逊宣布为开发者提供一个新工具，允许任何有线电视和卫星电视公司、流媒体服务或其他内容提供商在其应用程序中添加相同类型的语音控制。有了新的视频技能 API ，开发者可以让 Echo 和 Alexa 设备所有者通过说话来更换频道、寻找节目或电影、跨流派搜索、播放或暂停内容等等。

例如，你可以对你的一个流媒体视频应用说，“Alexa，找喜剧，”或者“Alexa，播放《摩登家庭》的剧集。”有线电视提供商可以提供应用程序，让你不用遥控器就能换频道，比如说“Alexa，调到 HBO。”

当然，这并不意味着你马上就能使用 Echo speaker 免提使用 Hulu、HBO 或网飞等热门应用。开发者实际上必须首先更新他们的应用程序来利用 API。不是所有人都会。

但亚马逊表示，一些公司已经推出了使用这种新 API 的应用，包括 Dish、NetGem 和英国的 YouView。

如果有什么不同的话，这种最初的牵引力似乎表明，付费电视提供商和电信行业的那些人渴望采用 API，以阻止他们在流媒体服务方面的损失。

Dish 在五月宣布整合 Alexa，成为美国第一家允许用户通过语音控制 DVR 的电视提供商。

拥有 Dish Wally 或 Hopper 接收器的客户现在可以配置他们的 Echo、Echo Dot 或亚马逊 Tap 扬声器，以便与 Dish 自己的设备配合工作。经过简单的设置过程后，看电视就变成了免提体验。

亚马逊告诉我们，除了最初的三家，它现在正积极与其他许多视频提供商合作，但不能分享细节，因为他们的应用程序和升级仍在开发中。

除了能够控制视频应用程序，视频技能 API 还可以导入提供商自己的目录，以帮助提高 Alexa 响应的准确性，而且它的工作无需客户首先调用特定技能。也就是说，客户可以给出更简单的指令，比如“Alexa，播放‘海边的曼彻斯特’”，而不是指示 Alexa“要求 Dish”播放一个标题，进行搜索或更换频道。"
