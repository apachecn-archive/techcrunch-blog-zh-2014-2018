Kai Brunner 是持续交付企业软件的首席设计师

Electric Cloud

.

墨菲定律规定:“任何可能出错的事情都会出错。”对于我们中任何一个靠劳动谋生的人来说，事情出错可能意味着:“任何可以自动化的东西都将自动化。”

我们用劳动或技能来换取报酬，这无疑让我们在永远被需要的观念中寻求安全感。然而，时间表明，我们对效率的独创性使我们摆脱了各种形式的重复性任务。能源和机械化把我们从田间带到了工厂附近的城市，在过去的几十年里，我们来到了显示器后面的办公室。工程师们甚至通过自动化他们至关重要的事情来使自己变得过时:思考和制造东西。

从自动化软件交付，到自我修改代码，再到自我组装机器人，我们还会被需要吗？我们通过创造性和想象力设计了我们的环境和每一个与我们互动的物体。这当然不能自动化，对吗？在这个职能上，我们是不可能被取代的。作为 DevOps automation 的企业软件设计师，我大胆地想知道，自动化能做我做的事情吗？我能设计我自己的过时吗？

在他的文章为什么网页设计已死中，拉丁美洲设计师 Sergio Nouvel 从“高质量模板、成熟的设计模式、自动化和人工智能(…)的演变中解释了网页设计的死亡。”按照我的理解，他的结论保证了体验设计的继续存在，因为内聚性地定义体验来自于深思熟虑的概念化。我同意他的观点，因为我们似乎无法自动化想象我们想要经历的过程。

回到自动化和人工智能，有两个最近的例子证实了我们正在走向一个自动化层，以消除设计工作的一个方面:视觉布局。结构元素、支撑元素、它们的定位、它们的尺寸、它们的排列、它们的层次、它们的背景和它们的调色板的令人愉快的和谐，都是熟练的设计师带来人类无形的审美的区别。

还是反过来？也许不是我们创造了美学，而是进化让我们习惯于模仿自然的模式:对称、比例、形状和颜色。如果这是真的，那么通过定义的参数，AI 可以学习这些模式，并像设计师一样，探索设计的可能性，直到用户的反应确认了一个令人满意的平衡点。

旧金山初创公司 The Grid ，提供“自我设计的网站”，其人工智能驱动的布局通过分析媒体，甚至调整调色板和排版来组织和发展内容的显示。基本上，他们的人工智能正在做出视觉设计师的决定。然而，一个人从目的、激情、奉献和教育中学会做决定。网格的布局引擎减轻了很大一部分设计工作，但结果的主要成分仍然是人:享受和情感。

由商业智能提供信息的 UX 设计并没有超出人工智能和自动化的范围。

今年 5 月，TechCrunch Disrupt New York 2016 黑客马拉松的最高荣誉授予了 AlexaSite ，这是一个使用语音命令更新网站设计的解决方案。他们最主要的使用案例是网页设计者在与客户会面时使用即时反馈来修改布局。我立刻想知道，如果这不是一个专业的设计师在说话呢？如果是一个愿意花时间看模板并指导修改的人呢？

迭代过程不需要任何技术技能，很快结果就会满足审美感受。与许多其他行业一样，技术将使专业技能大众化，省去中间人，并使非专业入门者发展最低限度的专业技能以产生令人满意的结果成为可能。

在这两种情况下，人工智能和自动化已经揭开了生产可交付设计的神秘面纱。从构思到交付的技巧已经让视觉设计师不再是这个过程的看门人和所有者。这正是我们回到最初不安全感的地方:害怕不再被需要。

考虑到我们的立足点在于做出太复杂的决策，以至于算法无法从我们手中夺走这些决策，自动化无法跨越的前沿是体验设计需要团队协作、创造力、财务要求、公司文化、工程能力和市场目标的领域。人工智能和自动化不可能承担所有这些变量——但它们能走多远？

由商业智能提供信息的 UX 设计并没有超出人工智能和自动化的范围。我们可能认为明显在我们的创造性合成的边界之内的部分是交互设计，其中来自反馈和验证的架构、框架和迭代塑造了不断发展的用户体验。

早在 1999 年，我和一位同事进行了一次对话，我想象了一个有机 UI。我们定义了一个用户界面，它随着个人用户知识的增长而改变。用户将享受与他们不断增长的能力相对应的定制产品体验。可以说，产品将具有 DNA，这将使它能够在进化框架的参数内变化。这难道不是 UX 的做法今天把我们带到的地方吗？

现在，在与产品管理和工程师的合作下，我正在为应用程序发布自动化设计一个企业软件解决方案，帮助 DevOps 团队致力于他们软件的持续交付。由于团队和我一直在为自动化而设计，我经常展望未来，想知道信息架构、工作流和交互设计的哪一部分可以自动化，以产生与我们迄今为止所取得的成果相当的结果。

又一次提到了 DNA。如果我们可以参数化我们开发的工作流、UI 模式和可视化语言来定义用户体验，会怎么样？设计周期中的劳动存在于探索工作流和模式的许多迭代中，这些工作流和模式导致验证我们的用户需求得到满足。在已定义的工作流和 UI 模式中，人工智能可以通过迭代，在很短的时间内提供多个可行的选项。接下来，它会将选项推给用户测试人员，记录行为，并通过分析确定哪个想法是最好的。

工程团队移交线框和规范的后续工作将被完全优化到实现引擎中，以产生完美和准确的 UI 布局和行为。整个质量保证过程将通过满足最低验收标准而得到优化，从而完全消除人为错误。这条没有摩擦的道路可能会促进团队之间的和平与和谐，这是以前做梦也想不到的。

软件工程中的某些角色正在走向过时。

UX 设计周期中的自动化概念试图模仿软件开发和 IT 团队已经在做的事情，通过创建自动化管道来更有效地交付版本。容易出现人为错误的重复性劳动和密集型任务被分解为一系列阶段的可重复流程，从开发、测试到生产，再到用户手中，从而确认可以自动化的内容确实实现了自动化。

因此，软件工程中的某些角色正走向过时，因为最佳实践淘汰了一些人，而自动化却让表演继续下去。但是所有行业都是如此，追溯到第一台蒸汽机的出现，一直到装配线的机械化。在每一种情况下，自动化都不是劳动者的工作。不，它发生在他们身上。随着软件发布的自动化，具有讽刺意味的是，工程师是想象它成为现实的人。当设计师想象下一步的构造时，他们的淘汰是自己造成的，这是唯一合适的。

我预测，一旦我们有了设计软件的自动化算法任务，就可以腾出时间来想象新的解决方案和体验，正如我们目前看到的 VR 的复兴和 360 视频内容的出现。自动化为设计的启发性方面腾出了时间，以发展为更深层次的社会联系、更有效的合作和更广泛的创造力创造体验。

然而，首先，消除算法设计任务有一个实用的动机:速度。永远的竞争优势。然而，通过牺牲质量来加快执行速度，往往是错误的。由于缺乏自动化，我们通过删除确保交付高质量产品的更好结果的关键步骤，而不充分地实现速度。

我的前老板经常说:“浪费是罪恶的”，我把它解释为浪费时间。努力白费了。对话被浪费了。天赋被浪费了。市场机会被浪费了。投资浪费了。我甚至可以说，生命本身就浪费在低效的生产中。

在软件中产生卓越的用户体验的情况下，当速度和质量的向量处于拔河状态时，自动化可以结束战争，并使两个向量朝着同一方向移动。这位工程师兼教授 W. Edwards Deming 在接受日本战后制造业重建委员会的委托时告诫道:“不要再依赖大规模检验来保证质量了。相反，首先要改进流程和将质量融入产品

在我们致力于追求持续软件交付的现代软件开发组织中，戴明的话很可能是建立设计过程的灯塔，在这一过程中，自动化消除了人类的检查，内置质量将直接来自与一组参数相关的人工智能渲染设计解决方案。

作为一名企业软件的设计者，我不担心我的劳动被淘汰。我欢迎它。我寻求使它成为可能。我渴望人工智能和自动化来解放我，以便我可以有精神上的可用性和时间来规划和设计接下来的体验。
