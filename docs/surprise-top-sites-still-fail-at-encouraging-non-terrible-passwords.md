# 惊喜！顶级网站在鼓励不可怕的密码方面仍然失败

> 原文：<https://web.archive.org/web/https://techcrunch.com/2018/07/19/surprise-top-sites-still-fail-at-encouraging-non-terrible-passwords/>

你可能会认为亚马逊、Reddit、维基百科和其他非常受欢迎的网站现在会告诉你“password1”或“hunter2”是一个糟糕的密码——非常糟糕。但是他们没有。一项研究项目在过去 11 年中对顶级网站及其密码习惯进行了跟踪，该项目显示，大多数网站仅提供基本的密码限制，对用户几乎没有帮助。

普利茅斯大学的 Steven Furnell 在 2007 年首次对网站的密码做法进行了调查，在 2011 年和 2014 年重复了这一过程——本周又重复了一次。他的结论？

> 发现 2018 年的整体情况与 2007 年大体相似，多少有些令人失望。在这中间的几年里，关于密码的缺陷和我们使用密码的方式已经写了很多，但很少鼓励或迫使我们遵循正确的道路。

尽管该大学的文章指出谷歌、微软和雅虎的密码实践最好，而亚马逊、Reddit 和维基百科最差，但它委婉地拒绝透露细节。幸运的是，我为自己获得了这份报纸，并准备点名羞辱。

十大独特的英语网站(由 Alexa 测量；这些年来阵容有些变化)被评估:谷歌，脸书，维基百科，Reddit，雅虎，亚马逊，Twitter，Instagram，微软 Live 和网飞。

最大的失败无疑是亚马逊，它将真正不充分的密码控制与极其有价值的个性化服务结合在一起。维基百科和 Reddit 的限制较少，但都不保护如此重要的数据；恶意用户访问亚马逊账户的危险要大得多。

亚马逊几乎接受了 Furnell 扔给它的每一个密码，包括重复的用户名、用户自己的名字，当然，还有空前经典的“密码”(网飞和 Reddit 也采用了“密码”，尽管维基百科没有。另一方面，维基百科接受单字符密码，如“b .”)

即使是有限制的网站，比如要求多种字符类型或拒绝常用密码，也很少解释。开始时没有任何反馈，创建帐户的用户可能会输入密码，但被告知密码必须更长…然后，再次，它不能有某个单词(如用户的姓氏)…然后，再次，它必须包括特殊字符。而且有些网站在你注册的时候和你设置新的时候有不同的要求！

为什么不一开始就把一切都计划好呢？就此而言，为什么不解释一下背后的原因呢？制作一个小小的信息框说“我们需要 X 因为 y”是微不足道的，但是几乎没有一个顶级网站这样做。

这份沉闷的报告中的一点亮点是，双因素认证——可以说比一个好的密码更重要——事实上正在取得进展，密码政策的一些最糟糕的违法者(看看你，亚马逊)允许这一点。现在，他们只需将它从短信转移到一个安全的认证应用程序上。

最后一句话和过去十年一样:

> 与该研究的早期版本和其他参考文献一样，这里的基本论点是提供面向用户的安全性，并配以相应的支持。密码是一个很好的例子，因为我们知道许多人不善于使用它们。然而，这个教训继续被忽视，我们继续批评这种方法，而不是责怪用户。

双因素是一个开始，但是:

> 用户可能需要更多的鼓励——或者说义务——来使用它们。否则，就像密码本身一样，它们将提供潜在的保护，但在实践中却做不到这一点。

换句话说，停止谈论密码有多糟糕，做点什么吧！
