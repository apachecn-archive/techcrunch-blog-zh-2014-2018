Kubernetes和Cloud Foundry越来越紧密TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2018/04/20/kubernetes-and-cloud-foundry-grow-closer/amp/

容器正在吞噬软件世界Kubernetes是容器之王。所以如果你正在做任何一个大型的软件项目，尤其是在企业中，你迟早会碰到它。Cloud Foundry本周在波士顿举办了半年一次的开发者大会，这是一个有趣的例子。

在企业开发人员的世界之外，Cloud Foundry仍然是一个不为人知的实体，尽管至少一半的财富500强公司都有它的用户(尽管在创业界，它几乎没有吸引力)。如果你不熟悉Cloud Foundry，你可以认为它有点类似于 Heroku ，但它是一个开源项目，拥有一个大型商业生态系统，能够在任何云或内部安装上大规模运行。开发人员编写他们的代码(遵循的十二因素方法)，定义它需要运行什么，Cloud Foundry处理所有的底层基础设施，并在必要时进行扩展。理想情况下，这让开发人员不必考虑他们的应用程序将在哪里运行，并让他们更有效地工作。

为了实现这一切，Cloud Foundry Foundation很早就在容器上下了赌注，甚至在Docker出现之前。由于Kubernetes当时还不存在，参与Cloud Foundry的各个公司联合起来构建自己的容器编排系统，该系统至今仍支撑着大部分服务。然而，随着它的起飞，为Kubernetes提供支持的压力在云铸造生态系统内部不断增长。去年，基金会宣布了在这个方向上的第一个重大举措，推出了基于Kubernetes的用于管理容器的容器运行时，它位于现有应用程序运行时的旁边。这样，开发人员就可以使用Cloud Foundry来运行和管理他们新的(和现有的)单一应用程序，并与他们开发的新服务并行运行。

但是还记得Cloud Foundry是如何为应用程序运行时使用自己的容器服务的吗？既然Kubernetes(及其生态系统中的各种其他项目)已经成为处理容器的默认选项，那么就没有理由这样做了。因此，现在有一个旨在淘汰旧的容器管理系统并用Kubernetes取而代之的云铸造项目也就不足为奇了。毕竟，容器管理并不是Cloud Foundry与众不同的地方。相反，这是开发人员的体验——归根结底，Cloud Foundry的全部意义在于，开发人员不应该关心基础设施的内部管道。

云铸造生态系统如何拥抱Kubernetes还有另一个方面。因为Cloud Foundry也只是软件，所以没有什么可以阻止你在Kubernetes上运行它。因此，包括SUSE和IBM在内的一些最大的云代工厂商正在这么做也就不足为奇了。

SUSE云应用平台是一个经过认证的Cloud Foundry发行版，可以在任何公共云Kubernetes基础设施上运行，包括微软Azure容器服务。正如SUSE团队告诉我的那样，这意味着它不仅更易于部署，而且运行时占用的资源也少得多。

类似地，IBM现在为其客户提供基于Kubernetes的Cloud Foundry，尽管它现在只称这是一个实验产品。IBM的云开发服务总经理Don Boulia强调，IBM的客户大多在寻找在隔离的环境中运行工作负载的方法，而不是与其他IBM客户共享。

Boulia还强调，对于大多数客户来说，这不是Kubernetes与Cloud Foundry的问题。对于他的大多数客户来说，使用Kubernetes本身就是将他们现有的应用程序迁移到云中。对于新的应用程序，这些客户会选择运行Cloud Foundry。

SUSE团队也强调了这一点。SUSE看到的一种模式是，潜在客户带着建立容器环境的想法来找it部门，然后在交谈过程中，决定也实施Cloud Foundry。

事实上，本周活动的主旨是Kubernetes和Cloud Foundry是互补的技术。这也是谷歌负责容器引擎和Kubernetes的工程总监陈·戈德堡(Chen Goldberg)在活动的小组讨论中强调的。

Cloud Foundry Foundation和Cloud Native Computing Foundation(CNCF )( Kubernetes之家)都隶属于Linux基金会。他们对自己的社区采取了有些不同的方法，Cloud Foundry对企业用户的压力远远大于CNCF。这里可能有一些政治因素在起作用，但在大多数情况下，这两个组织似乎足够友好——而且它们确实有一些共同的成员。“我们是CNCF的一部分，也是云铸造基金会的一部分，”Pivotal首席执行官Rob Mee告诉我们自己的Ron Miller。“这些社区越来越多地相互分享技术，共同发展。不完全独立，也没有竞争力。非常复杂和微妙。CNCF和Cloud Foundry是一个更大的生态系统的一部分，具有互补和融合的技术。”

我们可能会看到CNCF和云代工厂之间更多的这种技术共享，甚至合作。毕竟，CNCF是许多构建云原生应用程序的有趣项目的发源地，这些应用程序在Cloud Foundry中也有相当多的用例。
