随着音乐迷走向移动，音乐节也在效仿。美国的孩子们一手拿着 Natty Lite，一手拿着打开本地节日应用程序的手机，摇着 T-Swift。

时间表、场地地图、艺术家和供应商都掌握在参加派对的人手中。节日甚至还设计了其他功能，专门在活动实际开始前几周吸引大批热切的活动观众、铁杆粉丝和冷静的旁观者。

想象一下，如果这些应用程序(或一个新的应用程序，就此而言)在持续吸引人们方面将移动体验推进了一步。

考虑一下这个场景:你和你最好的朋友在一家咖啡馆里，背景音乐吸引了你们的注意力。甚至没有说一句话，你们看着对方，坚定地同意你喜欢这首歌。

通常，你们中的一个人会启动 2-3 个不同的应用程序来找出艺术家是谁，查找传记信息，搜索更多他们的音乐，也许会问你的社交网络谁熟悉这位艺术家，或者发布一些参考背景的东西。

如果一个应用程序，在听到歌曲的前几秒钟，识别出你喜欢这种类型的音乐，自己处理所有的搜索和分享活动，然后简单地提示你个性化的行动号召，会怎么样？

> 这不是火箭科学。这是现代的计算能力。

它可以提醒你，未来几周该艺术家的表演将在附近的一个地点举行，你可以在那里购买门票。它还建议航班，酒店，餐馆和艺术家的商品。通知驱动的智能手表也可以在这种互动中发挥作用。

简而言之，它就像你最好的朋友一样了解你(甚至可能更了解)，并且已经找到了与你交流和指导你下一步行动的最有效方式。

## 人性化体验设计

构建这种无缝体验所需的 SDK、API、传感器和所有其他技术已经存在。 [StubHub 最近与 Spotify](https://web.archive.org/web/20230319181433/https://techcrunch.com/2015/04/02/seatgeek-raises-62-million-in-a-series-c-led-by-technology-crossover-ventures/) 的合作允许整合一个人的音乐库和即将举行的音乐会和活动。随着用户建立他们的图书馆，StubHub 了解他们的收听行为，他们会得到更多定制的推荐。

不仅仅是移动设备成为人们进入互联世界的主要途径，还有各种各样的技术让移动数字体验更加“人性化”

连接无处不在已经不是新闻了。任何开发和设计移动应用程序的人，无论是大公司还是小公司，都应该利用这种技术来创建收集和利用行为信息的客户接触点。

抛开历史教训不谈，这种设计移动体验的方法是基于对世界的人文主义愿景，在这个世界中，一切都以人类为中心，我们不断进化的理性应该如何塑造我们的体验。

通过使用可测量的行为数据、经验分析、动态心智模型和理性决策方法，设计良好的移动体验可以解决问题，缓解和改善人类体验。

## 技术巨头铺平了道路

考虑一下 Facebook Messenger 应用程序可能会向 的方向发展。通过为开发者发布 SDK 和集成第三方工具，这家科技巨头正在建立一个丰富数据源的生态系统，以进一步定义和预测人们在任何给定的移动时刻分享什么样的内容。

一种类似 Siri 的体验的可能性并不遥远，它更像一个个人助理，在许多情况下预测你需要发送的信息和内容。

> 想象一下，如果这些应用程序在持续吸引人们方面将移动体验推进了一步。

除了整合到 messenger 应用程序中的工具和内容之外，他们的合作伙伴可能只能有限地访问人们的总体使用情况，但脸书本身可能会完全了解所有这些数据。

脸书可以将这些信息与它所知道的关于某人社交活动和个人资料的所有其他信息进行分层，设计一个消息应用程序，该应用程序可以独自完成管理、创建甚至概念化可共享内容的工作。谈谈如何从杂乱中抽身？

在谷歌如何重新设计其 Now on Tap 服务的过程中，[正在发挥核心作用。基于 Android 的手机现在更智能、更有效地计算出用户如何以更少的步骤完成他们的预定任务。](https://web.archive.org/web/20230319181433/https://techcrunch.com/2015/05/28/google-now-on-tap)

通过将来自 Fandango、OpenTable 和 IMDb 等其他应用的信息拼凑在一起，用户可以完成他们通常在几个应用中执行的多项任务。这些机器学习和深度神经网络的努力使数字体验“人性化”。

IBM 也在使用类似的合作模式构建计算平台。沃森健康云正在从超过 7 亿名 iPhone 用户那里收集数据，这些用户的移动设备正在从其他个人电子设备收集数据，如健身追踪器、联网医疗设备、可植入设备和其他传感器。

有消息称，脸书也在增加其人工智能驱动的产品，不同平台和设备的用户交互和体验可能会更加人性化。

这不是火箭科学。这是现代的计算能力。内容分享和消费可以变得更加个性化，智能地响应和预测每个人的情绪、思想甚至身体状况。

听起来像是独家科学俱乐部？不再是了。4 月初，亚马逊宣布，它正在向该领域的新软件开发人员提供其人工智能引擎，称为亚马逊机器学习。

它可以创建的预测模型可以解决广泛的问题，从检测欺诈性金融交易到改善客户体验。Amazon 将发布的这些 API 和向导也是可扩展的。

## 一个同样智能和“人性化”的移动世界

原则上，人性化设计是围绕人们在不同背景下的情感、身体、社会和认知行为而构建的。随着脸书伸出双臂，进入其不断增长的合作伙伴基础的数据库，他们可以利用的上下文数据量只会继续呈指数级增长。

Gartner 在最近的一份报告中概述了许多这些[技术，指出由于连接无处不在，公司可以收集关于其客户的实时上下文数据。下一步是使用一个人性化的框架将所有移动的部分拼凑在一起。](https://web.archive.org/web/20230319181433/http://www.gartner.com/newsroom/id/2867917)

以上面的音乐节应用程序为例，人性化设计将把人们体验中更好的元素融入到应用程序中，不仅收集已经可用的信息，还以一种方式将这些信息串联起来，为享受音乐的更大体验添加更多意义。

听起来很简单？那么，为什么没有更多足够智能的应用程序来预测我们想去哪里吃饭、购物、参观和娱乐，而不需要用户一遍又一遍地进行相同的信息收集交互呢？

这里的挑战不是技术的稀缺或缺乏。恰恰相反:开发者、公司和其他人应该将这些进步建立在一个更深刻的信念上，即技术应该为我们人类服务——为我们的幸福、生产力和福祉服务——而不是相反。

人性化的技术肯定会带来感觉更自然的数字交互，更有预见性或预测性，并对我们复杂思维的新信息做出实时反应。
