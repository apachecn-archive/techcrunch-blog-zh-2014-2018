Red Hat为Enterprise Linux和OpenShift带来Docker支持

Red Hat和 Docker.io 今天宣布扩大合作，将Docker的容器技术引入Red Hat的Enterprise Linux high-touchbeta program及其 OpenShift 平台即服务产品。

Docker 的最初发布是在大约一年前，但在这么短的时间内，它已经迅速成为一个非常受欢迎的工具，供那些希望使用软件容器而不是更传统的虚拟化技术的开发人员使用。Docker.io是指导Docker项目的商业实体。

Red Hat Enterprise Linux 7目前处于测试阶段，使用Docker作为其主要支持的容器格式。Docker还宣布了一个新的JumpStart计划，用于组织创建基于Red Hat Enterprise Linux和OpenShift的试点计划。该计划将为Docker Registry提供培训和安装支持，以及Docker商业支持。

“RedHat在开发、投资和培育创新技术方面有着悠久的历史，包括Linux Containers，这是一种开源方法，多年来，RedHat做出了许多根本性的贡献，”Red Hat的执行副总裁兼首席技术官Brian Stevens在今天 的一份声明中表示。“Docker技术有助于消除企业采用容器所面临的障碍——易用性、应用程序打包和基础设施集成——这让我们非常兴奋。”

这并不是红帽和Docker第一次合作。正如两个组织去年宣布的一样，他们也在为Red Hat的开源Fedora项目Linux发行版打包Docker，以及在文件系统依赖和容器供应方面进行合作。上个月，红帽公司宣布扩展其应用程序认证计划，将容器化应用程序纳入其中。该公司当时表示，这确保了使用Red Hat Enterprise Linux构建的容器化应用程序可以跨认证的容器主机工作。
