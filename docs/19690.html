衡量编程语言受欢迎程度到底是怎么回事？TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2018/09/30/what-the-heck-is-going-on-with-measures-of-programming-language-popularity/?utm_source=dlvr.it&utm_medium=twitter

我今天看了 TIOBE 指数，就像我经常做的那样，就像我认识的大多数软件专家经常做的那样。它旨在衡量世界编程语言的受欢迎程度，它的受欢迎程度随时间变化的图表讲述了一个简单的故事:Java 和 C 是，而且自古以来，在某种程度上一直是语言的共同国王。

但是等等。没那么快。其竞争对手“ PYPL 指数”(编程语言的流行程度)称 Python 和 Java 并驾齐驱，而 C(令人惊讶的是，它与 C++被归为一类)在列表中垫底。这是怎么回事？

正在发生的事情是，这两个指数有非常不同的方法…虽然他们的方法的共同点是都非常值得怀疑，如果目标是衡量编程语言的受欢迎程度。TIOBE 测量搜索引擎点击量的绝对。PYPL 测量了语言教程被谷歌搜索的频率。

两者都是糟糕的措施。我们可以预期在线资源的可用性是一个非常滞后的指标；一种曾经占主导地位的死亡语言很可能仍然有数百万专门针对它的残余网页、僵尸网站和多年无人阅读的博客帖子。教程搜索的频率将非常偏向于大量教授给学生的语言。对于从业者实际使用的语言来说，这并不是一个有意义的衡量标准。

当你仔细观察这些数字时，会发现很多奇怪的异常现象。根据 TIOBE 的说法，last C 在五个月内从其历史最低评级变为年度编程语言。我相信 C 语言已经在嵌入式系统中复兴了。但我也可以很容易地想象这是一个非常不完美的措施的产物。

然而，在这两项指标中，更明显的异常是 Objective-C 和 Swift 的相对表现，这两种语言用于编写本地 iOS 应用程序。我当然可以相信，结合起来看，他们最近看到了跨平台替代品(如 Xamarin 和 React Native)的受欢迎程度下降。但是我很难相信，在苹果推出 Swift 四年后——在我看来，客观上是一种更优越的语言——Objective-C 仍然更受欢迎/使用更广泛。在我的日工作中，我处理了许多 iOS/tvOS/watchOS 应用程序，并采访了许多 iOS 开发者。找到一个还没有从 Objective-C 转到 Swift 的人是极其罕见的。

但是，嘿，轶事不是数据，对不对？如果唯一可用的衡量标准与我自己的个人经验相冲突，我可能会得出结论，后者受到了选择偏差的影响。我非常愿意这样做…

…除此之外，还有另一种衡量编程语言受欢迎程度的方法。我指的是 GitHub 平台上十五种最流行的编程语言的年度报告。这些数字基本上完全符合我自己的经验…而且它们与 TIOBE 和 PYPL 的说法完全不相干。

根据 GitHub 的 2016 和 2017 报道，目前世界上最流行的编程语言是 Javascript。Python 次之。Java 排名第三，Ruby 紧随其后。这与 TIOBE 形成鲜明对比，它有 Java 和 C，然后是巨大的差距，然后是 Python 和 C++ (Javascript 是第八)——也与 PYPL 形成鲜明对比，它声称顺序是:Python 和 Java，巨大的差距，然后是 Javascript 和 PHP。

显然 GitHub 的数字也不能代表整个领域；他们的样本量非常大，但是只考虑开源项目。但我注意到，GitHub 是唯一一个认为 Swift 比 Objective-C 更受欢迎的衡量标准。对我来说，这使它更有说服力……但它的开源选择偏见意味着它还远远不是决定性的。

这些统计数据确实很重要，不仅仅是有趣的好奇心和/或行业快照。语言不是最重要的，但也不是无关紧要的。人们根据语言的受欢迎程度和他们(相关的)预测的未来价值来决定学习什么语言，有时甚至决定寻找和接受什么工作。因此，这三个指标如此明显、根本不同，有点令人不安。然而，可悲的是，我们似乎仍然被茶叶而不是硬数字所困。
