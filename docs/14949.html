谷歌、IBM和其他公司发布了一个开源API来监控软件供应链TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2017/10/12/google-ibm-and-others-launch-new-open-source-api-for-keeping-tabs-on-software-supply-chains/?ncid=mobilenavtrend

感谢容器和微服务，我们构建软件的方式正在迅速改变。但是和所有变化一样，这些新模式也带来了新问题。您可能仍然想知道实际上是谁构建了一个给定的容器，以及在其中运行了什么。为了解决这个问题，谷歌、 JFrog 、Red Hat、IBM、Black Duck、Twistlock、Aqua Security和CoreOS今天宣布了 Grafeas (希腊语中的“scribe”)，这是一个新的联合开源项目，为用户提供了一种审计和管理其软件供应链的标准化方法。

此外，谷歌还推出了另一个新项目Kritis(希腊语为“judge”，因为在Kubernetes取得成功后，为谷歌新的开源项目挑选任何其他语言的名字肯定会带来厄运)。Kritis允许企业在部署Kubernetes集群时强制实施某些容器属性。

Grafeas基本上定义了一个API，它收集代码部署和构建管道的所有元数据。这意味着保留作者和代码出处的记录，记录每段代码的部署，标记代码是否通过了安全扫描，它使用了哪些组件(以及这些组件是否有已知的漏洞)以及Q&A是否签署了它。因此，在部署一段新代码之前，系统可以通过Grafeas API检查关于它的所有信息，如果它已经过认证并且没有漏洞(至少就系统所知)，那么它就可以投入生产。

乍一看，这一切似乎很平淡，但像这样的项目确实需要。随着持续集成、去中心化、微服务、越来越多的工具集和每一种其他值得关注的技术的出现，企业正在努力跟踪他们的数据中心实际发生的事情。如果您不确切知道您实际运行的是什么软件，那么坚持您的安全和治理策略是非常困难的。目前，开发人员使用的所有不同工具都可以记录他们自己的数据，但是Grafeas代表了一种公认的跨工具收集和访问这些数据的方式。

像谷歌的许多开源项目一样，Grafeas基本上模仿了谷歌自己处理这些问题的方式。由于其巨大的规模和对容器和微服务的早期采用，谷歌毕竟早在这些问题成为整个行业的问题之前就看到了它们。正如谷歌在今天的公告中指出的，Grafeas的基本租户反映了谷歌自己为其构建系统开发的最佳实践。

这里涉及的所有合作伙伴都带来了不同的东西，但是例如JFrog将在它的 Xray API 中实现这个系统。Red Hat将使用它来增强OpenShift(其容器平台)中的安全性和自动化功能，CoreOS将将其集成到其构造Kubernetes平台中。

Grafeas的早期测试者之一是Shopify，它目前每天建造大约6000个容器，并在其主要容器注册表中保存了33万张图像。有了Grafeas，it现在可以知道给定的容器当前是否正在生产中使用，例如，它是何时从注册表下载的，它正在运行什么包，以及容器中的任何组件是否包含任何已知的安全漏洞。

该公司在今天的公告中写道:“使用Grafeas作为容器元数据的核心来源，使安全团队能够回答这些问题，并为我们在Shopify向用户提供的软件制定适当的审计和生命周期策略。”
