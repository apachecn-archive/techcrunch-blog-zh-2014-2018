Mitchell Baker Memo | TechCrunch 称，Mozilla 希望分拆其雷鸟电子邮件/聊天客户端

原文：https://web.archive.org/web/https://techcrunch.com/2015/11/30/thunderbird-flies-away-from-mozilla/

Mozilla Foundation 看起来即将采取另一项措施，加强对 Firefox 浏览器开发的关注。Mozilla 现在想一劳永逸地放弃对雷鸟的支持，雷鸟是免费的电子邮件、聊天和新闻客户端，它于 2004 年首次开发，但 T2 在 2012 年有效地停止了直接更新。这些计划是在今天由董事长米切尔·贝克撰写的全公司备忘录中披露的。(我们已经和 Mozilla 确认确实是她发的。)

“我认为雷鸟应该通过脱离对 Mozilla 开发系统的依赖，在某些情况下，脱离对 Mozilla 技术的依赖，才能发展得最好，”贝克在她的公开备忘录中写道，该备忘录发布在 Mozilla 的公共治理论坛上。“目前的设置还不稳定，我们应该开始积极研究如何有序地过渡到雷鸟和火狐不耦合的未来。”

贝克指出，目前还不清楚 Mozilla 是否会尝试剥离雷鸟作为自己的开源实体，或者它是否会寻求一个商业伙伴来接管该产品；现在下结论似乎还为时过早。

更清楚的是，Mozilla 一直在试图简化其运行方式及其工程师的关注点，这是重振 Firefox 的更大运动的一部分，也是与谷歌 Chrome 等竞争对手争夺更多市场份额的长期反击的一部分。

Mozilla 现在认为对雷鸟的任何支持，甚至是过去三年来一直提供的有限支持，都类似于“纳税”，用贝克的话说，是在那些工程师花在开发 Firefox 上的工作之上。

“这些相互竞争的需求对两个项目都没有好处，”她写道。“开发雷鸟的工程师必须专注于跟上和适应火狐浏览器的网络驱动的变化。从事 Firefox 和相关项目的工程师最终会考虑雷鸟的竞争性需求，并且/或者想知道他们是否应该帮助雷鸟以及应该帮助多少。这两个项目都无法全心全意地专注于对自己最有利的事情。”

除了 Mozilla 已经在 2012 年切断了大部分开发支持，雷鸟已经成为一个有些不合时宜的产品。

雷鸟于 2004 年进入市场，当时许多消费者仍然坚持使用桌面客户端来访问电子邮件服务，它取得了良好的开端，在其诞生的头 10 天就有 100 万次下载。但在接下来的几年里，许多人转向了移动应用和基于网络的客户端，雷鸟的受欢迎程度有所下降。

Mozilla 和更广泛的 Thunderbird 社区没有提供任何关于今天有多少 Thunderbird 用户，客户端有多少下载量，或者应用程序的使用情况的更新。

对于 Mozilla 本身来说，重点主要转移到为更多的平台开发其核心的 Firefox 浏览器，并总体上使 Firefox 更加商业化，与雅虎和谷歌整合以围绕搜索广告产生收入。

当 Mozilla 在 2012 年将雷鸟开发交给一个志愿者领导的社区时，它承诺只提供“扩展支持版本”，只专注于主要针对使用雷鸟的大型组织的安全和维护更新。在这方面，对于雷鸟及其用户来说，这可能是一个艰难的转变，但这并不令人惊讶。
