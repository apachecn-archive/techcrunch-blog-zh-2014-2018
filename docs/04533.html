电池属性可用于跟踪网络用户 TechCrunch

一组欧洲安全研究人员发表了一篇论文，分析了移动设备的电池寿命如何被用来跟踪 Linux 上 Firefox 用户的网络浏览习惯，使用 HTML5 电池状态 API(通过卫报)。

截至今年 6 月，他们在论文中披露的具体方法已经被 Firefox 修复，此前他们提交了一份错误报告，指出不同平台之间电池水平报告的实现不一致。

这篇题为泄漏的电池:HTML5 电池状态 API 的隐私分析的论文，也提到了对一种 API 的隐私担忧，这种 API 允许网站检查用户的电池状态，而不必获得他们的许可。

研究人员写道:

该 API 不需要用户许可来读取电池信息，任何网站或第三方脚本都可以使用该 API。API 也不要求浏览器在访问电池信息时通知用户。这使得网站和第三方脚本可以在用户不知情的情况下透明地访问电池信息。

电池 API 的表面目的是允许网站在检测到用户需要节省电池时，自动从高功率版本切换到节能版本。然而，研究人员发现，Firefox 在 GNU/Linux 上实现 API 的方式是有问题的，允许 API 提供的“看似无害的信息”作为跟踪标识符(也称为“新的设备指纹向量”)——潜在地允许网站跟踪不同会话的浏览活动，如私人浏览模式或清除 cookies 后。

Firefox 的具体问题集中在它在 Linux 上跟踪电池寿命百分比的精度上——读取“64 位双精度浮点格式的值并乘以 0.01 以获得电池水平。”该双精度值随后通过电池状态 API 暴露给网站脚本——提供了一种粒度测量，他们认为可以用来识别单个移动设备，如智能手机或笔记本电脑。

据研究人员称，问题仅限于 GNU/Linux 上的 Firefox 浏览器，Windows、Mac OS X 和 Android 上的 Firefox 暴露的电池电量远没有那么精确。

他们指出:

在我们对电池状态 API 实现的探索性调查中，我们观察到，GNU/Linux 上的 Firefox 浏览器报告的电池电量以双倍精度呈现给 Web 脚本。在我们的研究中观察到的示例电池水平值是 0.9301929625425652。我们发现，在 Windows、Mac OS X 和 Android 上，Firefox 报告的电池电量只有两位有效数字(例如 0.32)。

在 Linux 上，Firefox 使用一个名为 UPower 的 Linux 工具读取电池电量信息——这是更全面的电源管理数据的最终来源。

他们描述了跟踪脚本如何利用电池信息，他们写道:

存在于多个网站的第三方脚本可以通过利用提供给 Web 脚本的电池信息，在短时间间隔内链接用户的访问。为此，脚本可以使用电池电量、放电时间和充电时间的值。每个站点上的读数将是一致的，因为更新间隔(及其时间)是相同的。这可以使第三方脚本链接这些并发访问。此外，如果用户离开这些站点，但随后不久又访问了具有相同第三方脚本的另一个站点，则这些读数很可能被用来帮助将当前访问与之前的访问联系起来。

除了根据电池电量对设备进行指纹识别之外，研究人员还建议电池容量也可以用作跟踪向量——注意到电池状态 API 可以用于“推断设备的当前电池容量(EnergyFull ),如果它允许高精度水平读数的话。”他们再次指出，这种方法只适用于 Linux 上的 UPower 和 Firefox。

在讨论防御这些与电池 API 相关的跟踪攻击的方法时，研究人员表示，API 的实现应该避免提供高精度的值——舍入电池水平值是最大限度地减少暴露此类识别标记的一种方法。

他们得出结论:

对 Web 标准、API 及其实现的分析可以通过研究暴露给网页的信息来揭示意想不到的 Web 隐私问题。新的 Web APIs 的复杂和庞大的性质以及它们与设备的更深层次的集成使得很难抵御这样的威胁。隐私研究人员和工程师可以通过分析这些标准及其实施对网络隐私和跟踪的影响，来帮助解决这些 API 带来的风险。这不仅可以为 API 设计者和浏览器制造商提供可操作的反馈，还可以提高这些新技术的透明度。

在论文中，研究人员还指出了其他浏览器指纹识别技术的相关研究——如 Panopticlick 研究，该研究展示了浏览器属性(如屏幕大小、字体列表和插件)如何用于指纹打印机用户；或者使用其他属性(如时钟偏差、字体度量、网络协议特征、JavaScript 引擎性能或 WebGL 和画布渲染)演示跟踪方法的研究人员。

在上个月的结束时，安全研究人员还演示了一种收集用户击键并分析他们打字风格的技术如何被用来描述和暴露匿名 Tor 浏览器的用户。
