Snapchat 提醒我们，照片泄露应归咎于用户 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2014/10/14/snapchat-reminds-us-that-users-are-to-blame-for-photo-leaks/

为了回应最近的 Snapchat 泄露事件——现在被称为“The Snappening”以纪念最近名人裸照泄露事件——在公司博客上发布了一份关于非法使用其非官方 API 的声明。按照 Snapchat 的传统方式，该公司几乎不承担任何责任。

上周，一个名为 Snapsaved 的第三方应用程序被黑客攻击，该应用程序使用了逆向工程的 Snapchat API，导致Snapchat 用户超过 500MB 的图片泄露。虽然 Snapsaved 肯定并且承认对最近的违规行为负责，但 Snappening(以及其他过去的不检点行为)提出了一个问题，即 Snapchat 是否做了足够多的工作来阻止第三方应用程序访问其 API。

正如这里解释的Snapchat 从未正式允许第三方应用程序访问其 API，尽管逆向工程并不难。官方 API 或应用程序编程接口通常用于允许第三方应用程序访问 Twitter 或脸书等服务，但 Snapchat 不提供这种接口，因此程序员自己找出这些特性，并创建非官方克隆或实用程序，通过 Snapchat 的网络发送数据。

重要的是第三方应用确实可以访问 Snapchat 照片，而 Snapchat 对此几乎无能为力。当这些第三方应用程序利用不受监管的 API 时，Snapchat 的照片安全性将一直受到质疑。

这是许多成功进入大联盟的社会服务的诅咒。这就是为什么许多这样的服务，如 Twitter 和脸书等，都开发了官方 API，使他们能够跟踪第三方应用程序，并完全控制自己的平台。

Snapchat 作为一个年轻的应用程序，还没有做到这一点。在过去的一年中，该公司的主要公告集中在他们的“故事”产品的扩展上，这很可能会成为这家高价值、非创收公司的主要收入来源。

根据 Snapchat 的声明，该公司“对开发 Snapchat 平台的兴趣感到兴奋”，但“将需要(他们的)时间来做好这件事。”没有时间表的暗示，甚至没有提到积极开发私有 API，这也是实际解决第三方应用程序访问用户数据问题的唯一方法，并最终为 Snapchat 的用户社区负责。Snapchat 似乎几乎不愿意这么做。

Snapchat 敦促用户远离任何声称使用或修改 Snapchat 服务的第三方应用程序。他们说“常识和安全对策的结合”是保持社区安全的最好方法。再次，没有提到对混乱的责任。

不幸的是，即使是从未下载或使用第三方 Snapchat 应用程序的用户，也仍然可能受到其他用户的惩罚或侵犯，这些用户犯了使用第三方应用程序的不幸错误(或恶意决定)。这些应用程序是生态系统中的病毒，尽管如此。

即使应用程序本身没有恶意，关于存储图像或安全的糟糕决定也会暴露用户数据——在这种情况下，他们希望保持隐私或永远消失的照片。

在昨天给 the Verge 的一份声明中，Snapchat 重申使用第三方应用程序是完全违反规定的(这是他们无法控制的)。

Snapchatters 因使用第三方应用程序发送和接收快照而受害，我们在使用条款中明确禁止这种做法，正是因为它们危及我们用户的安全。我们警惕地监控 App Store 和 Google Play 中的非法第三方应用，并成功移除了其中的许多应用。

所以，请记住，如果你使用第三方 Snapchat 应用程序，你实际上是一个规则破坏者，很可能是一个坏人。你在拿自己和他人的安全冒险。是你。不是 Snapchat。

非正式访问 Snapchat API 也是违反 Snapchat 的服务条款的。

有趣的是，有了正确的产品和正确的“品牌”，Snapchat 用户(定义为一群对分享消失的内容感兴趣的人)觉得我们的故事更像是一个功能，而不是一个广告渠道。

无论如何，安全必须成为 Snapchat 的重中之重。这项服务绝不是在考虑安全性的情况下构建的。但人们仍然用它来发送机密和敏感的材料，即使那些是愚蠢的自拍。它肯定是作为一个“安全”空间来设计和营销的。在这里，安全和保障的区别是最重要的。

Snapchat 的基础是活在当下，与其他用户分享真实的体验，无论他们离得多近或多远。它是关于拥有一个与你现实生活中的身份相匹配的数字身份和你真正交流的方式。毕竟，现实世界中的对话不会被跟踪或记录。

重点是不要让用户觉得他们可以通过安全、加密的渠道发送高度机密的文档和文件。重点是用户需要有安全感。

如果你想阅读 Snapchat 关于 Snapchat 开放的完整声明，你可以查看博客帖子或阅读下面的文字记录。我希望这不会是我们最后一次听到关于 Snapchat 安全性的消息，我也希望这些人最终会做正确的事情。

在过去的几天里，在一个提供保存快照的网站表示他们的数据库被入侵后，我们回答了许多关于我们的 API 和第三方应用程序的问题。我们很感激服务提供商承认 Snapchat 从未受到威胁，但我们希望借此机会重申这些第三方应用程序可能给我们的社区带来的不幸威胁。

第三方应用是访问 Snapchat API 的任何应用，但不是由我们公司构建和维护的。鉴于 Snapchat 的受欢迎程度和我们社区的规模，出现一个由应用程序制造商组成的家庭手工业来为 Snapchat 用户提供额外服务就不足为奇了。不幸的是，这些应用程序经常要求 Snapchat 登录凭据，并使用它们来发送或接收快照和访问帐户信息。

当您将登录凭据提供给第三方应用程序时，您就允许开发人员(可能还有罪犯)访问您的帐户信息并代表您发送信息。

构建一个开放可信的第三方应用生态需要时间和大量资源。这就是为什么我们没有向开发者提供公共 API，为什么我们禁止访问我们用来提供服务的私有 API。不要误解我们——我们对为 Snapchat 平台开发的兴趣感到兴奋——但我们会花时间把它做好。在此之前，这意味着任何不是我们的应用程序，但声称提供 Snapchat 服务，违反了我们的使用条款，不能被信任。

Snapchat 一直是一个与朋友分享照片的有趣地方。保持我们社区安全的最好方法是结合安全对策和常识。我们将继续尽我们的职责，提高 Snapchat 的安全性，并呼吁苹果和谷歌关闭访问我们 API 的第三方应用程序。您可以通过避免使用第三方应用程序来帮助我们。
