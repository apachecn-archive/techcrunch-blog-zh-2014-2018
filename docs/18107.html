Slack 引入了一些措施，让用户无需离开 TechCrunch 就能更轻松地创建和完成任务

原文：https://web.archive.org/web/https://techcrunch.com/2018/05/22/slack-introduces-actions-to-make-it-easier-to-create-and-finish-tasks-without-leaving/

随着 Slack 试图超越硅谷的宠儿，成为公司内部的沟通平台，它必须想办法越来越多地将自己定位为公司的智能瑞士军刀——而不仅仅是一个简单的聊天应用——它今天试图通过一个名为 Actions 的新功能再次延续这一点。

公司现在可以将他们自己的用户体验直接嵌入到 Slack 应用程序中，而不是绑定到他们服务中的另一个聊天机器人。开发人员可以在 Slack 核心聊天体验中，通过一个名为 action 的下拉窗口，为任何类型的操作创建一个定制的提示，比如提交支持票。虽然 Slackbots 可能是这种情况的早期体现，但 Slack 的平台已经发展到包括超过 200，000 名开发人员，并且内部仍然不断需要健壮的工具。这为合作伙伴和开发人员提供了更多的灵活性，让他们能够确定什么样的体验对那些整天无所事事、但必须不断从自己的工具中移植信息的人最有意义。

Slack 首席产品官 April Underwood 表示:“人们对专业软件以及易于使用并能与所有应用程序互操作的优秀工具有着如此强烈的需求。“我们认为这很好，我们认为更多的工具意味着客户有更多的选择。最终，市场上会有更多的竞争，这意味着最好的工具，那些真正帮助公司做好工作的工具，会脱颖而出。但是你的工作经验变得越来越孤立。Slack 需要具有高度的可配置性，但在这样做的时候，我们相信 Slack 是将所有这些整合在一起的协作中心。”

每一家希望建立集成的公司——如任务管理的 Asana 或票证管理的 Zendesk 都努力在核心 Slack 体验中创建一个新的流程，其中包括一个新的消息下拉列表和一个将某些内容加入聊天流程的提示。一旦发生这种情况，所有这些信息都将被移植到集成中，并以员工在该环境中创建信息的相同方式进行创建。如果有人通过 Slack 中的一个动作创建了一个 Zendesk 票证，Zendesk 会自动在他们这边生成票证。

随着时间的推移，Slack 已经扩张开来，特别是随着使用它的公司越来越大，该公司必须找到一种方法来表明，它可以保持一个非常简单的应用程序，而不会变成一个充满数千条即时消息的臃肿窗口。Actions 是一种潜在的方法，用户可以从一开始就知道在哪里协调某些活动，如设备采购或管理一些客户信息，而不必去任何其他地方。

另一个好处是，它使得完成任务的目的地不一定是“什么”，而是“谁”Slack 正在依靠它的机器学习工具来越来越容易地找到有正确答案的正确的人，无论这些问题在哪里已经得到了回答，或者他们知道谁可以马上给你提供信息。Actions 也是这里的另一个扩展，因为用户可以习惯于带着完成任务的目的去找某些同事，例如他们每天早上去喝咖啡时路过的办公室的 IT 主管。

该公司表示，它还在开发一款名为 Block Kit 的产品，该产品将这些任务和其他元素直接集成到 Slack 聊天流中，从某种视觉上看起来更加用户友好。这里的想法是，再次为人们创建一个直观的流程，这不仅仅是一个简单的聊天应用程序，而是提供一些额外的交互方式，将 Slack 变成一个更合理的提要，而不仅仅是一个人们相互交谈的窗口。吉拉、Bitbucket、Asana、Zendesk、HubSpot 和其他几个网站都提供动作。

Actions 是 Slack 今天早上在自己的开发者大会 Spec 上发布的一个工具。这本身就是 Slack 的又一个例子，它不仅仅是一个与小公司合作良好的简单信息来源。开发人员通常是为任何平台找出最佳利基用例的人，因为这意味着 Slack 可以专注于试图找出所有这些集成如何融入其设计精神。该公司必须想出如何说服大公司，他们需要这样的工具，它不会失去控制，并确保小公司不会发展成更灵活的东西，可以在他们变大时服务于那些利基案例。

可以肯定的是，懈怠正在增长。该公司表示，本月早些时候，其日活跃用户达到 800 万，付费用户达到 300 万。这帮助它迅速跃升至 51 亿美元的估值(截至其最近一轮融资)，该公司一直在小心翼翼地推出一些工具，这些工具可能会让大公司内部的沟通变得更容易一些——包括一年多前期待已久的 threads 的推出。

但随着时间的推移，Slack 也面临着越来越大的竞争，不仅来自那些寻求构建更强大但更简单的工具的传统公司，还来自那些花了大量时间开发协作工具、现在正在探索沟通的公司。今年二月，Atlassian 向 T4 的开发者开放了它的通讯平台 Stride。微软也在继续更新其团队产品。Slack 能够暴露出对这种方法被压抑的需求，但它也必须捍卫这种方法——让它更加灵活，而不出现功能爬行将是它未来最大的挑战。
