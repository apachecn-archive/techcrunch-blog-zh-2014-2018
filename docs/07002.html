方形副本带新的在线结账 API TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2016/03/30/square-copies-stripe-with-new-online-checkout-api/

当心 Braintree、T2、Stripe 和其他在线结账系统，Square 正在追赶你。该公司今天为线下和在线商店推出了新 API。但更大的消息是，电子商务商家现在可以使用一个简单的表格来处理信用卡支付。在幕后，Square 处理付款。

和 Stripe 一样，你不需要注册电子钱包。你可以在结账屏幕上输入你的信用卡信息。像 Stripe 一样，很容易用几行代码集成支付，因为 Square 也押注于 API 策略和与现有结账屏幕的原生集成。像 Stripe 一样，在线结账每笔交易的费用为 2.9%加 30 美分。

那么你为什么要注册 Square 而不是 Stripe 呢？与 Stripe 相比，Square 有一个明显的优势——它可以在线和离线工作。如果您是商店中现有的 Square 用户，那么利用 web 上的电子商务 API 是有意义的。这是一个具有统一后端的统一解决方案。你所有的销售额都会自动显示在 Square 的仪表盘上。

这并不是 Square 第一次为电子商务客户开发产品。有了 Square Market(现在叫做网上商店，该公司让你创建一个带有拖放界面的网上商店。它就像网站创建和托管服务 Squarespace ，但只针对网上商店。如果你开始混淆 Square 和 Squarespace 这两个名字，Squarespace 还可以让你创建一个网上商店——讽刺的是，Squarespace 使用 Stripe。

但是让我们回到起点。有了新的 Square 电子商务 API，从 Joe 的自行车店到沃尔玛，任何人都可以使用 Square 进行结账。你不需要把你的目录挤进 Square 的电子商务网站创建工具。

今天还有一个新消息，该公司正在为线下客户引入一个新的注册 API。有了这个 API，任何 iOS 销售点 app 都可以为支付砖集成 Square。当然，它可以与 Square 的读卡器配合使用。因此，如果你有非常具体的需求，很快就会有多个注册应用程序与你的 Square 阅读器一起工作——而不仅仅是官方的 Square 应用程序。

这些新的 API 也是说服有特殊需求的大客户的好方法。现在，麦当劳没有理由不保留现有的收银机，转而使用 Square 支付。同样，所有大型 Stripe 客户，如 Kickstarter、Postmates、Lyft 和 Shopify，现在都有了另一个选择。
