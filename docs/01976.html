亚马逊云驱动获得自己的 API TechCrunch

原文：https://web.archive.org/web/http://techcrunch.com/2014/11/11/amazon-cloud-drive-gets-its-own-api/?ncid=rss&utm_source=feedburner&utm_medium=feed&utm_campaign=Feed:+Techcrunch+(TechCrunch)

亚马逊正在努力使其云驱动服务在拥挤的在线存储市场上更具竞争力。最近，亚马逊 Prime 客户在 Cloud Drive 上捆绑了免费、无限的照片存储空间，今天，该公司正在用一种新的 Cloud Drive API 追逐开发者。该 API 将允许第三方开发人员将 Cloud Drive 集成到他们自己的应用程序中，因此他们可以更多地关注他们应用程序的功能集，而不是存储所涉及的复杂性。

当然，开发人员使用 Amazon Web Services 已经有一段时间了，但 Cloud Drive API 专注于将类似的、更面向消费者的技术构建到应用程序中。例如，照片编辑应用程序可以让用户浏览和编辑他们存储在亚马逊云驱动器上的照片。

亚马逊表示使用云驱动 API 的开发者将不必担心各种屏幕分辨率、元数据管理、索引、搜索或同步功能等问题；它将在 API 集成后包含在内。Cloud Drive 也可以在其他开发者工具中使用，包括 Filepicker 和 Temboo，这使得那些使用一系列云服务的人更加容易。

在推出时，许多移动应用程序都集成了 Cloud Drive 的 API，包括: OfficeSuite 、完美清晰、 FreePrints 、 Photo Gifts by MailPix 、 ScanCafe 、 FileThis 、 CloudHQ 、 File Commander 、 Jumptuit 、 Secrata 这些现在都出现在在线云驱动应用程序图库这里。

从早期的产品阵容中你可以看出，这些应用程序专注于处理最终用户的文件或照片，但这些文件的实际托管并不是应用程序的关键价值主张。例如，一些应用程序提供照片、礼品或打印服务，而 FileThis ( 个人最喜欢的)将从网站上下载的文件存档，比如银行对账单或信用卡账单。通过 Cloud Drive API，这些应用的用户可以上传、下载、组织和操纵或编辑他们自己存储在亚马逊云中的数据。

应用程序可以直接访问消费者的云驱动器存储和保存在那里的内容，他们用它来保存游戏设置、偏好和其他应用程序状态数据。

对于开发人员来说，让用户能够访问存储在云中的文件扩大了可以呈现的文件数量，而不受移动设备有限存储空间的限制。亚马逊指出，这还意味着用户可以在他们选择使用的任何设备上获得文件，而无需开发者设置和管理他们自己的服务器、存储或负载平衡器。

Cloud Drive API 基于 REST，构建于 AWS 之上，利用“登录亚马逊”协议来保证终端用户和亚马逊之间以及应用和亚马逊之间的通信安全。

正如本周早些时候指出的，云计算正在成为的一种“归零竞赛”——这意味着，随着竞争的加剧，云存储公司不断降价并提高存储限制。亚马逊是推动这一趋势的公司之一，多年来多次下调存储价格。现在，它正致力于让开发者更容易利用其存储服务，从而加快上市时间。
