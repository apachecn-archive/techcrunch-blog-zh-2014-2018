# Cloud Native Foundation 将 Prometheus 监控工具添加为其第二个托管项目

去年七月，[云本土基金会](https://web.archive.org/web/20221209171744/https://cncf.io/) (CNF)的[发布会引起了一些轰动，因为它汇集了非常广泛的公司，包括 AT & T、Box、思科、CoreOS、Docker、谷歌、IBM、英特尔、红帽、Twitter 和 VMware。该基金会背后的理念是“协调新兴技术，促进容器封装、动态调度和基于微服务的应用开发和运营的创新。”](https://web.archive.org/web/20221209171744/https://beta.techcrunch.com/2015/07/21/as-kubernetes-hits-1-0-google-donates-technology-to-newly-formed-cloud-native-computing-foundation-with-ibm-intel-twitter-and-others/)

通过 Kubernetes，谷歌向该项目捐赠了其开源容器管理服务，CNF 技术监督委员会于 3 月正式接受 Kubernetes 作为其首个托管项目[。](https://web.archive.org/web/20221209171744/http://www.linuxfoundation.org/news-media/announcements/2016/03/cloud-native-computing-foundation-accepts-kubernetes-first-hosted-0)

现在这个团队已经有了所有的治理系统，它也准备好接受第二个项目了。这第二个项目是 [Prometheus](https://web.archive.org/web/20221209171744/https://prometheus.io/) ，一个开源云监控解决方案。普罗米修斯的早期开发是由 SoundCloud 赞助的。

“云 原生计算基金会很高兴能够托管 Prometheus 作为我们的第二个开源软件项目。CNF 技术监督委员会主席 Alexis Richardson 在今天的声明中说:“像 Prometheus 这样的现代监控和分析工具包是新的云 原生范式的关键部分。“TOC 期待与该项目合作，通过基础设施、指导和营销支持，帮助 Prometheus 取得成功。”

虽然 Kubernetes 构成了许多新容器部署和商业服务(如 CoreOS 的 constructive)的基础，但构建企业级云部署需要更多的东西。

普罗米修斯是专门为在集装箱环境中运行而设计的，虽然它是在 SoundCloud 孵化的，但它的灵感来自谷歌的内部 Borgmon 监控工具。像 Borgmon 一样，Prometheus 使用时间序列数据作为生成警报的来源。

该工具目前被 Digital Ocean、爱立信、CoreOS、Docker、Red Hat 和 Google 本身使用。
