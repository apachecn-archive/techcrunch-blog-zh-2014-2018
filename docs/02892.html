这个脸书漏洞允许任何人删除你的照片

这个脸书病毒允许任何人删除你的照片

你有多少张脸书的照片？其中有多少是你从未想过要备份的照片？

这个刚刚披露的脸书漏洞会让任何有点技术知识的人删除脸书上的任何照片。

幸运的是，发现窃听器的人(印度的拉克西曼·穆西亚赫)很快给了脸书一个提醒——由于他的麻烦，他得到了 12500 美元的奖金。(当然，这种病毒很容易给脸书造成超过 12500 美元的损失——但这并不是病毒奖励项目的工作方式。)

脸书转过身来，在大约两个小时内修复了这个 bug。

拉克斯曼在这里对它是如何工作的进行了分解，但这里有一个简短的版本:脸书的图形 API 没有正确检查权限。如果你向 Graph API 发送一个删除另一个用户相册的请求，并抛出你自己的 Android 脸书令牌作为所需的批准印章，它会盲目接受，相册就会消失。

在攻击者端，album delete 命令看起来应该是这样的:

请求:-
DELETE/[受害者的 _photo_album_id] HTTP/1.1
主持人:graph.facebook.com
Content-Length:245
Access _ Token =[你的(攻击者)_ 脸书 _for_Android_Access_Token]

在受害者这边，相册会…消失。

这是一个相当简单的错误，真的——你永远也不会想到它会真正工作。

但它确实发生了——而且可能会产生相当恶劣的后果。正如 Sophos security 指出的那样，脸书的相册是用简单的序列号来识别和存储的。如果有人在服务器上弹出这个东西，并编写一个基本的数字增量器来盲目地挖掘相册，攻击者可能会在脸书知道之前删除 T2 的大量照片。

更新:脸书的一名代表告诉我们，要想全部删除并不容易:

我们收到了一份关于我们的 Graph API 的问题的报告，并在验证声明的两个小时内迅速修复了它。需要说明的是，触发此问题需要知道目标相册的 ID，以及根据相册的隐私设置查看相册的权限。我们要感谢通过我们的 bug 赏金计划向我们报告这个问题的研究人员

让它成为一个温和的提醒:脸书不是一个备份驱动器。虽然你的照片有望不会毫无预警地消失，但脸书的代码并不是绝对可靠的。备份你喜欢的东西。
