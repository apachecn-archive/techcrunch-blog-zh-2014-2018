谷歌揭示了“物理网络”，这是一个让物联网互动无应用的项目

谷歌的Scott Jenson是一名交互和UX设计师，去年11月才离开公司回到Chrome团队，他透露了公司正在进行的一个名为 The Physical Web 的项目，该项目旨在提供“按需交互”，这样人们就可以随时随地使用任何智能设备，而无需插入移动应用程序。这将使用户只需走到一个公共汽车站，就可以知道下一辆公共汽车到达的时间，而不需要任何额外的软件。

该项目是对智能设备未来的一次雄心勃勃的押注。分析师预测，未来几年互联设备将呈爆炸式增长，思科预计到2020年将有500亿部联网设备投入使用，英特尔预计到明年这一数字将达到150亿部。由简森牵头的谷歌项目将使人们更容易与日益增长的互联设备网络进行互动。

“人们应该能够走向任何智能设备——自动售货机、海报、玩具、公交车站、出租汽车——而不必先下载应用程序，”简森在实体网络项目页面上解释道。“一切都应该只是一个水龙头。”

然而，这个项目的目的不仅仅是创造一些可以被谷歌设备和软件利用的东西；它想创建一个开放的、人人都可以使用的标准，就像那些用于网络基本构件的标准一样。共享标准肯定有助于扩大联网设备的用途，但苹果已经通过iBeacon和iOS 8中新的上下文相关应用建议，朝着这种技术的专有版本迈出了一步。

简森在项目页面上列出了拟议标准的一些实际应用，包括提供快速无应用支付的停车计时器和自动售货机，通用的实体零售购物体验，以及通过广告停车位本身的标志工作的ZipCar租赁系统。至于Chrome为什么要带头努力，该团队认为这是另一个网络标准，因为它超越了孤立的本地应用程序。

如果我们有一个被接受的开放标准，可能还需要几年的时间，但物联网的想法甚至不需要像SmartThings和其他公司提供的那样的单一集中式软件中心，这很有吸引力，特别是因为它使系统控制民主化。
