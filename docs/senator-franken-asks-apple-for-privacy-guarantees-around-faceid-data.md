# 参议员 Franken 要求苹果公司保证 Face ID 数据 TechCrunch 的隐私

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/09/13/senator-franken-asks-apple-for-privacy-guarantees-around-faceid-data/>

参议员艾尔·弗兰肯给苹果写了一封友好的信，要求后者提供更多关于 Face ID 系统背后技术的细节，该系统允许用户使用面部识别解锁他们的 iPhone X。

它离一个令人讨厌的节目很远；这位参议员很明显只是想涵盖比苹果昨天在[的展示](https://web.archive.org/web/20221206205841/https://beta.techcrunch.com/gallery/everything-apple-announced-at-its-iphone-x-keynote/)中时间更多的领域。他写道:

> 苹果公司声明它已经采取措施负责任地实施该系统，这让我很受鼓舞。然而，关于 Face ID 将如何影响 iPhone 用户的隐私和安全，以及该技术在不同人群中的表现是否一样好，仍然存在实质性的问题。为了让数百万使用你们产品的美国人明白，我要求你们提供更多信息，说明该公司如何在内部处理这些问题，以及它打算采取哪些额外措施来保护用户。

Face ID 使用一种类似 Kinect 的系统来扫描用户的面部，只让匹配的面部指纹解锁手机，这在技术界受到了一些质疑。iPhone 发布会上的一个失误没有帮助，但有一些可用性问题(当手机放在桌子上几英尺远的时候，你如何解锁手机？当然不是一根针？)[还有隐私的](https://web.archive.org/web/20221206205841/https://beta.techcrunch.com/2017/09/13/iphone-xs-face-id-raises-security-and-privacy-questions/)。

[![](img/dad6fae6a65c385d6c26cf661dd6e3ff.png)](https://web.archive.org/web/20221206205841/https://beta.techcrunch.com/wp-content/uploads/2017/09/frontcam_depth_iphone-x.jpg) 当然，当苹果首次推出触控 ID 的指纹识别时，也有类似的担忧——果然，[参议员弗兰肯当时也写了一封信](https://web.archive.org/web/20221206205841/https://www.franken.senate.gov/?p=press_release&id=2562)。

他今天的信充分说明了面部识别系统的潜在弱点。例如，他问苹果吹嘘为系统训练集的 10 亿张人脸图像的来源是什么，因为缺乏多样性可能导致代表性不足的群体无法使用 Face ID。

他还询问苹果是否有任何计划将面纹数据用于 Face ID 以外的任何目的，苹果或任何感兴趣的第三方是否有可能从手机中提取这些数据，以及这些数据是否可能被远程存储。

有趣的是，他问是否有任何保护措施可以防止有人拿着手机对着自己的脸强行解锁。有点暗，参议员！

最后，他询问苹果将如何回应执法部门对面纹数据的要求。考虑到科技公司在识别用户身份、回应执法请求等方面面临的巨大压力，这是一个棘手的问题。

如果苹果的回答与它在对 2013 年信件的[回复中给出的答案类似，要点将是，因为 faceprint 存储在安全的飞地中，因此苹果、其服务或合作伙伴无法访问，这些问题中的许多都将没有实际意义。](https://web.archive.org/web/20221206205841/https://www.franken.senate.gov/files/letter/140718AppleResponse.pdf)

然而，对于剩下的问题，我期待苹果的回应和回避，每一个都可能以自己的方式说明问题。苹果被要求在 10 月 13 日之前回复参议员。
