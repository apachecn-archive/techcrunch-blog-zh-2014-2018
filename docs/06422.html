GitHub 通过集群支持更新其企业产品，更新设计 TechCrunch

原文：https://web.archive.org/web/http://techcrunch.com/2016/02/09/github-updates-its-enterprise-product-with-clustering-support-updated-design/?ncid=rss

GitHub 用集群支持更新了它的企业产品，更新了设计

该公司管理代码的内部解决方案 GitHub Enterprise 今天进行了重大更新。它的出现正值公司内部围绕管理层对该产品的重视程度似乎发生了一些剧变。

GitHub Enterprise 2.5 的突出特性是支持集群。有了这个，企业现在可以建立一个 GitHub 企业服务器集群，作为一个单一的安装，使它能够支持更大的团队。

“有了 GitHub Enterprise 2.5，随着团队的成长，可以在一个系统上支持更多的用户，”GitHub 的产品副总裁 Kakul Srivastava 告诉我。“我们的客户有成千上万的开发人员，他们需要能够一起工作，这是非常重要的功能，使他们能够以可扩展的方式做到这一点。”

她还指出，集群目前不会给 GitHub 的企业用户带来额外的成本。

这个新版本还包括界面改进，从登录屏幕到 GitHub 库的外观和感觉都有更新的设计。这使得 GitHub 的企业产品与其托管版本保持一致。

这个版本中的新功能还包括改进的 Subversion 支持——对于那些不使用 git 作为版本控制系统的人来说——以及一个用于管理受保护分支的 API(也就是说，分支开发人员既不能删除也不能强制推送他们的代码)。这个新的 API 目前正在预览中。

我忍不住问 Srivastava 关于 GitHub 对企业客户的关注是如何在公司内部及其与托管平台上的用户的关系中造成许多摩擦的谣言。

在一份书面声明中，Srivastava 表示，她认为 GitHub 正在“建立一种新型的企业公司；对于我们和我们的客户来说，这不是关于开源或企业，而是开源和企业。”她补充说，GitHub“对开发人员的关注触及到了为他们服务的核心，无论他们在哪里编码——个人项目、开源社区，甚至是银行、零售和制造等行业中最大的企业。”

当然，所有这些都没有解决围绕该公司不断变化的文化的担忧，但毫无疑问，GitHub 的未来与采用其服务来管理代码的企业客户直接相关。
