微软的新工具让企业将 Skype 整合到自己的网络和移动应用程序中

原文：https://web.archive.org/web/http://techcrunch.com/2016/03/31/microsofts-new-tools-let-businesses-integrate-skype-into-their-own-web-and-mobile-apps/?utm_source=inshorts&utm_medium=inshorts_full_article&utm_campaign=inshorts_full_article

继昨天关于将机器人引入 Skype 平台的消息之后，微软今天为其 Skype 平台发布了另外两个 SDK，这将允许企业和应用程序开发人员将 Skype 的功能集成到他们自己的应用程序中。在年度 Build 大会的舞台上，该公司推出了两款面向开发者的新工具包:Skype Web SDK 和 Skype for Business 应用 SDK。两家公司都有一个相似的目标，即通过允许第三方利用 Skype 更大的网络和功能集来更广泛地扩展 Skype 的体验。

通过 Skype Web SDK ，微软为企业提供了一套 JavaScript Web APIs 和 HTML 控件，允许开发者在自己的应用中建立实时协作。例如，这包括在线状态、聊天、音频和视频。

SDK 可以在浏览器中运行的任何客户端应用程序中工作，如 Internet Explorer、Chrome、Safari 或 Firefox。

与此同时，Skype for Business 应用程序 SDK 旨在将 Skype 引入在 iOS 和 Android 上运行的企业应用程序中。预览版即将推出，这个 SDK 对微软所谓的“远程顾问体验”特别有用这意味着像移动设备用户和远程位置的人之间的会议，比如医生或财务顾问。这里的想法是，你将启动企业的应用程序来建立这种联系，但 Skype 将为双方之间的通信提供动力。

在台上，该公司展示了一个来自 MDLIVE 的应用程序，展示了企业如何使用 Skype 新的商业友好工具集，这是一种让你在虚拟环境中接触医生、儿科医生和治疗师的服务。在演示过程中，您可以看到潜在患者如何在企业自己的应用程序中预约、安排时间，然后与医生进行实时聊天。Skype 熟悉的通话和挂断按钮似乎是它为视频聊天提供连接功能的唯一标志。

微软还指出，其 SDK 符合 HIPAA 标准，能够实现医生和患者之间的安全对话。

尽管没有演示，另一家利用新 SDK 的公司是 Smartsheet，这是一种基于云的企业工作管理和协作解决方案。这家公司正在为其云应用程序创建一个插件，让团队看到存在并参与与手头任务和条目相关的 IM 对话。

实际上，通过这些 SDK 版本，微软正在利用 Skype 后端的力量，将其转化为可以嵌入到各种设备的应用程序中的东西，包括桌面和移动设备，以及 web 和本地应用程序。

然而，该公司没有回答这些企业可能会提出的几个问题，包括是否涉及许可和服务器成本，或者 Skype SDKs 是否支持 WebRTC 标准等。

更新:微软表示，如果该公司获得了 Skype for Business Server 或 Skype for Business Online 的许可，那么使用 SDK 就不会产生额外的费用。此外，它说 Skype Web SDK 将在支持 WebRTC 的浏览器中运行。作为该版本的一部分，它还预览了与 Microsoft Edge 的 Skype for Business 之间的插件免费音频和视频通话支持。
