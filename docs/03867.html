政府害怕软件。本月，英国宣布了一项法律，要求“谷歌、脸书和其他互联网巨头”给予英国间谍访问加密对话的权限；商务部建议将“入侵软件”归类为两用民用/军用技术；司法部声称API 应该受到版权保护。

旧的都是新的。还记得最初的密码大战吗？那是美国政府将重要软件归类为两用弹药，并试图在所有移动通信上强加政府后门的日子。

但那是九十年代！几乎是史前时代。(尽管我们还在遭受宿醉的折磨。)然而，在我们大胆而美丽的今天，美国政府……将重要软件归类为两用武器，并试图在所有移动通信上强加政府后门。看到我们走了多远了吗？

如果我们能够纪念第一次加密战争，赢得第二次加密战争，然后进入一场由代理人进行的轻松愉快的加密冷战，那就(相对)好了。唉，冲突不再以 20 世纪的方式运作了。相反，对加密的战争就像反恐战争。它的目标是一种策略，而不是真正的对手；所以战争是无休止的，敌人是无定形的，胜利的条件是不存在的。我们一直在与密码作战。

…即使我们的政府通常不完全清楚这到底意味着什么。以美国商务部工业与安全局(BIS)的新“入侵软件”提案为例。“入侵软件”到底是什么？很高兴你问了:

为避免被计算机或有网络功能的装置的"监测工具"检测或破坏"保护性反措施"而"专门设计"或修改的软件，并具有下列特性之一:

(a)从计算机或有网络功能的设备提取数据或信息，或修改系统或用户数据；或者

(2)修改程序或过程的标准执行路径，以便允许执行外部提供的指令。

他们继续定义“监控工具”和“保护性对策”，同时明确排除了管理程序、调试器、逆向工程工具、DRM 软件(这非常滑稽)和资产追踪器——但这仍然是一个非常宽泛的规范。

出于仁慈的目的而击败“保护性对策”和“监控工具”的情况真的不鲜见。越狱你的手机跃入脑海。或者 SSH 隧道。此外，如果/当这些监控工具和防护措施是由其他人(制造商或运营商)安装在您的计算机上，而您希望将其删除，该怎么办？

这还没有考虑到世界各地仁慈的安全研究人员的合法工作，以及那些声称许多主要技术公司乐意向那些发现安全漏洞的人提供“漏洞奖金”的人，因为这极大地提高了他们的安全性。最初的意图可能是高尚的——限制向暴虐的国家出售监控系统——但是，正如 EFF 的 Nate Cardozo 和 Eva Galperin 所说:

本月提出的规则(BIS)是一场灾难…一套不可行的广泛控制…从表面上看，似乎 BIS 刚刚提议禁止未经许可共享漏洞研究…这非常令人担忧…英国的实施并没有试图控制漏洞或“入侵软件”本身的出口，而简单阅读 BIS 的提议似乎正是如此。同样，英国的实施不影响越狱、模糊或漏洞报告，而 BIS 规则可以解释为包括它们…

打个比方:想象一下，在据称试图打击杀人的民兵组织的同时，政府还顺便取缔了当地的邻里监督组织。

与此同时，当政府的一只触手(理论上)试图限制国家监控时，其他人正试图极大地扩大它。英国和美国都想要苹果、谷歌等。，为他们提供某种“安全金钥匙后门”的味道，这样他们就可以在那些平台上读取加密信息。(当然，也是这些政府多年来一直在悄悄地搜集自己公民的元数据，几乎所有人都不知道，直到爱德华·斯诺登出现。)

所有这一切都是以徒劳的尝试为名，试图把强加密术的精灵塞回它的灯里。开源的强加密技术长期以来一直可以免费获得，例如来自的 Open Whisper Systems (这是斯诺登最喜欢的。)任何真正想加密通信的人，哪怕只有一点点技术能力，都可以在没有苹果和谷歌等公司支持的情况下自己加密。你不能禁止数学，也不应该尝试。

美国司法部(Justice Department)公开抨击 API 应该受到版权保护的观点，这加剧了美国政府最近的技术过失。这对于公民权利来说不是一个可怕的威胁，但不知何故比关于加密的战争更令人恼火，因为这太 明显错了。对于任何曾经写过软件的人来说，显而易见的是，API 坚定地站在版权法“思想/表达”二分法的“思想”一边，比如最初裁决此案的法官 T4。我也是一名作家，因此是版权的忠实粉丝，但这是荒谬的。

在过去，我会把这一切简单地归结为官僚对不断变化的世界的恐惧，这种恐惧如此容易引发反黑客道德恐慌，以至于你可能会因为增加一个 URL 而被判三年监禁，或者因为经营一个在线市场而被判终身不得假释。但现在不会了。现在我认为更有趣的事情正在发生。

我怀疑这些限制软件的悄悄尝试是对其对世界秩序日益重要的认识。很能说明问题的是，加拿大的可怕的新反恐立法已经扩大了“破坏加拿大安全的活动”的定义以包括“干扰全球信息基础设施”

在我看来，权力集体越来越多地得出这样的结论:国家对使用暴力的垄断必须扩展到某些类型软件的使用(例如， 0-days )，而裙带资本主义知识产权的范围必须扩展到软件 API。他们错了吗？道德上，是的。但实际上，从自我保护的角度来看，他们很可能是对的。
