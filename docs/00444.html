为了提升其在旅游大数据领域的竞争力，Sabre推出了新的开发人员门户网站Advanced APIs TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2014/05/20/to-raise-its-game-in-travel-big-data-sabre-launches-new-developer-portal-advanced-apis/

上市一个月后， Sabre 推出了一些新的服务来提升自己在开发商中的形象，试图通过第三方来发展业务。

今天，这家旅游数据公司推出了一个新的门户网站 Sabre Dev Studio ，它提供了大约150种API、测试工具、文档、原型和样本代码，以及一组专注于电子商务和语义搜索的新API。

开发者门户——代码主要由Sabre自己开发，但门户由Mashery构建——是Sabre的第一次:该公司表示，这将是第一次将大量这些服务集中到一个可访问的地方。

此举正值Sabre和更广泛的旅游业正在发生许多变化。Sabre在首次公开募股(IPO)后一直试图精简其业务，该公司的估值低于私募股权公司过去支付的50亿美元。它一直在进行重组，包括裁员、外包部分It业务等等。

但在如今许多公司都提供免费API来扩大受众的情况下，Sabre采取了更保守、更底线的方法。总的来说，它的API将保持商业化。

Sabre Travel Network的首席商务官肖恩·阿瑞纳(Sean Arena)表示:“根据所讨论的产品，它可能是一种交易模式、一种订阅模式、一种收入分成模式。”他补充说，有些可能在一定时间内免费，或者用来测试新服务。

然而，他也承认这种方法并不总是容易维持。“我要说的是，随着价格点的下降，我们已经看到了对(我们的API中)更高级服务的积极需求，但我们认为整体组合是有价值的，”他表示。

旅游业的大背景是，依赖旅游数据的应用和基于互联网的服务出现了爆炸式增长。Arena表示，在20世纪90年代，Sabre使用其数据平均每秒进行3000次交易，这是衡量这种增长的一个指标。今天，这个数字已经膨胀到每秒99，000次交易。

Sabre等公司传统上与Travelport、Amadeus和谷歌旗下的ITA等公司竞争，向旅行社和在线旅游预订门户网站提供航班、酒店和旅行计划其他方面的数据。

现在，Sabre及其竞争对手瞄准的网络已经扩大到包括一系列依赖这些数据作为第二信息来源的应用程序——例如，想想Gogobot这样的网站，它主要专注于人们相互分享他们的旅行计划，但也让你预订住宿的地方。此外，考虑一些像谷歌有一个非常直接的链接到消费者的方式，其搜索门户网站。

Sabre已经与使用其基于API的数据的传统合作伙伴开展了相当广泛的业务。Arena表示，该公司每天处理超过15亿笔交易，每年花费1000亿美元。现在，它希望其基于“智能”和语义算法的新API将有助于它成为更广泛对话的一部分。

那是什么意思？Sabre给出的新一轮搜索的例子涵盖了六个月的机票购物、数百个备用机场的搜索，以及按预算或“主题”购物。

Arena将其新技术描述为“正在申请专利的扩展……其工作方式是，它采用数百万预先计算和众包的旅行选项，这些选项来自我们的搜索算法，并将它们同步到云环境中，云环境可以以闪电般的速度询问大量数据。这是非常突破性的专业知识。”

对消费者来说，结果将是更多、更高效的服务，符合我们更自然地看待旅行的方式。例如，现在Sabre的网站上有API和代码，允许开发者创建一个服务，让人们根据特定的预算、海滩目的地或试图购买机票的最佳时间的细节进行搜索——利用门户网站在特定时间提供飙升价格的想法。如果你仔细想想，这些新的旅行方式涵盖了许多与旅行相关的初创公司或那些希望在旅行中做更多事情的公司也在努力的相同领域。
