Linus Torvalds 宣称英特尔对 Meltdown/Spectre 的修复“完全是垃圾”

原文：https://web.archive.org/web/https://techcrunch.com/2018/01/22/linus-torvalds-declares-intel-fix-for-meltdown-spectre-complete-and-utter-garbage/?ncid=rss&utm_source=tcfbpage&utm_medium=feed&utm_campaign=Feed:+Techcrunch+(TechCrunch)&utm_content=FaceBook&sr_share=facebook

一向直言不讳的 Linus Torvalds 因其在 Linux 系统最深层代码方面的持续工作而闻名，他对英特尔有严厉的言辞和指控。他对英特尔针对 Meltdown/Spectre 问题提出的最新修复方案的评价是:“补丁是完整的，完全是垃圾。”作为一个潜在的调查线索，他建议:“有没有人跟他们谈过，并告诉他们，他们他妈的疯了？”(星号 his。)

这些和其他类似的绰号是托瓦尔兹在的一个公开电子邮件链中授予的，这是他和英国亚马逊的工程师大卫·伍德豪斯之间的一个电子邮件链，认为英特尔的解决方案与 Linux 内核有关。托沃兹认为，问题在于(据我所知，有些人已经力不从心了)一个笨拙且“疯狂”的修复实现，它本质上什么都不做，同时还做了一堆不必要的事情。

该修复需要解决 Meltdown(主要影响英特尔芯片)，但它不是全面解决问题，而是让用户或管理员在启动时选择整个修复。为什么还要问，这是不是一个巨大的漏洞？当未来的 CPU 可能不需要它时，为什么要在如此低的级别进行呢？在这种情况下，这种选择充其量是不必要的，在最坏的情况下会产生误导或导致性能问题。

与此同时，托瓦尔兹指出，同一个补丁中添加了许多其他东西，这些东西对于现有的解决方案来说是多余的，例如，添加了针对已经被谷歌 Project Zero 的“retpoline”技术缓解的漏洞的保护。

为什么要这么做？托沃兹推测，英特尔技术的主要部分，在这种情况下“间接分支限制推测”或 IBRS，是如此低效，以至于普遍推出它会导致广泛的性能打击。因此，相反，它使主要熔毁修复可选，并增加了多余的东西，使补丁看起来更全面。

英特尔真的打算把这玩意变成建筑吗？有人跟他们说过，告诉他们他们疯了吗？

他们确实做了疯狂的事情。他们做一些没有意义的事情。这使得你所有的[即伍德豪斯的]论点都成了问题和可疑的。补丁做一些不正常的事情。

…所以这里有人没有说实话。有人正在推动完全垃圾不明原因。抱歉，不得不指出这一点。

伍德豪斯(他以一种长期忍受的方式要求他们“不要再大喊大叫了”)，在后面的帖子中承认了托沃兹的批评，称 IBRS 是“卑鄙的黑客”，并同意“T2 没有理由选择加入”但他指出了一些观点，如果不是完全支持英特尔的方法，至少可以解释一下。

英特尔方面发表了以下声明:“我们认真对待行业合作伙伴的反馈。我们正积极与 Linux 社区(包括 Linus)合作，寻求共同解决方案。”所以至少他们看起来仍然是直呼其名。

无论如何，这都是非常深入的讨论，实际上只是其中的一小部分。我没有强调这一点，因为我认为这在技术上很有趣(我真的没有资格这么说)，或者在用户将看到什么方面很重要(目前很难说)，而是简单地指出 Meltdown/Spectre 的崩溃远未结束——事实上，它才刚刚开始。

几周前我们看到的是最初的疯狂浪潮和正在建立的第一道防线。但是，保护受这些问题影响的数十亿设备的工作将持续多年，因为像这样的冲突会自行解决。Linus Torvalds，尽管他的批评通常是亵渎性的，但他是许多为开源社区努力工作的人之一，也是最终从中受益的人之一。

如果没有注重细节、不讲废话、守旧派的程序员在那里提防着像你和我这样的人，以及 userland 中那些自鸣得意的无知的人，我们将不得不接受英特尔和其他人交给我们的任何东西，并感谢他们的无知。就我个人而言，我很高兴有比我更聪明、更坚定的人为我们而战，不管他们有多“大喊大叫”。
