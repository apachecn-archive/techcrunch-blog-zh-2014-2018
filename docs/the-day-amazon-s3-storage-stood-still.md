# 亚马逊 S3 存储停滞不前的那一天|技术危机

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/03/01/the-day-amazon-s3-storage-stood-still/>

到目前为止，您可能已经听说亚马逊的 S3 存储服务昨天在北弗吉尼亚州的数据中心“T1”瘫痪了 4 个多小时，并带走了一些著名网站和服务的部分内容。

值得注意的是，截至今天早上，[亚马逊仪表盘](https://web.archive.org/web/20230326071054/https://status.aws.amazon.com/)显示一切正常。

虽然昨天的停电对受影响的人来说是一件大事，但重要的是要记住，多年来，S3 被证明非常可靠。尽管北弗吉尼亚州的数据中心关闭，但 S3 在其他 13 个地区仍保持正常运行。

为了提供一些额外的视角， [CloudHarmony，](https://web.archive.org/web/20230326071054/https://cloudharmony.com/)一项跟踪云中断的服务报告称，S3 通常已经超过了[其服务级别协议(SLA)](https://web.archive.org/web/20230326071054/https://aws.amazon.com/s3/sla/) ，该协议承诺该服务将在 99.9%的时间内可用，并在不可用时提供退款。CloudHarmony 发现，自该公司 2014 年开始监控云服务以来，S3 在大多数情况下实现了 100%的年可用性。值得注意的例外是 2015 年 8 月 S3 的一次停电。

该公司指出，它跟踪了 2 月 19 日[微软 Azure 虚拟机和对象存储的类似中断，持续了 5 个多小时，但没有得到昨天事件的关注。](https://web.archive.org/web/20230326071054/https://azure.microsoft.com/en-us/status/history/)

云计算分析师兼评论员本·凯普斯(Ben Kepes)表示，西雅图 AWS 总部的员工昨晚可能没睡好觉，但这类停电肯定会时有发生。“就许多数量级而言，AWS 是最大的公共云供应商。因此，任何停电对整个市场都有很大影响。如果说有什么不同的话，那就是这次停电显示了有多少第三方依赖 AWS 作为他们的基础设施。尽管听起来令人不快，但事实是失败时有发生，组织需要为失败做好计划，”他说。

Kepes 补充道，任何 IT 专业人士都知道，停机在任何地方都会发生。它们通常不像碰巧遇到像 AWS 这样受欢迎的供应商时那样引人注目。“虽然人们会对此感到焦虑，但事实是，各种基础架构都会发生宕机，”他说。

涵盖云行业的 Forrester 分析师 Dave Bartoletti 同意这一观点，并表示这是对服务的一个警醒，无论存储在哪里，都要将冗余存储到存储中。“你可以在多个地区存储你的数据，或者你可以建立一个不仅仅依赖 S3 作为唯一存储的网站，”他解释道。

这些分析师说这话的时候并不是在责怪受害者。这是 IT 专业人员多年来一直在他们的系统中构建的一种冗余。他们所说的是云真的没有任何不同。

但 Moor Insight & Strategy 的分析师帕特里克·穆尔黑德(Patrick Moorhead)不准备这么宽容，他说，该说的都说了，该做的都做了，这次中断可能会造成数百万美元的停机时间，亚马逊必须更好地代表客户设置这些冗余系统。

“仅仅因为这是公共云的一部分，我不认为我们应该期待这样的事情发生。这在银行业等行业很少发生，因为容错是内置在架构中的，而不是一个插件，”他说。

虽然这可能是也可能不是一个公平的批评，但事实是，无论您拥有何种类型的基础架构，是内部数据中心还是 AWS 这样的云服务，停机都是不可避免的。没有一种方法是可靠的。这并不意味着 AWS 逃脱了责任，但如果你看看它的记录，它可能值得怀疑。
