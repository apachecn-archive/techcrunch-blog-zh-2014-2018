在Salesforce工作了20年后，Marc Benioff对云技术的看法是对是错

原文：https://web.archive.org/web/https://techcrunch.com/2018/06/17/after-twenty-years-of-salesforce-what-marc-benioff-got-right-and-wrong-about-the-cloud/?utm_source=dlvr.it&utm_medium=twitter

随着我们进入Salesforce的第二十个年头，有一个有趣的机会来回顾Marc Benioff通过推出Salesforce.com为企业软件创建的软件即服务(SaaS)模型所带来的变化。

这种模式已经被SaaS公司的年度收入流所验证，据大多数估计，这一收入流已经接近1000亿美元，而且在未来几年里，它可能会继续改变许多发展较慢的行业。

然而，对于IT的基石市场——大型企业软件交易——来说，根据大多数市场估计，SaaS占总收入的比例不到25%。这种分裂甚至在最近微软高调收购GitHub的【SaaS】中也很明显，GitHub超过50%的收入 来自于出售他们的本地服务、 GitHub企业 。

数据隐私和安全也正在成为一个重要问题，贝尼奥夫本人甚至推动美国隐私法与欧盟的GDPR隐私法相提并论。虽然消费者数据通常是此类讨论的焦点，但值得记住的是，SaaS提供商代表他们的客户存储和处理数量惊人的个人数据，这些数据的内容远远超出了销售线索的电子邮件地址。

是时候在现代背景下重新考虑SaaS模式了，整合过去近二十年的发展，这样企业软件才能发挥其全部潜力。更具体地说，我们需要考虑IaaS和“云原生计算”对企业软件的影响，以及它们如何模糊SaaS和内部应用之间的界限。随着企业软件世界的转变和构建工具的进步，我们真的需要如此明显的区分什么可以在哪里运行吗？

最初的云软件论文

在他的书《云背后的中，贝尼奥夫列出了引入基于云的SaaS模式的四个主要原因:

  1. 通过创建基于订阅的定价模式，使供应商的成功与客户的成功相一致，该模式会随着每个客户的使用而增长 (提供“落地和扩展”的机会)。以前，软件许可证通常要花费数百万美元，并且是预先支付的，此后客户每年都必须支付20%的额外支持费用。这种传统的定价结构对采用造成了巨大的财务障碍，并使采购变得痛苦和漫长。
  2. 在浏览器中放置软件会扼杀客户端-服务器企业软件交付体验。Benioff认识到消费者越来越习惯于使用网站来完成复杂的任务。通过使用浏览器，Salesforce避免了复杂的本地客户端安装，并允许随时随地在任何设备上访问其软件。
  3. 通过利用多租户架构，在多个客户之间分担昂贵计算资源的成本。 这确保了没有个人客户需要投资昂贵的计算硬件来运行给定的单片应用。举个例子，在1999年，一千兆字节的内存需要1000美元，一千兆字节的硬盘需要30000美元。贝尼奥夫举了一个典型的企业硬件采购案例，为了运行Siebel的CRM产品，该产品可能为200个最终用户提供服务，需要花费385，000美元。
  4. 通过消除安装、维护和升级挑战，使软件的可用性民主化。凭借他在甲骨文公司的背景，他列举了花费 6-18个月 完成安装过程的经历。此外，升级因其复杂性而臭名昭著，并会给客户带来大量停机时间。管理企业应用程序是一个非常手动的过程，通常每个IT组织都成为运营团队，为他们购买的每个应用程序执行物理操作手册。

这些争论或多或少也是基础设施即服务(IaaS)提供商，如亚马逊网络服务(Amazon Web Services)在21世纪中后期的早期提出的。然而，IaaS在比SaaS更深的层次上增加了价值，提供了原始构件而不是最终产品。他们在租用云计算、存储和网络容量方面取得成功的结果是，如果每个人都必须遵循Salesforce几年前的模式，那么更多的SaaS应用程序将成为可能。

突然之间，能够按小时访问计算资源——并且无需大量前期资本投资或管理复杂的客户安装——初创公司以经济、简单和 更快的用户增长的名义放弃了SaaS的软件。

2018年是不一样的It世界

快进到今天，从某些方面来看，贝尼奥夫推动世界走向SaaS是多么有先见之明。在上面列出的四个原因中，贝尼奥夫抓住了前两个

  • 订阅是正确的定价模式: 软件的订阅定价模式已被证明是创造客户和供应商成功的最有效方式。几年前，像微软Office和Adobe Suite这样的强大产品已经成功地从前期模式转变为蓬勃发展的订阅业务。今天，订阅价格是许多软件和服务的标准。
  • 更好的用户体验至关重要: 通过浏览器或瘦的原生移动应用(利用相同的API并通过应用商店无缝交付)访问的软件早已无处不在。IT消费化是一个真实的趋势，它已经将我们个人生活中的习惯带入了我们的商业生活。

然而，在其他领域，今天的情况与1999年时大不相同。特别是，贝尼奥夫拥护SaaS的另外两个主要原因似乎不再那么令人信服了。具有讽刺意味的是，IaaS的规模经济(尤其是当谷歌和微软开始认真地与AWS竞争时)和那些“网络规模”公司内部开发的软件开发实践在刺激这些变化中发挥了主要作用

  • 计算现在很便宜: 计算和存储的成本已经大幅降低，以至于共享资源的成本节约有限。今天，如果你直接购买，1gb的内存大约是5美元，1tb的磁盘存储大约是30美元。云提供商向小用户赠送资源，而 对标准规模的实例每小时只收取几分钱 。相比之下，在Salesforce成立的同时，谷歌正在运行其第一个数据中心——总计算和RAM相当于一部iPhone X。这不是开玩笑。
  • 安装软件现在变得更加容易: 随着持续集成和部署(CI/CD)以及配置管理工具的出现，安装和升级现代软件的过程已经变得自动化。随着容器和微服务的快速采用，云原生基础设施已经成为本地开发的事实标准，并正在成为更加可靠、灵活和可扩展的云部署的标准。例如，由Kubernetes或Docker Swarm编排的打包成一组Docker容器的企业软件可以安装在几乎任何地方，并在几分钟内上线。

贝尼奥夫没有预见到的是

在过去的几年里，还出现了其他一些因素，这些因素引发了这样一个问题:SaaS的传统定义是否真的是唯一的未来定义。这里，具有讽刺意味的事实是，许多推动软件回归自托管和管理的力量可以直接追溯到SaaS本身以及整个云计算的成功:

  1. 云计算现在可以“私有”了:IaaS世界中的虚拟私有云(VPCs)允许企业保持对OS的根控制，同时将机器的物理管理外包给像Google、DigitalOcean、微软、Packet或AWS这样的提供商。这使得企业( ，如Capital One )可以放弃硬件管理以及随之而来的头痛问题，但保留对网络、软件和数据的控制。对于企业来说，获得亚马逊、微软和谷歌的安全态势的必要保证，也比获得世界上成千上万家SaaS供应商的同等保证容易得多。
  2. 监管可以惩罚集中服务: 爱德华·斯诺登泄密事件的一个未被充分认识的后果，以及对脸书等公司有时令人质疑的数据隐私做法的觉醒，是政府和企业越来越多地试图保护自己和公民免受窥探。使用托管在另一个国家或由第三方管理的应用程序会使企业面临一系列法律问题。例如， 欧盟的GDPR法 ，让SaaS公司因其存储的每一条EU-公民数据而承担更多潜在责任，并让企业为其SaaS提供商如何管理数据而受到牵连。
  3. 数据泄露风险比以往任何时候都高:上述观点的一个必然结果是，公司在扩大其SaaS足迹时面临的风险增加。只需要SaaS提供商的一名员工点击错误的链接或安装错误的Chrome扩展，就可以将该提供商的客户数据暴露给犯罪分子。如果一般的大型企业 使用1000个以上的SaaS应用程序 ，而这些厂商 中的每一个平均拥有250名员工 ，那么对于攻击者来说，这就意味着额外的250，000个可能的进入点。T29】
  4. 应用程序的可移植性更强:SaaS革命促使软件供应商开发云优先的应用程序，但他们现在使用技术(如容器)来构建这些应用程序，这些技术可以帮助在任何基础设施上复制这些应用程序的部署。这种向所谓的 云原生计算 的转变意味着，您可以注册在多租户云环境中使用的相同复杂应用程序也可以部署到私有数据中心或VPC中，这比以前容易得多。像BigID，stack rox，dash base等公司正在采取私有云-本机-实例-优先的方法来提供应用程序。与此同时，像Atlassian，Box，GitHub和许多其他人正在过渡到Kubernetes驱动的云原生架构，以便在未来提供这种可选性。
  5. 剧本转向首席信息官: 现在，大公司内的个人和小团队通过选择最能满足他们需求的工具(如GitHub、Slack、HipChat、Dropbox)来推动软件的采用，这些工具通常是SaaS。一旦首席信息官了解到使用的是什么以及它是如何工作的，他们就面临着一个决定，要么限制网络访问以屏蔽it，要么为这些服务申请企业许可证(或最接近的许可证)。这一趋势影响如此之大，以至于催生了一个全新的类别，称为 云访问安全经纪人——另一个需要付费的供应商，一个额外的复杂性层，以及潜在问题的另一个途径。管理这些应用程序的本地版本将控制权交还给首席信息官和CISO。

软件的未来是位置不可知的

随着技术颠覆的步伐加快，上一代SaaS公司面临着与他们曾经取代的传统软件提供商相似的未来。从大型机到云原生(甚至是)计算，首席信息官们的目标一直是在成本、功能、控制和灵活性之间取得恰当的平衡。云原生计算包含各种各样的IT方面，通常强调开源软件，准备以适应新趋势的方式提供这些好处。

当今许多最大的SaaS供应商面临的问题是，他们是在前云原生时代创建和扩展的，这意味着他们背负着一些严重的技术和文化债务。如果他们无法进行必要的过渡，他们将被新一代的SaaS公司(可能还有传统软件供应商)所颠覆，这些公司不知道他们的应用程序部署在哪里，也不知道谁应用了预先构建的自动化来简化管理。这种下一代供应商将把更多的控制权交给最终客户(他们渴望控制)，同时保持供应商对云原生开发和基于云的资源的喜爱。

是的，马克·贝尼奥夫和Salesforce在过去二十年里支持“无软件”运动是完全正确的，因为他们所针对的企业软件模式需要被摧毁。然而，在这个过程中，Salesforce帮助推动了一场云计算运动，这场运动最终改写了企业IT的规则，现在也改写了SaaS本身的规则。
