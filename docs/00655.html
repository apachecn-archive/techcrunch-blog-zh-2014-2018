谷歌云平台获得 SSD 持久磁盘和 HTTP 负载平衡

谷歌的 I/O 开发者大会可能还有几天就要召开了，但这并没有阻止该公司在大会之前为其云平台推出一些新功能。正如谷歌今天宣布的那样，云平台将获得两个长期以来一直在开发者愿望清单上的功能: HTTP 负载平衡和基于 SSD 的持久存储。这两个功能现在都处于有限的预览阶段，想要测试它们的开发者可以在这里注册。

开发人员的应用程序需要固态硬盘每秒进行大量的输入/输出操作，现在他们可以以每月每千兆字节 0.325 美元的固定费用获得这一功能。值得注意的是，这比谷歌对常规持久存储收取的 0.04 美元要贵得多。与亚马逊网络服务不同，谷歌对实际的输入/输出请求不收取任何额外费用。亚马逊也提供基于 SSD 的 EC2 实例，但这些实例没有任何持久存储功能。

虽然标准持久磁盘的读取速度约为 0.3 IOPS/GB，写入速度约为 1.5/GB，但基于 SSD 的服务可获得高达 30 IOPS/GB 的读取和写入速度。基于 SSD 的卷的最大持续吞吐量也明显更高。

 

至于 HTTP 负载平衡功能，谷歌表示，它可以在没有任何预热时间的情况下，将 p 扩展到每秒超过 100 万个请求。它支持基于内容的路由，谷歌特别指出，用户可以在不同地区实现负载平衡，这应该有助于希望减少服务延迟的全球玩家。想要跨实例优化负载请求而不是延迟的开发人员也可以这样做。网络本身只有一个外部 IP 地址，根据谷歌的说法，对于想要使用这一功能的用户来说，这应该使网络操作非常简单。

然而，到目前为止，HTTP 负载平衡还不支持 SSL 协议。想要使用该功能的开发者将不得不使用谷歌现有的基于协议的网络负载平衡系统。
