Quiqup 与 Hungryhouse 合作，让伦敦的外卖外包配送 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2017/01/23/quiqup-partners-with-hungryhouse/?ncid=rss&utm_campaign=Steve%27s%20ITK&utm_medium=email&utm_source=Revue%20newsletter

总部位于伦敦的外卖初创公司 Quiqup ，提供类似于美国 Postmates 的应用和服务，与外卖订餐服务 hungryhouse 合作，让合作餐厅外包外卖。这可以是在高峰时段增加额外的送货能力，也可以是让外卖员完全取消他们自己的送货司机。

这种想法认为，快速服务餐馆并不真的想参与最后一英里的物流业务，而是可以专注于他们——据说——最擅长的领域，即准备和出售食物。

具体来说，这种合作关系让 hungryhouse restaurants 可以按需使用 Quiqup 的自行车、滑板车和汽车，以及它的“基础技术和客户支持服务”，类似于这家送货初创公司之前签署的其他 B2B 交易，这是一个它一直在悄悄扩展的业务领域，与它面向消费者的“购买和交付我任何东西”应用程序并列。

它还看到 Quiqup 的 API 被集成到 TabLink 中，TabLink 是 hungryhouse 基于平板电脑的应用程序/设备，允许外卖通过该服务接受订单，从而允许餐厅立即从 Quiqup 请求送货司机来响应这些订单。(顺便说一句，我听说汉堡王与 Quiqup 有类似的 API 级合作关系)。

在一份声明中，Quiqup 的首席执行官兼联合创始人 Bassel El Koussa 评论说:“这种合作关系是 Quiqup 如何以最适合他们的方式与任何企业合作的一个很好的例子。无论是卖外卖食品、杂货还是衣服，
我们的合作伙伴都能够通过将我们的技术集成到他们现有的系统中，快速、轻松地接入我们的基础设施。

然而，那些关注伦敦/英国食品配送领域的人不应该对这种合作关系感到太惊讶。直到最近，hungryhouse 一直由 Delivery Hero 所有(在等待监管机构批准的情况下，它将被出售给竞争对手 Just Eat), Delivery Hero 也是 Quiqup 的早期投资者。

在后者的首轮融资中，两家公司谈到了潜在的合作关系。“鉴于 Quiqup 在最后一英里交付方面的专业知识和 Delivery Hero 的全球规模，这两个集团之间有广泛的协同作用和知识可以分享，”新闻稿中说。

然而，更值得注意的是，这不是第一次创业公司试图提供外卖，这些外卖已经提供了解雇或增强自己司机的机会，以支持外包。在以相当壮观的方式内爆之前，Delivery Hero 自己的 Valk 舰队曾有过一次很好的尝试。

最近，Just Eat 宣布 T2 将与 Stuart 合作，在其市场上提供同样的外卖服务。考虑到 Just Eat 目前正在收购 hungryhouse，而 Stuart 也在伦敦运营着，这是一家需要密切关注的公司。
