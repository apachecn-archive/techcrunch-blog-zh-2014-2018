# TableGrabber 的 RezGuru 帮助餐馆采取更明智的定价和预订方法 

> 原文：<https://web.archive.org/web/https://techcrunch.com/2014/09/09/tablegrabber-resguru-disrupt-launch/>

TableGrabber 旨在帮助餐厅填补他们的空座位，为此，它今天在 Disrupt SF 创业战场推出了一个名为 [RezGuru](https://web.archive.org/web/20221006185312/http://www.rezguru.com/) 的新平台。

正如联合创始人索尼娅和帕万·马尔瓦哈(兄弟姐妹)所说，目标是以最大化收入的方式调整定价和预订可用性等事项。例如，一家餐馆可能非常拥挤，以至于在晚上 8 点到 10 点之间拒绝人们，但在 6 点钟它有一半是空的——所以鼓励人们早一点出现，如果他们出现，也许会给他们提供更好的交易，这难道不是有意义的吗？

Pawan(他也是该公司的“首席执行官”)和 Sonia 带我快速浏览了 RezGuru 仪表盘，上面突出显示了餐厅的当前收入，以及每个可用座位每小时的平均收入。餐厅确定他们希望最大化的数字，然后 RezGuru 将为一天中的不同时间制定定价策略，并详细预测业绩将如何变化。

索尼娅告诉我，RezGuru 是根据 21 个不同的数据点做出这些预测的。一些数据来自餐厅本身(RezGuru 可以与他们的销售点系统集成)，而一些数据来自外部，包括即将到来的活动和天气等。

虽然预测是 RezGuru 的主要卖点，但 TableGrabber 实际上提供了一个完整的内容管理系统，因此它可以使用餐厅的照片和数字化菜单在网络、移动设备以及 Seamless 和 GrubHub 等服务上创建一个存在。(在印度、巴西和智利，在线预订业务规模不大，TableGrabber 在[运营自己的预订和交易网站](https://web.archive.org/web/20221006185312/http://www.tablegrabber.com/)。)然后，它会动态更新这些渠道中的每一个，以反映更广泛的战略。

实体餐厅的灵活性较小，很可能会有印刷菜单，但帕万说，即使在那里，餐厅也可以添加特殊交易，并创建更多的定制菜单(午餐和晚餐，工作日和周末等)。).

餐厅不仅可以使用 RezGuru 来填补座位，还可以避免浪费。如果他们有几天后会变坏的牛排，他们不必在最后一天疯狂地试图处理掉它们，因为 RezGuru 可以预测可能的消费量，并建议是否需要降低价格以更快地出售牛排。

这也不仅仅是价格的问题。帕万说，RezGuru 还可以推荐一些策略，比如将四人桌分成两人桌(如果大多数需求来自情侣)，或者提前一小时关门(因为迟到的顾客没有带来足够的钱来证明继续营业是合理的)。

我对餐饮业了解不多，但最近 Priceonomics 的一篇博客文章给我留下了深刻的印象，这篇文章讲述了餐馆对疏远常客是如何的敏感。所以我想知道如果 RezGuru 的客户采用这种动态定价，他们是否会冒这个风险。帕万指出，一般来说，顾客不会支付高于标准菜单价格的价格——他们将看到的是折扣和交易。

“另一方面，当我坐在空无一人的餐厅里，却仍然要付全价时，我觉得被骗了，”索尼娅补充道。

除了今天为美国餐馆推出 RezGuru 之外，TableGrabber(成立于新德里)正在帕洛阿尔托开设一个办事处。

**问&答评委**

问:这一领域不是有一家大公司吗？
答:OpenTable 只是一个预订系统，而不是像我们一样的预测系统。

问:我不喜欢在高峰时间支付更多费用的想法。是的，我们习惯在航空公司或酒店里这样做，但在餐馆里不会。我们不鼓励餐馆提高价格，但在高峰时段，他们可以为承诺最低消费的人留出几张桌子。

问:如果一家餐馆想提供一笔交易，你如何让他们意识到这一点？答:我们不在 B2B 市场，但我们有开放的 API 来更新其他服务。

问:你是否完全依赖第三方应用来促进交易？
答:我们在其他国家创建了自己的消费者平台 TableGrabber，但在美国，我们想发起自己的战斗。
