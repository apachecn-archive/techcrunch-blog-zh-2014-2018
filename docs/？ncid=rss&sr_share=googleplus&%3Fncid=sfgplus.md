# 甲骨文加入云计算原生计算基金会成为白金会员

> 原文：<https://web.archive.org/web/https://techcrunch.com/2017/09/13/oracle-joins-the-cloud-native-computing-foundation-as-a-platinum-member/?ncid=rss&sr_share=googleplus&%3Fncid=sfgplus>

甲骨文今天宣布[加入](https://web.archive.org/web/20230224212417/https://www.cncf.io/about/members/)云计算原生计算基金会(CNCF)成为白金会员。随着这一举措，它加入了亚马逊、思科、谷歌、IBM、英特尔、微软和 RedHat 等公司，成为基于 Linux 基金会的团体的顶级赞助商，该团体是 Kubernetes 容器编排项目和相关工具的家园。

CNCF 的会员费可不便宜。白金会员支付 37 万美元的会员费(现有 Linux 基金会会员有折扣)，所以加入基金会是一个公司支持 Kubernetes 生态系统的明确标志。

事实上，Oracle 不仅加入了该基金会，还将 Kubernetes 引入 Oracle Linux，并为 Oracle 云基础设施开源其 Terraform Kubernetes 安装程序。值得注意的是，Oracle 已经为 Kubernetes 和相关的容器工具做出了[许多贡献](https://web.archive.org/web/20230224212417/https://blogs.oracle.com/developers/kubernetes-community-engagement-time-to-roll)，所以在许多方面，今天的举动只是正式确定了它在生态系统中的现有投资。

“甲骨文在满足世界级企业的需求方面有着数十年的经验，”云原生计算基金会首席运营官克里斯
Aniszczyk 在一份声明中说。“我们很高兴甲骨文加入 CNCF 成为白金会员，并相信他们的关键角色将有助于定义企业云的未来。”

在这一点上，Kubernetes 无疑将赢得容器编排之战，因为几乎每个主要参与者都在支持这个项目，无论是财政上还是代码贡献上。

除了宣布 Oracle 成为新成员之外，CNCF 今天还宣布了第一批 Kubernetes 认证服务提供商，这些提供商展示了对 container orchestrator 的深刻了解，以帮助企业采用它(这些提供商是埃森哲、博思艾伦汉密尔顿、Canonical、CoreOS、Giant Swarm 和三星 SDS)。

该集团还向其开源工具系列增加了两个新项目: [Jaeger](https://web.archive.org/web/20230224212417/https://github.com/uber/jaeger) ，一个分布式追踪系统，和 [Envoy](https://web.archive.org/web/20230224212417/https://lyft.github.io/envoy/) ，一个来自 Lyft 开发团队的 edge 和服务代理。

甲骨文今天也发布了另一个开源声明:该公司将把之前关闭源代码的 Java 企业版(Java EE)平台转移到 Eclipse Foundation 中，而[代码现在在 GitHub 上](https://web.archive.org/web/20230224212417/https://github.com/javaee/)。
