不出所料，微软看好开发者对其新工具的反应

今天早上，微软在三藩市的 Build 大会上宣布了一系列新的开发者工具 T1 和 T2 其他好东西 T3。在冗长的主题演讲中，观众的反应似乎对舞台表演非常友好。

今天下午尘埃落定后，我和微软公司副总裁索玛·索玛瑟加坐在一起，听取他的意见——如果你愿意的话，可以说是成绩单——关于开发者如何看待他参与其中的微软新发布的成果。他认为迄今为止的反应“非常好”

具体来说，Soma 指出，宣布允许所谓的“通用应用程序”的工作已经受到欢迎，这些应用程序将看到 Windows 和 Windows Phone 应用程序之间共享的代码大大增加。这很公平，符合我自己听到的。

Soma 还表示，Roslyn 得到了开发者的强烈回应。你会问罗斯林是什么？Soma 称之为编译器即服务，微软今天宣布它将成为一个开源工具。在 Soma 看来，编译器类似于“黑匣子”。Roslyn 的想法是通过让开发人员完全接触到事物的内部来改变这种情况。

我向 Soma 询问了 Azure、微软云存储及其计算服务的发展势头。据他估计，当 Azure 引入基础设施即服务(IaaS)功能时，它达到了一个转折点。市场对 Azure 的兴趣正在“突飞猛进”地增长。

上述情况表明，微软很乐观，它今天构建、发布和宣布的东西很受它极力争取的人的欢迎。

有趣的是，我们将能够独立审查市场反应:如果开发速度加快，更多的应用程序发布到微软的平台上，它的赌注就成功了。如果我们看到一个扁平化，甚至下降，微软将会失败。此外，轻微的倾斜是不够的。微软需要大大加快开发者为其更大的 Windows 平台构建和发布应用的速度。
